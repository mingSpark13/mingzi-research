# 📖 花火格式笔记

## 🎯 题目

**Monocular Open Vocabulary Occupancy Prediction for Indoor Scenes**

## 📝 三句摘要

1. **问题背景**：现有语义占据预测方法在封闭词汇（Closed-vocabulary）设定下只能识别训练预定义的有限类别，无法处理真实场景中开放且长尾的语义查询（如"带条纹的马克杯"、"破损的木椅"）。
2. **核心方法**：提出 LegoOcc，以语言嵌入高斯（Language-embedded Gaussians）作为统一三维表示，在仅使用二值占据标签训练的情况下，支持任意自然语言文本查询，实现开放词汇三维语义占据预测。
3. **关键结果**：在 Occ-ScanNet 数据集上达到 21.05 mIoU 和 59.50 IoU，mIoU 相比此前最强开放词汇模型提升超过 2 倍，同时几何占据预测精度超越多种封闭词汇方法。

## 💎 价值评估

- **🔬 研究价值**：
  - 首次实现单目开放词汇占据预测，降低了对昂贵 3D 标注数据的依赖
  - 将语言嵌入与 3D 高斯表示结合，为开放词汇 3D 感知提供了新思路
  - CVPR 2026 Oral，表明工作获得顶会高度认可
- **🚀 实践价值**：
  - 仅需几何标签（占据标签）训练，大幅降低数据标注成本
  - 支持任意文本查询，可直接用于家庭机器人等开放场景目标定位
  - 单目输入降低了硬件成本，有利于落地部署
- **📈 扩展潜力**：
  - 可扩展到室外场景和更大规模环境
  - 可与 VLA 模型结合实现开放词汇任务规划
  - 可探索将语言嵌入高斯扩展到其他 3D 感知任务（深度估计、位姿估计等）

## 🎯 可落地实验点

**实验设计：开放词汇物体定位与抓取验证**

- **任务**：在真实室内场景中给定自然语言目标描述（如"找到那个红色杯子"），验证 LegoOcc 能否正确定位目标物体的 3D 占据区域
- **对比基线**：SSC（Semantic Scene Completion）方法 + CLIP嵌入方案
- **度量指标**：
  - 开放词汇 mIoU（文本查询类别IoU）
  - 定位精度（3D Bounding Box Acc@0.5m）
  - 推理速度（FPS）
- **预期结果**：LegoOcc 在长尾/开放类别上显著优于封闭词汇方法，推理速度达到实时（>15 FPS）
- **数据集**：使用 HSSDN（Home Social Scene Description Network）或自采集家庭场景

## 📌 待探索问题

- **问题1**：LegoOcc 的语言嵌入高斯表示能否扩展到动态场景（如人体跟踪、移动物体）？当前方法假设静态场景。
- **问题2**：在复杂遮挡和部分可见情况下，文本查询的歧义性如何处理？例如"那个杯子"在多个杯子共存时如何消歧？
