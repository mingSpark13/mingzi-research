# 📖 花火格式笔记

## 🎯 题目

LiveVLN: Breaking the Stop-and-Go Loop in Vision-Language Navigation

## 📝 三句摘要

1. **问题背景**：很多 VLN 系统 benchmark 分数不错，但真实部署时仍会“走一下停一下”，根因不是模型不够准，而是 sense-inference-execution 串行阻塞导致动作续航不连续。
2. **核心方法**：LiveVLN 提出一个 **training-free runtime framework**，把动作序列拆成 executed prefix、guard buffer 与 revisable tail，用 guarded handoff + real-time adaptation 在机器人执行当前动作时并行刷新下一段动作。
3. **关键结果**：在 R2R/RxR 与真实部署上基本保持任务性能，同时把平均等待时间最多减少 **77.7%**，并将 wall-clock episode time 在 StreamVLN / NaVIDA 上分别缩短 **12.6% / 19.6%**。

## 💎 价值评估

- **🔬 研究价值**：这篇很妙，不卷新 backbone，直接切运行时层，把“导航性能”从精度问题扩展到系统时延问题，补了 D04/D06 常被忽略的 deployment gap。
- **🚀 实践价值**：对主人未来做 UAV 语言导航或在线规划很有启发——很多卡顿未必来自策略差，而是闭环刷新节拍设计得烂。
- **📈 扩展潜力**：guard buffer / revisable tail 机制可迁到空中 VLN、VLA 在线控制甚至 MPC-VLM 混合系统，适合和实时推理框架结合。

## 🎯 可落地实验点

**实验设计**：把主人现有无人机语言导航/在线决策系统做成“阻塞执行 vs guard-buffer 连续执行”对照。
- 对比基线：标准串行 sense-inference-execution / 固定长度 action chunk
- 度量指标：平均等待时长、pause count、轨迹平滑度、任务成功率
- 预期结果：在不改主模型的前提下，运行时并行刷新策略即可显著降低停顿并缩短总任务时长
