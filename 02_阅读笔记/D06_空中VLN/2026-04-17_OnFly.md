## 三句话摘要

1. OnFly 提出共享感知双 agent 架构，解耦高频目标生成与低频进度监控，稳定化 VLM 决策，解决「单流 VLM 决策不稳定」和「长时程进度监控不可靠」问题
2. 混合 keyframe-recent-frame 记忆保留全局轨迹上下文同时维持 KV-cache 前缀稳定性，语义-几何验证器对齐 VLM 预测目标，几何安全约束下生成优化无碰轨迹
3. 仿真中任务成功率从 26.4% 提升至 67.8%，超越最强基线；真实机载飞行验证实时部署可行性

## 核心贡献

- **双 agent 架构**：高频目标生成（感知）+ 低频进度监控（规划）解耦，频次分离设计
- **混合记忆机制**：keyframe 全局上下文 + recent-frame KV-cache 稳定性，平衡记忆长度与计算效率
- **语义-几何验证器**：VLM 特征 + 深度线索双重校验目标指令一致性 + 几何安全性
- **滚动时域规划器**：几何安全约束下生成优化无碰轨迹，安全与效率兼得

## 关键结果

- 仿真：成功率 26.4% → 67.8%（vs 最强基线）
- 真实机载零样本部署验证可行性
- Jetson Orin NX 延迟：Decision Agent 0.81s（4.73x 加速），Monitoring Agent 1.15s（6.50x 加速）

## 对 D06 的价值

- **零样本 + 机载**是 D06 最高难度验收位，OnFly 直接提供参考基线
- **双 agent 频次解耦**是解决 VLM 推理延迟与控制频率不匹配的有效方案
- **语义-几何验证器**思路可迁移到 D06 的「感知掉线 vs 探索失效」区分

## 局限 / 不足

- 零样本泛化依赖 VLM 能力，未在极远分布场景测试
- 进度监控仍依赖语言对齐，语义歧义场景可能失效
- 延迟 0.81s 对高速飞行仍偏慢
