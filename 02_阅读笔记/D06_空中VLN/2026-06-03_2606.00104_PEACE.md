# 📖 花火格式笔记

## 🎯 题目

PEACE: A Planner–Executor Agent with Constraint Enforcement for UAVs

## 📝 三句摘要

1. **问题背景**：LLM 驱动无人机若放在紧耦合控制环里，容易带来推理时延、幻觉动作和难审计的安全风险，端到端 VLA 方案也依赖大量任务专属数据。
2. **核心方法**：论文提出 PEACE，用单次 LLM 生成完整 mission plan，再由 ROS 2 tool-calling executor 通过 MAVLink 确定性执行，同时接入 2D 检测+深度投影构造 3D world model，并用独立约束层校验高度限制与水平地理围栏。
3. **关键结果**：在 PX4 + Gazebo SITL 中，PEACE 相比紧耦合 LLM control 更可解释、更易插入安全护栏且显著减少 LLM 调用次数，适合作为空中任务规划代理的安全工程范式。

## 💎 价值评估

- **🔬 研究价值**：这篇不是单纯“把 LLM 接到无人机上”，而是把 planner–executor、world grounding、constraint enforcement 三件事拆清了，对 D06 很有参考价值。
- **🚀 实践价值**：主人若做 PX4/ROS2 无人机智能体，这套 typed mission plan + safety validator 的架构几乎可以直接借鉴到工程原型里。
- **📈 扩展潜力**：后续可接视觉语言感知、语义目标搜索、多机协作，甚至与空中 VLN/VLA 形成“高层规划 + 低层技能”混合系统。

## 🎯 可落地实验点

**实验设计**：在主人现有 UAV 栈里复现“单次规划 + 确定性执行 + 安全约束校验”三段式代理，比较其与 ReAct 式循环调用在延迟和安全违规率上的差异。
- 对比基线：ReAct 式 LLM tool loop、纯手写任务状态机、无 safety validator 的 planner-executor
- 度量指标：任务完成率、LLM 调用次数、平均决策时延、越界/碰撞违规率、失败重规划成功率
- 预期结果：PEACE 式结构在长任务和安全敏感场景下更稳，尤其能显著降低高频 LLM 调用带来的不确定性
