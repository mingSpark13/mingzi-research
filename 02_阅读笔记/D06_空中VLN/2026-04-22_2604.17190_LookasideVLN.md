# 📖 花火格式笔记

## 🎯 题目

LookasideVLN: Direction-Aware Aerial Vision-and-Language Navigation

## 📝 三句摘要

1. **问题背景**：现有空中 VLN 方法更依赖地标描述和多级 lookahead 搜索，对方向词理解浅、计算开销也偏高，长指令导航容易走偏。
2. **核心方法**：LookasideVLN 显式建模语言中的方向线索，构建 Egocentric Lookaside Graph、Spatial Landmark Knowledge Base 和导航代理，把地标与方位关系一起送入多模态规划。
3. **关键结果**：论文在单层 lookahead 条件下就显著超过 CityNavAgent，说明“方向感”本身就是高性价比的空中导航增益，而不只是堆更深搜索。

## 💎 价值评估

- **🔬 研究价值**：它把 D06 里常被忽略的方向词从“提示细节”提升成主建模对象，补上语言空间关系理解这一块短板。
- **🚀 实践价值**：对主人做 UAV 视觉语言导航很直接，尤其适合拿来改现有语义-几何融合链路，而不必一上来就堆更重的 memory graph。
- **📈 扩展潜力**：后续可以和 FineCog-Nav 的模块化认知栈拼接，用方向图做规划先验、用细粒度模块做错误归因，形成更稳的机载导航系统。

## 🎯 可落地实验点

**实验设计**：在主人现有空中 VLN 栈中加入“方向词解析 + 地标方向图”模块，验证是否能提升长指令执行稳定性
- 对比基线：CityNavAgent 风格 landmark-only 规划 / 现有 D06 语义几何导航器
- 度量指标：导航成功率、子句完成率、平均偏航误差、每步推理时延
- 预期结果：加入方向关系建模后，复杂转向和多阶段指令场景成功率明显提升，且比深层 lookahead 更省算力
