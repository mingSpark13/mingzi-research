# 📖 花火格式笔记

## 🎯 题目

LEXI-SG: Monocular 3D Scene Graph Mapping with Room-Guided Feed-Forward Reconstruction

## 📝 三句摘要

1. **问题背景**：现有机器人语义建图大多依赖深度相机或激光雷达，单目 RGB 做开放词汇 3D scene graph 既容易尺度漂移，也很难兼顾全局一致性和导航可用性。
2. **核心方法**：LEXI-SG 先用 foundation model 语义先验把场景切成 room-level 单元，再在“房间完整观测后”触发 feed-forward reconstruction，并用 room-based factor graph 对各房间重建做全局对齐，同时维护开放词汇物体分割、跟踪与层次 scene graph。
3. **关键结果**：作者在 Habitat-Matterport3D 和自采办公室序列上证明，LEXI-SG 相比已有 feed-forward SLAM 与 scene-graph baseline 拿到更稳的轨迹估计、更密更一致的 3D 地图，以及更适合导航与语义推理的层次表示。

## 💎 价值评估

- **🔬 研究价值**：这篇把单目重建、房间级全局对齐和 scene graph 统一起来，正好补“语义导航依赖廉价传感器却缺少高质量 3D 语义地图”这条关键链路。
- **🚀 实践价值**：对主人后面做室内 UAV/UGV 语义导航很有用，尤其适合“先快速探索、再补全局语义地图、最后做语言/任务规划”的三级系统设计。
- **📈 扩展潜力**：后续可以把 room-based factor graph 接到空中 VLN 或主动探索策略里，看能不能把“哪里还没看清、该先飞哪间房”也变成规划信号。

## 🎯 可落地实验点

**实验设计**：在主人后续室内导航/空中探索任务里，对比“纯 2D 语义地图”与“room-aware 单目 3D scene graph”对长程目标搜索的帮助。
- 对比基线：2D 语义占据图、传统单目 SLAM、无房间分块的 feed-forward 重建
- 度量指标：目标定位成功率、建图漂移、开放词汇物体查询准确率、导航路径长度、推理延迟
- 预期结果：room-aware 3D scene graph 在未见房间和复杂语义目标查询时更稳，能明显降低长程搜索回退成本
