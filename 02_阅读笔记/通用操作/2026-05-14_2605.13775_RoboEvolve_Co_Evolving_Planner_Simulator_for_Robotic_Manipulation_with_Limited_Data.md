# 📖 花火格式笔记

## 🎯 题目

RoboEvolve: Co-Evolving Planner-Simulator for Robotic Manipulation with Limited Data

## 📝 三句摘要

1. **问题背景**：机器人操作受限于高质量交互数据稀缺，而单独用 VLM 规划或视频生成做数据合成又常出现语义-空间错位和物理幻觉。
2. **核心方法**：RoboEvolve把 VLM planner 和 VGM simulator 绑成共进化闭环，用探索/巩固双阶段机制、渐进课程与 near-miss failure 挖掘，持续生成更物理可信的训练经验。
3. **关键结果**：论文声称只用 500 张未标注 seed 图像就能超过全监督基线，带来 30 个点的 planner 提升和 48% 的 simulator 成功率提升，数据效率很凶。

## 💎 价值评估

- **🔬 研究价值**：它把“规划器生成意图 + 生成模型模拟交互 + 失败再回灌优化”串成闭环，很像真正可扩张的操作数据飞轮雏形。
- **🚀 实践价值**：这和主人最近关心的合成数据、自动标注、3DGS/仿真闭环特别同频，适合拿来启发低标注操控数据生成管线。
- **📈 扩展潜力**：若把视频生成器替换成更强的 3D/物理仿真模块，可能进一步逼近真实机器人可用的数据闭环平台。

## 🎯 可落地实验点

**实验设计**：构建一个“小型 planner + simulator”闭环，为单一 pick-place 任务自动合成 near-miss 数据
- 对比基线：纯人工示教 / 单独视频生成扩充 / 单独 VLM 规划扩充
- 度量指标：数据利用率、策略成功率、生成轨迹物理可执行率、人工筛选成本
- 预期结果：共进化闭环在少标注场景下比单模块数据扩充更稳，尤其能补齐失败边界样本
