## 🎯 题目

"SOMA: Strategic Orchestration and Memory-Augmented System for Vision-Language-Action Model Robustness via In-Context Adaptation"

## 📝 三句摘要
SOMA 通过对比式双记忆 RAG + LLM Orchestrator + MCP 干预模块，无需微调即可让冻结 VLA 在 OOD 场景下获得长期记忆、因果失败归因和动态干预能力，长时序任务链成功率提升 89.1%。

## 核心贡献
1. **Dual-Memory RAG**：在线对比记忆检索，融合执行轨迹到可靠先验
2. **LLM Orchestrator**：归因驱动的 LLM 调度器，诊断失败原因并触发 MCP 干预
3. **MCP 干预协议**：可扩展的模型上下文协议，无需参数更新
4. 在 pi0/pi0.5/SmolVLA 三个 backbone 上验证，LIBERO-PRO + LIBERO-SOMA

## 三句精华
- VLA 在 OOD 场景失败的根本原因是缺乏长期记忆和因果归因能力，而非运动不可行
- SOMA 用对比记忆检索 + LLM 调度器在推理时增强冻结 VLA，长时序任务平均提升 56.6%，最高单项 +89.1%
- 无需微调任何 VLA 参数，通过 LLM Orchestrator 做失败诊断并调用 MCP 干预模块

## 💎 价值评估
- **学术价值**：⭐⭐⭐⭐（LLM Orchestrator 作为 VLA 失败诊断层的思路非常新颖，IROS 2026 投稿）
- **工程价值**：⭐⭐⭐⭐（直接提升 VLA 鲁棒性，无需 retrain，主人的 FASTER 架构可借鉴）
- **与主人研究方向相关度**：⭐⭐⭐⭐（Paper A 高层 VLM 意图层的 in-context adaptation 方案；LLM Orchestrator 可作为 SafeFlow 安全层的失败诊断/干预模块）

## 🎯 可落地实验点
1. **Paper A 高层 VLM 意图层**：SOMA 的 LLM Orchestrator 作为 VLM 意图层与 SafeFlow CBF 安全层之间的"失败诊断调度器"，接收 SafeFlow 的安全警报并决定是否覆盖 VLA 输出
2. **Paper A 安全层增强**：MCP 干预协议可封装为安全干预接口，SafeFlow 的 CBF 输出直接映射为 MCP call，物理可行性验证由 ACT 层接管
3. **空中机械臂操作**：长时序任务链（主人的空中机械臂操作天然需要多步规划）的 OOD 鲁棒性提升
