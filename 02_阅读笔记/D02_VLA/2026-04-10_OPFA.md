# One-Policy-Fits-All: Geometry-Aware Action Latents for Cross-Embodiment Manipulation

> **arXiv**: 2603.14522 | **方向**: D04_跨载体泛化 | **深挖时间**: 2026-04-10 R600 | **状态**: ✅ 摘要级入库

## 一句话与本方向关系
这篇工作把跨载体统一策略的重点，从“只在 backbone 里塞 morphology token”推进到“先学共享几何感知 latent action space，再统一解码到不同 embodiment 动作”，很适合作为 D04 的 geometry-aware latent 主线代表。

## 核心贡献
1. **Geometry-Aware Latent Representation (GaLR)**：结合 3D 卷积和 transformer，学习可跨 embodiment 共享的 latent action space。
2. **Unified latent retargeting decoder**：从共享 latent 中解出不同 embodiment 的具体动作，不需要每个 embodiment 单独重新调 decoder。
3. **End-to-end co-training across embodiments**：支持不同 gripper、灵巧手、不同自由度平台联合训练，提高数据效率。
4. **Single versatile policy**：目标不是“先迁移后适配”，而是直接学一套可跨 embodiment 使用的统一策略。

## 技术路线
```text
多载体观测 / 动作数据
  → Geometry-Aware Latent Representation (GaLR)
  → 共享 latent action space
  → unified latent retargeting decoder
  → embodiment-specific actions
```

## 可借鉴的技术点
1. **latent 层几何对齐**：跨载体泛化不一定只能靠显式 morphology token，也可以在共享 latent action space 里做几何对齐。
2. **统一 decoder 思路很适合空中机械臂**：如果主人后续要做不同末端执行器/自由度版本，OPFA 这条路比“每个平台单独 head”更省维护。
3. **适合作为 D04 三线对照之一**：可与 paired-data、morphology-aware backbone 形成并列实验轴。

## 局限 / 不足
1. **更偏动作空间对齐**：若任务差异和感知差异同时很大，latent 对齐未必够用。
2. **摘要级结论为主**：后续仍需全文复核 GaLR 具体结构、retargeting 细节和实验边界。
3. **几何先验仍需可靠输入**：若本体几何描述不完整，latent 对齐效果可能打折。

## 对主人的落地启发
- D04 当前可稳定收束成三条主线：**paired-data budget / morphology-aware backbone / geometry-aware latent action space**。
- OPFA 特别适合拿来对照“显式形态先验注入”到底比“latent 几何对齐”更划算，还是二者应组合。
- 对空中机械臂场景，后续可单列一个实验位，比较 **统一 latent decoder** 和 **按 embodiment 单独 head** 在样本效率、跨任务迁移、部署维护成本上的差异。
