# CeRLP: A Cross-embodiment Robot Local Planning Framework for Visual Navigation

> **arXiv**: 2603.19602 | **方向**: D04_跨载体泛化 | **深挖时间**: 2026-04-09 R567 | **状态**: ✅ 已入库（摘要级深挖）

## 一句话与本方向关系
CeRLP 把跨载体泛化从机械臂操作继续前移到视觉导航局部规划层，说明“统一接口”不只可以定义在动作空间，也可以定义在**几何感知到局部 planner 的抽象层**。

## 核心贡献
1. **统一几何表述**：将不同机器人尺寸、相机内参与相机类型的视觉输入统一到同一几何形式，服务通用局部规划。
2. **深度尺度校正**：通过离线预标定修正单目深度的 scale ambiguity，恢复可用的 metric depth。
3. **Visual-to-Scan 抽象**：把异构视觉输入投影为 **height-adaptive laser scans**，让下游 policy 对不同机器人/传感器更鲁棒。
4. **跨配置泛化验证**：仿真与真实世界都覆盖 point-to-point navigation 和 vision-language navigation，证明它不只适用于单一机器人配置。

## 技术路线
```
异构视觉输入
  → 深度估计 + 离线尺度校正
  → visual-to-scan 抽象（height-adaptive laser scan）
  → 通用 local planner policy
  → 跨机器人/跨相机配置部署
```

## 可借鉴的技术点
1. **接口层前移**：跨载体统一不一定从 action token 开始，也可以先把 perception-to-planner 接口统一，再接控制层。
2. **几何中间层比原始 RGB 更稳**：对主人后续空地异构平台共享导航策略，几何抽象层可能比直接共享视觉 backbone 更实用。
3. **离线标定 + 在线统一推理**：先用低成本标定补齐尺度，再让在线策略保持统一输入格式，这条工程链很适合部署场景。

## 局限/不足（我们可改进的点）
1. **更偏局部规划，不是完整任务框架**：它解决的是 local planner 层的跨载体问题，尚未覆盖长程语言推理与任务规划。
2. **依赖深度恢复质量**：若单目深度本身不稳，后续 visual-to-scan 仍会受影响。
3. **对动力学差异覆盖有限**：它主要统一感知与局部避障，还没有显式处理空中平台的动力学约束。

## 对主人当前研究的启发
**C6: 几何接口层跨载体复用**。D04 后续不该只盯操作/抓取，也可以把“统一几何抽象 + 载体特化控制器”作为导航与局部规划层的主线之一。这样能自然衔接 D06 的空中 VLN 中层 explorer / local planner 设计。
