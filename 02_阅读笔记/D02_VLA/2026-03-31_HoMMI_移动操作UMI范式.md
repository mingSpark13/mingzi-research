# 📖 花火格式笔记

## 🎯 题目

HoMMI: Hands-on Multi-arm Mobile Manipulation with UMI —— 如何用 UMI 范式把人类全身移动操作技能迁移到双臂移动机器人

## 📝 三句摘要

1. **问题背景**：UMI（Universal Manipulation Interface）范式在桌面双手操作上效果显著，但往移动操作（mobile manipulation）扩展时面临全身协同难题：底盘移动、腰部姿态、头部感知如何与双手协调。
2. **核心方法**：用 3 台 iPhone 采集人体数据（头+双手），通过 3D 视觉表征 + look-at point 动作抽象 + whole-body IK 控制器，将人类技能迁移到双臂移动机器人，**无需机器人遥操作**。
3. **关键结果**：三个长程真实机器人任务（Laundry/Delivery/Tablescape）上，HoMMI 达到 80-90% 成功率；对比实验证明头部主动感知（Active Neck）和腕部局部信息缺一不可。

## 💎 价值评估

- **🔬 研究价值**：把 UMI 从桌面双手操作扩展到移动操作，提出“手眼意图 + whole-body IK”的跨具身迁移范式，证明主动感知在长程移动操作中是核心变量。
- **🚀 实践价值**：对主人后续空中操作/移动操作研究很有启发，尤其是 look-at point、腕部局部观测和全身协调的解耦设计。
- **📈 扩展潜力**：可继续往空中平台、轻量多传感器采集和更强恢复策略方向扩展。

## 🧩 核心问题：UMI 往移动操作扩展卡在哪里？

### 桌面操作 vs 移动操作：不是同一难度级别

| 维度 | 桌面双手操作 | 移动操作 |
|------|-------------|---------|
| 关节空间 | ~12 DoF（双臂+夹爪） | ~20+ DoF（+底盘+腰部+头部） |
| 感知范围 | 腕部局部视角够用 | 需要全局上下文（导航方向、目标位置） |
| 协调内容 | 双手协同 | 底盘+腰部+双手+头部全身协同 |
| 规划目标 | 手部末端位姿 | 空间锚点（手到哪里+眼到哪里） |

### 两大难点

1. **如何用少数锚点控制全身运动？** → 移动操作的高层目标是"左右手最终到哪里 + 视线落在哪"
2. **观察和动作同时变难**：引入头戴视角会放大具身鸿沟（人臂 vs 机器臂外观/身高/关节完全不同）

## 🔑 核心技术方案

### 1. 数据采集：3 台 iPhone + Apple ARKit

- 2 台装在左右手持夹爪上，1 台装在帽子上
- 60Hz 采集：RGB + 深度 + 6DoF 位姿 + 夹爪宽度
- **关键**：三台设备在同一个全局一致坐标系里同步记录

### 2. 视觉表示：从 RGB 到具身无关的 3D 特征

- **不直接用头戴 RGB**（会放大具身鸿沟）
- 策略输入：3D 点云（patch 特征 + 3D 点位置编码绑定）
- 进一步：转到**左手夹爪坐标系**，遮掉人类手臂和身体点
- 策略关注的不再是人的外观，而是**手和物体、场景的几何关系**

### 3. 动作表示：look-at point（最关键的设计选择）

- **不回归人头 6DoF 姿态**（机器人颈部 DoF 与人不同，物理上不可行）
- 只提取"看向哪里"的信息 → **3D look-at point**
- 保留主动感知意图，绕开头部具身差异最难处理的部分
- 三个空间锚点：`左手末端位姿 + 右手末端位姿 + 3D 注视点`

### 4. Whole-Body IK 控制器

策略输出"手眼意图"，控制器负责实现：
- 双手 SE(3) 末端位姿跟踪（高权重主任务）
- 注视点 → 机器人可实现的云台朝向
- 多层正则 + 约束：
  - Nominal posture regularization（更像人示范的姿态）
  - Current posture regularization（抑制关节突变）
  - CoM-over-base（质心压在底盘支撑范围内，**最高优先级**）
- 硬约束：关节位置/速度边界、底盘速度、自碰撞、躯干竖直

### 5. 系统时序

- 策略服务器（10Hz）和执行桥异步工作
- 观测时间对齐，过期动作丢弃
- IK 频率 100Hz，控制回路 500Hz
- IK 迭代时做时间插值（位置线性插值，姿态球面插值）

## 📊 实验结果

| 方法 | Laundry | Delivery | Tablescape |
|------|---------|----------|------------|
| **HoMMI (Ours)** | **90%** | **85%** | **80%** |
| Wrist-Only | 20% | 15% | 0% |
| RGB-Only | 0% | 45% | 0% |
| Head-Only | 0% | 5% | 0% |
| w/o Active Neck | 75% | 55% | 55% |

### 三个核心发现

1. **RGB-Only 在 Laundry 为 0%**：简单加头戴 RGB 直接预测头部 6DoF 反而放大具身鸿沟，必须做 3D 视觉表示和 look-at point 抽象
2. **Head-Only 在 Delivery 仅 5%**：腕部相机提供的局部接触信息在抓放时不可替代，**全局视角 + 局部视角缺一不可**
3. **w/o Active Neck 分别掉 15/30/25 个百分点**：主动感知不是点缀，尤其在搜索、对位、放置时主动转头收集信息明显提高成功率

## ⚠️ 边界与局限

- **短历史窗口**（To=2）：缺乏长期记忆和失败恢复机制
- **纯视觉**：无力和触觉信号，精细力控任务（拧瓶盖、插接头）可能不够
- **硬件共设计**：相机位置和夹爪几何对齐是关键，换差异更大的机器人当前方案存疑
- **数据规模**：481 条示范验证了可行性，要覆盖更多场景仍需扩展

## 🎯 可落地实验点

**实验设计**：在四旋翼空中操作平台（+ 夹爪）上复现 HoMMI 范式
- **改装方案**：用超轻量相机（Insta360 GO/RealSense）替代 iPhone，用双醋酸酯标记追踪位置
- **核心改造**：从"Look-at Point"思想延伸，空中机器人头部云台跟踪末端夹爪与目标的几何关系，而非复制人头部姿态
- **数据采集**：人在地面演示精细抓取/放置动作（双手 + 头部），录制 200+ 条无机器人示范
- **迁移策略**：空中机器人的 Whole-Body Controller 替代底盘为位置保持（hover），加入末端执行器可达性约束
- **对比基线**：直接腕部相机的 UMI（无头部信息）、无主动感知版本
- **核心预期**：头部主动感知信息对空中操作（如空中抓取、移动中放置）同样关键
