# RDT2: 探索UMI数据缩放实现零样本跨载体泛化

- **arXiv**: 2602.03310
- **方向**: D04 跨载体泛化
- **阅读深度**: 摘要级深挖（候选→入库）
- **日期**: 2026-04-16

## 核心贡献

**RDT2** 是基于 7B VLM 的机器人基础模型，核心三件套：

1. **数据**：10,000+ 小时 UMI（Universal Manipulation Interface）演示数据，跨多机器人族系开源
2. **训练配方（三阶段）**：
   - Stage 1: Residual Vector Quantization (RVQ) — 离散语言知识对齐
   - Stage 2: Flow-matching — 连续动作生成
   - Stage 3: Distillation — 实时推理蒸馏
3. **零样本能力**：未见物体/场景/指令/机器人平台 同时泛化

## 与D04四轴实验表的关系

| D04验收轴 | RDT2对应贡献 |
|-----------|-------------|
| paired-data budget | 🟢 10K小时开源数据集（大规模 paired demo） |
| morphology-aware backbone | 🟡 VLM backbone 跨平台通用，隐式 morphology 无关 |
| geometry-aware latent action | 🟢 RVQ + Flow-matching 统一动作生成（latent action 路线） |
| new-embodiment onboarding cost | 🟢 零样本新载体接入（无需微调） |

## 关键发现

RDT2 的三阶段训练配方值得重点关注：
- **RVQ** 解决了"语言离散知识 ↔ 连续动作"的语义对齐问题
- **Flow-matching** 是扩散模型的替代方案（π0 系列用的是扩散，RDT2 用流匹配）
- **Distillation** 让大模型能实时推理（直接关系到机载部署）

这说明 D04 的 latent action 轴不一定非要走扩散路线，**flow-matching + distillation** 可能是更高效的零样本跨载体方案。

## 对主人研究的价值

RDT2 的 post-training 零样本路线与 LAP (语言动作预训练) 可做对比：
- LAP：语言作为跨载体统一接口
- RDT2：流匹配+蒸馏作为零样本迁移机制

两者都是 **无需目标载体训练数据** 即可泛化，适合与 paired-data 路线做正面对照。
