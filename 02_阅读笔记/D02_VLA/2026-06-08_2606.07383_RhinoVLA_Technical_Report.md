# 📖 花火格式笔记

## 🎯 题目

RhinoVLA Technical Report

## 📝 三句摘要

1. **问题背景**：现有 VLA 在真实机器人上常被视觉 token 数和大上下文拖慢，边端闭环部署很难稳定跑到实用控制频率。
2. **核心方法**：论文提出 RhinoVLA，用 token-efficient 的 Qwen3-VL 做骨干，叠加 continuous Action Expert、View Registry、72D state-action slot 和 robot-instance LoRA，把异构机器人数据对齐到统一接口，并配合芯片级编译优化。
3. **关键结果**：作者报告在相近参数规模下，下游表现可比 π0.5，同时在 Huixi R1 上实现 11.69 Hz 端到端推理，达到 10 Hz 实时闭环门槛。

## 💎 价值评估

- **🔬 研究价值**：这篇抓得很准——不是再堆更大模型，而是把 token 结构、动作接口和边端编译一起 co-design，正中“VLA 真落地”最缺的系统层问题。
- **🚀 实践价值**：如果主人后面做机载/板端 VLA，这篇给了很具体的优化抓手：先砍视觉 token，再统一动作槽位，再看混精和并行编码，不必盲目追大模型。
- **📈 扩展潜力**：后续可把这套统一接口往无人机/空地多平台扩展，验证 slot 化动作接口是否真能减少跨载体负迁移。

## 🎯 可落地实验点

**实验设计**：在现有 VLA baseline 上复现“少 token 视觉编码 + 统一 state-action slot”组合，测部署频率和跨平台迁移收益
- 对比基线：π0.5 / LLaVA-VLA / Continuous Reasoning for VLA
- 度量指标：端到端 Hz、任务成功率、跨机器人迁移掉点、显存与延迟占比
- 预期结果：token 压缩与统一接口能在基本不伤性能的前提下显著提升部署速度，并减少异构机器人共训时的接口噪声
