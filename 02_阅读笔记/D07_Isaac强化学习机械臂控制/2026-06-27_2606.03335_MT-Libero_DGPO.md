# 📖 花火格式笔记

## 🎯 题目

**GPU-Parallel Multi-Task Reinforcement Learning with Demonstration Guided Policy Optimization**

## 📝 三句摘要

1. **问题背景**：现有 RL benchmark 多为单任务，难以支撑"同一 loop 训练跨任务 generalist policy"；且稀疏奖励 + 少先验数据场景下，纯 PPO / 纯 BC 都失败——前者探索难、后者天花板低。
2. **核心方法**：双贡献——(a) **MT-Libero**：把 LIBERO 资产 + 任务谓词转成 GPU 并行多任务 RL benchmark（Isaac Lab 上跑，支持并行渲染 + 物理随机化 + state/visual 输入 policy）；(b) **DGPO**：on-policy demonstration guided 方法，把 importance-weighted PPO 与 adaptive BC 在匹配动作上融合，提供"先验偏置可调"的 RL 训练。
3. **关键结果**：MT-Libero 跑通异构任务族 + 视觉 policy；DGPO 在稀疏奖励 + 有限先验场景下稳定超越纯 RL 与现有 demonstration-based 方法，同时保留 on-policy PPO 的稳定性与在线改进能力。

## 💎 价值评估

- **🔬 研究价值**：D07 + D05 交叉点——MT-Libero 补了"多任务 RL benchmark 缺失"；DGPO 补了"少先验 + 稀疏奖励下 demonstration guided RL"的方法空白，对 D07 是双重价值。
- **🚀 实践价值**：对主人具身 RL 极有用——可直接用 MT-Libero 当作 D07 训练 + 评测底座（已成熟），再把 DGPO 接到主人 UAV / 灵巧手 RL 流程上，省去自研 demo+RL 融合代码。
- **📈 扩展潜力**：MT-Libero 与 Isaac Lab 体系深度耦合，未来可与 Newton 可微物理 / Cross-embodiment / VLA pre-train 等联动；DGPO 的 "tunable demonstration preference" 思路可推广到 offline-to-online / human-in-the-loop 等更广场景。

## 🎯 可落地实验点

**实验设计 1（最小落地）**：把 DGPO 应用到主人现有 Isaac Lab 灵巧手 / 机械臂 RL 任务上
- 对比基线：纯 PPO / BC + PPO 两阶段 / GAEIL / 现有 demo-guided 方法
- 度量指标：稀疏奖励场景 SR / 收敛步数 / 最终 SR
- 预期结果：在少 demo 场景下，DGPO SR 提升 15-30%，且收敛步数比纯 PPO 显著缩短
- 落地周期：1-2 周（复用现有 Isaac Lab 环境 + demo 采集）

**实验设计 2（多任务扩展）**：用 MT-Libero + DGPO 训练主人 D02 VLA 模型的 RL 微调阶段
- 对比基线：单任务 PPO 微调 / 多任务 PPO / MT-Libero + DGPO
- 度量指标：跨任务平均 SR / 任务间干扰（catastrophic forgetting）
- 预期结果：DGPO 多任务 RL 微调能保留 VLA 已有先验，又显著提升稀疏奖励任务 SR
