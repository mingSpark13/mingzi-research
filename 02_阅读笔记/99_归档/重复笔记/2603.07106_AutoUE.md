# 📖 花火格式笔记

## 🎯 题目

AutoUE: Automated Generation of 3D Games in Unreal Engine via Multi-Agent Systems

## 📝 三句摘要

1. **问题背景**：传统游戏开发中，3D场景和玩法的创建依赖大量人工，自动化程度低；LLM虽能生成代码，但难以直接操控复杂的UE工程系统。
2. **核心方法**：提出多智能体协作框架：模型检索Agent从858K 3D模型库中找资产，场景生成Agent用UE PCG生成可编辑layout graph，代码Agent生成玩法逻辑，最后playtesting Agent运行测试命令验证结果。
3. **关键结果**：实现了从自然语言描述到可玩游戏关卡的端到端自动化生成，验证了多Agent协作在复杂工程系统中的可行性。

## 💎 价值评估

- **🔬 研究价值**：为路线A提供了更工程化的多Agent参考架构，特别是"检索-生成-验证"三阶段分离的设计模式。
- **🚀 实践价值**：检索Agent思路可迁移到主人低空场景——从资产库检索合适建筑类型/道路规格/植被模型比让LLM从头生成更靠谱；playtesting Agent对应主人的critic验证闭环。
- **📈 扩展潜力**：可扩展为"低空场景检索+PCG生成+飞控验证+轨迹采集"的四Agent流水线。

## 🎯 可落地实验点

**实验设计**：参考AutoUE的四Agent架构，为低空场景生成设计流水线：检索Agent（从低空资产库选建筑/道路/植被模型）、场景生成Agent（UE PCG生成布局）、飞控验证Agent（检查可行航线和安全clearance）、轨迹采集Agent（在Isaac Sim中执行expert policy）。
- 对比基线：单Agent统一生成、多Agent无检索
- 度量指标：Scene Validity Rate、Path Feasibility Rate、Collision Rate、数据生成吞吐量
- 预期结果：四Agent流水线在场景合法性和生成效率上优于简化版本
