# 📖 花火格式笔记

## 🎯 题目

CodeGraphVLP: Code-as-Planner Meets Semantic-Graph State for Non-Markovian Vision-Language-Action Models

## 📝 三句摘要

1. **问题背景**：传统 VLA 往往把最新观测当作动作决策的充分条件，在长程、遮挡、非马尔可夫操作任务里容易遗忘早期关键信息。
2. **核心方法**：论文把持久化 semantic graph、可执行 code planner 和 progress-guided visual-language prompting 结合起来，用图状态做进度检查与子任务分解，再把与子任务相关的对象筛出来给执行器。
3. **关键结果**：在真实世界非马尔可夫长程操作任务上，CodeGraphVLP 相比强 VLA 基线和带历史的变体都提高了任务完成率，同时比 VLM-in-the-loop 规划延迟更低。

## 💎 价值评估

- **🔬 研究价值**：它不是简单给 VLA 加 history，而是把“状态记忆 + 可执行规划 + 目标相关观测裁剪”拆成明确层次，直接命中长程具身系统最常见的记忆与规划耦合痛点。
- **🚀 实践价值**：semantic graph + progress check 很适合接到主人现在关心的 verifier / progress trace / packet 式架构里，能作为 VLA 上层轻量 planner 设计参考。
- **📈 扩展潜力**：后续可以继续接 world model rollout、失败恢复策略、graph update uncertainty 估计，往可验证长程操作系统演进。

## 🎯 可落地实验点

**实验设计**：做一个“progress-trace + object-filtered observation”轻量版，对比 plain VLA 与 history-VLA 在遮挡式多步骤桌面任务上的成功率和延迟。
- 对比基线：plain VLA、history-window VLA、VLM-in-the-loop planner
- 度量指标：task success rate、subtask completion accuracy、planning latency、错误恢复率
- 预期结果：显式 graph state + 子任务相关对象裁剪能在不显著拉高时延的前提下提升长程任务稳定性
