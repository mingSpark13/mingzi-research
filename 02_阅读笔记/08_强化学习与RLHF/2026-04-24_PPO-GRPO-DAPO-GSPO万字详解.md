# 📖 花火格式笔记

## 🎯 题目

万字详解：PPO → GRPO → DAPO → GSPO 的演进逻辑与核心差异

（原标题：同是强化学习，为什么"字节方案"上限更高，而"阿里方案"更适合 MOE？）

---

## 📝 三句摘要

1. **问题背景**：PPO 在 LLM RLHF 训练中需要 4 个模型（Actor/Critic/Reward/Reference），奖励模型训练复杂、价值模型不稳定，导致训练脆弱；GRPO 通过去掉价值模型、用规则奖励替代奖励模型来简化，但引入了熵坍塌、奖励函数依赖、token/序列粒度不一致等新问题。
2. **核心方法**：DAPO（字节）针对长 CoT 场景提出非对称 Clip-Higher + 动态采样 + token 级损失 + 长序列奖惩四项改进；GSPO（阿里/Qwen3）针对 MoE 模型提出将重要性采样修正项从 token 级改为序列级几何平均，从根本上解决粒度不一致问题。
3. **关键结论**：DAPO 通过提升裁剪上限缓解熵坍塌，适合 Dense 模型长思维链训练，上限更高；GSPO 通过序列级重要性采样消除 MoE 专家路由带来的方差爆炸，更适合 MoE 架构。

---

## 💎 价值评估

- **🔬 研究价值**：系统梳理了 RLHF 主流算法的演进脉络，从策略梯度视角统一解释了 PPO/GRPO/DAPO/GSPO 的设计动机与局限，是理解当前 LLM 强化训练的最佳入门综述之一。
- **🚀 实践价值**：对于 VLA/具身智能的 RL 训练有直接参考价值——GRPO 的熵坍塌问题、奖励函数设计困难、token 粒度不一致等问题在机器人 RL 训练中同样存在；DAPO 的动态采样和 GSPO 的序列级修正思路可迁移到机器人策略优化。
- **📈 扩展潜力**：GSPO 的序列级重要性采样思路可能对具身智能中的 trajectory-level RL（整条轨迹作为奖励单元）有启发；DAPO 的 Clip-Higher 机制可用于解决机器人 RL 中低概率动作探索不足的问题。

## 🔍 核心内容详解

### 一、PPO 算法回顾

PPO 在 LLM RLHF 中需要 **4 个模型**：

| 模型 | 作用 | 是否更新 |
|------|------|---------|
| Actor（主模型） | 生成输出，训练目标 | ✅ 更新 |
| Critic（价值模型） | 估计状态价值 V(s) | ✅ 更新 |
| Reward Model | 对输出打分 | ❌ 固定 |
| Reference Model | KL 散度约束基准 | ❌ 固定 |

**优势函数**：A(s,a) = Q(s,a) - V(s)，即当前动作相对所有动作均值的价值优势。

**PPO 的核心问题**：
- 奖励模型训练复杂（需要人工偏好标注）
- 价值模型估计不准确 → 梯度方向错误 → 训练崩溃
- 4 个模型显存开销大

---

### 二、GRPO：去掉价值模型

**核心简化**：
- 去掉 Reward Model → 用规则函数替代（适合数学/代码等有明确正确答案的任务）
- 去掉 Critic → 用组内输出的奖励值标准化计算优势

**优势函数计算**：对同一输入生成 G 个输出，优势 = (该输出奖励 - 组内均值) / 组内标准差

**GRPO 的固有缺陷**：

| 问题 | 原因 | 影响 |
|------|------|------|
| 熵坍塌 | 高概率 token 被持续强化，低概率 token 提升受限 | 输出多样性下降，无法探索新策略 |
| 奖励函数依赖 | 去掉价值模型后完全依赖奖励函数质量 | 奖励函数设计困难，容易被钻空子 |
| 粒度不一致 | 奖励是序列级，动作是 token 级 | 无法从 token 粒度精细调整策略 |
| 重要性采样方差 | 单 token 概率代替整体分布，方差随序列长度累积 | 长序列训练不稳定 |
| 全对/全错组无效 | 组内优势全为 0 | 训练后期大量 step 无效 |

---

### 三、DAPO（字节）：针对长 CoT 的 4 项改进

**应用场景**：带思考过程的长序列输出（长 CoT）

#### 3.1 Clip-Higher（非对称裁剪）

GRPO 使用对称裁剪 [1-ε, 1+ε]，DAPO 改为非对称 [1-ε_low, 1+ε_high]，其中 ε_high > ε_low。

**原理**：旧策略中低概率 token 在对称裁剪下提升空间极小（0.1 × 1.2 = 0.12），导致这些 token 永远无法被充分探索，加剧熵坍塌。提高上限后，低概率 token 有更大的提升空间，增强探索能力。

#### 3.2 动态采样

**问题**：训练后期组内全对/全错的比例越来越高，这些 step 对梯度无贡献，浪费计算资源，且增大训练方差。

**方案**：直接剔除全对/全错的组，用新采样的有效组补充，直到凑满一个 batch。实验证明收敛更快，可以抵消额外采样的开销。

#### 3.3 Token 级损失（重新平衡动作权重）

GRPO 对组内每个序列赋予相同权重（序列级均值），导致长序列中每个 token 的权重随长度增加而稀释。

DAPO 改为对所有 token 赋予相同权重（token 级均值），使长序列中每个 token 的梯度贡献与短序列一致。

**梯度视角**：GRPO 退化为序列级行为克隆（SFT），DAPO 保留了 token 级策略梯度的本质。

#### 3.4 长序列奖惩

对被截断的长序列不直接给负分，而是用动态惩罚函数区分"内容错误"和"长度超限"，避免模型误学"长推理 = 错误"。

---

### 四、GSPO（阿里/Qwen3）：针对 MoE 的序列级重要性采样

**应用场景**：MoE 模型训练

**核心问题**：MoE 中新旧策略的专家路由不同，导致 token 级重要性采样修正项极不稳定（方差爆炸），大量 token 被裁剪，梯度贡献为 0，专家功能混乱。

**GSPO 的解决方案**：将重要性采样修正项从 token 级改为**序列级几何平均**：

```
IS_seq = (∏ IS_token_i)^(1/T)
```

即对序列中所有 token 的重要性采样修正项做几何平均，作为整个序列的修正项。

**优势**：
- 几何平均消除了少数极值 token 带来的方差偏移
- 序列级修正与序列级奖励粒度一致，逻辑自洽
- 避免 MoE 专家路由差异导致的训练不稳定

**代价**：序列内每个 token 的梯度贡献相同，退化为序列级 SFT（但这在 MoE 场景下是可接受的权衡）。

---

### 五、四种算法对比总结

| 维度 | PPO | GRPO | DAPO | GSPO |
|------|-----|------|------|------|
| 模型数量 | 4 | 2 | 2 | 2 |
| 奖励来源 | 奖励模型 | 规则函数 | 规则函数 | 规则函数 |
| 价值估计 | 价值模型 | 组内标准化 | 组内标准化 | 组内标准化 |
| 重要性采样 | token 级 | token 级 | token 级 | 序列级几何平均 |
| 损失粒度 | token 级 | 序列级均值 | token 级均值 | 序列级 |
| KL 约束 | ✅ | ✅ | ❌（长 CoT 场景） | ✅ |
| 裁剪方式 | 对称 | 对称 | 非对称（Clip-Higher） | 对称 |
| 动态采样 | ❌ | ❌ | ✅ | ❌ |
| 适用场景 | 通用 RLHF | 规则任务 | Dense 长 CoT | MoE 模型 |
| 主要优势 | 理论完备 | 简单易用 | 探索能力强 | MoE 稳定 |

---

### 六、On-Policy vs Off-Policy 补充

- **On-policy**：生成数据的模型 = 训练的模型，重要性采样修正项 ≈ 1
- **Off-policy**：使用旧策略数据训练新策略，需要重要性采样修正

实践中 LLM RL 训练通常是 off-policy：一批 rollout 数据分多个 mini-batch 训练，只有第一个 mini-batch 是 on-policy 的。

**关键洞察**：如果完全 on-policy 训练 GRPO，重要性采样修正项 = 1，目标函数退化为 SFT 的交叉熵梯度（每个 token 优势相同），失去 RL 的本质优势。

## 🎯 可落地实验点

**实验设计**：将 DAPO 的 Clip-Higher 机制迁移到机器人 RL 训练

- **背景**：机器人 RL 中低概率动作（如精细操作动作）在标准 PPO/GRPO 训练中探索不足，导致策略收敛到次优解
- **方案**：在 Isaac Lab 的 manipulation 任务中，将 PPO 的对称裁剪改为非对称裁剪（ε_high = 0.4, ε_low = 0.2），观察低概率精细动作的探索频率和最终成功率
- **对比基线**：标准 PPO（对称裁剪 ε=0.2）
- **度量指标**：策略熵、动作分布多样性、任务成功率、收敛速度
- **预期结果**：Clip-Higher 在精细操作任务中成功率提升 5-15%，策略熵更高

**实验设计 2**：GSPO 序列级重要性采样在轨迹级 RL 中的应用

- **背景**：机器人 RL 中奖励通常是轨迹级（episode 结束才给奖励），与 GSPO 的序列级奖励设定天然对齐
- **方案**：在 navigation 任务中，将 GRPO 的 token 级重要性采样改为轨迹级几何平均，对比训练稳定性
- **度量指标**：训练曲线方差、最终 SR、训练崩溃频率
