# 📖 花火格式笔记

## 🎯 题目

Mem-World: Memory-Augmented Action-Conditioned World Models for Persistent Robot Manipulation

## 📝 三句摘要

1. **问题背景**：action-conditioned world model 在机器人操作上已经被证明是低成本合成演示的可行路径,但当操作任务具有长时间跨度、手腕摄像头频繁被遮挡或快速运动时,当前的 observation-only 模型不足以预测未来状态,rollout 很快就漂移或者塌缩,无法支撑数据飞轮需要的 rollout 长度。
2. **核心方法**：Mem-World 在标准 action-conditioned world model 之外加一个显式的 long-horizon memory 模块,持续压缩历史 observation 与 action 轨迹,生成一个 task-aware memory token 与当前的视觉 token 一起参与未来帧预测,把"当前看得见的"与"长期记得住的"在 latent 层解耦。
3. **关键结果**：在多个长时程 manipulation benchmark 上,Mem-World 比纯 observation-based world model 在 rollout fidelity、video FVD 与下游 policy 提升幅度上都有明显改进,并且支持跨 episode 的 memory reset / replay 协议,适合作为数据飞轮的 rollout backbone。

## 💎 价值评估

- **🔬 研究价值**：直接命中 D05 的 §3.2-3.4 —— world model 不是只预测一帧,而是要在长时间尺度上保持 coherent 状态、支撑长时程任务的合成 rollouts。Mem-World 把 memory 从 observation 抽出来做显式 token,这是把 world model 从 "frame predictor" 升级到 "persistent task simulator" 的关键步骤,跟 D05 §3.4 Recovery-Centric Data Flywheel 直接咬合。
- **🚀 实践价值**：主人如果要把世界模型作为空中操作合成数据源,几乎一定要解决"长时间飞行 + 频繁遮挡 + 视角跳变"下的状态保持问题。Mem-World 的 memory 模块设计可以减少 observation 漂移,提升合成轨迹的下游策略可用性,降低 promotion gate 阶段因为 rollout 不连贯而被筛掉的比例。
- **📈 扩展潜力**：Mem-World 的 memory token 设计与 D05 §3.5 Disturbance-Bucketed Promotion 的"disturbance token"几乎是同构的,可以顺着同一思路把 disturbance regime / recovery boundary / task progress 编码进 memory,让 promotion 不再依赖单 rollout 标签,而是直接基于 memory state 做 bucket routing。

## 🎯 可落地实验点

**实验设计**：复现 Mem-World 的 memory 模块,套到主人现有的 manipulation world model 训练链路上,测它能否提升合成数据质量
- 对比基线：observation-only world model / naive frame stacking baseline / Mem-World
- 度量指标：(1) 长时间 rollout 的 FVD / LPIPS drift 曲线,(2) 不同遮挡比例下的 prediction SSIM,(3) 用合成数据 finetune 下游 policy 后真实任务的 SR / 覆盖率
- 预期结果：在 wrist-camera 遮挡比例 >30% 的任务上,Mem-World 在 rollout fidelity 与下游策略 SR 上明显领先,且 memory token 可被复用为 recovery / near-failure 桶的 routing 信号

**与 D05 主线的耦合实验**：
- 把 Mem-World 输出的 memory token 接到 D05 §3.5 disturbance-bucket classifier,测是否可以直接省去一部分人工 disturbance 标注
- 用 Mem-World 做 recovery rollouts,验证 §3.4 描述的"recovery 数据 promotion"是否真的能让 promotion gate 保留更多非平凡样本
