# 📖 花火格式笔记

## 🎯 题目

**Scalable Behavior Cloning with Open Data, Training, and Evaluation**（ABC 全栈开源 BC）

## 📝 三句摘要

1. **问题背景**：行为克隆（BC）已成具身主流，但当前公开数据集与训练栈都太小、太专、不可复现——研究者改个 backbone 或训练配方都很难拿到可信 ablation。
2. **核心方法**：ABC 把"硬件 + 仿真 + 训练 + 数据 + 评测"全栈开源，核心 **ABC-130K**：迄今最大开源遥操作数据集（3,500 小时、130K episodes、195 个任务），配套 400h 模拟遥操数据 + **sim-real co-training 配方**，让仿真评估和真实评估高度相关；并在 DiT vs VLA 上做了一系列架构 ablation。
3. **关键结果**：ABC 全套工具复现并跑通 box folding / 从钱包抽信用卡等灵巧任务，验证了"开放栈 + 大数据 + sim-real 联动"是当前 BC scaling 的可靠范式；仿真代理可作为真实评估的低成本前置过滤。

## 💎 价值评估

- **🔬 研究价值**：D05 直接相关——把"数据飞轮"从单点工具升级为"全栈开源基础设施"；sim-real co-training 配方给出"仿真能做 ablate、真实做最终验证"的双层证据，是 2026 BC scaling 的标志性工程化范式。
- **🚀 实践价值**：对主人具身/VLA 项目极有价值——无需从头搭数据采集 + 训练 + 评测链，直接复用 ABC-130K 数据集 + ABC co-training 工具，把精力集中在 backbone / 动作表征 / 决策频率等真正能发力的层。
- **📈 扩展潜力**：ABC-130K 的 195 任务覆盖面、3,500 小时规模本身就可作为"具身基础数据"长期 baseline；sim-real co-training 后续可扩展到更多 backbone（ACT / DP / π₀ 等）的横向 benchmark；硬件层也可对接主人的 PX4 / Genesis / Isaac 链路。

## 🎯 可落地实验点

**实验设计 1（最小可落地）**：把 ABC co-training 配方移植到主人现有 UAV / 灵巧手数据上
- 对比基线：纯真实 BC / 纯 sim BC / ABC 风格 co-train
- 度量指标：真实硬件 SR、仿真 SR、二者相关性（co-training 关键 KPI）
- 预期结果：co-train 真实 SR 提升 10-20%，且仿真 SR 与真实 SR 相关系数 > 0.7
- 落地周期：1-2 周（已有真实数据 + Isaac Sim 环境的前提下）

**实验设计 2（横向 ablation）**：用 ABC-130K + 主人自己小规模（~50h）真实数据，复现 DiT vs VLA 横向 ablation
- 对比基线：ACT / Diffusion Policy / OpenVLA / π₀ / π₀-FAST 各自 baseline
- 度量指标：单任务 SR / 跨任务平均 SR / 跨 embodiment 迁移 SR
- 预期结果：得到 backbone 横向"scaling 收益曲线"，作为 D02 / D05 后续选型的实证依据

## 📌 待探索问题

1. ABC-130K 全开源意味着其他实验室可"搭便车"——后续是否会出现"专门在 ABC 上微调"的 VLA 子流派？社区是否会以 ABC 为事实标准做新一轮 scaling law？
2. sim-real co-training 配方的 400h 仿真数据来自自家仿真器，对其他仿真平台（Isaac Lab / Genesis / MuJoCo）是否同样有效？co-training 配方对仿真器精度 / 物理一致性有多敏感？
3. ABC 评测里 DiT 和 VLA 的胜负随数据规模如何变化？是否存在"VLA 在小数据上更强、DiT 在大数据上更强"的相变？这对主人后续 backbone 选型至关重要。
4. 130K episodes 中 195 个任务的覆盖度对"长尾操作 / 灵巧度 / 接触丰富"任务的代表性如何？ABC 没有专门测这三类极端场景，可作为后续 ablation 切入点。
