# 📖 花火格式笔记

## 🎯 题目

GenSim2: Scaling Robot Data Generation with Multi-modal and Reasoning LLMs

## 📝 三句摘要

1. **问题背景**：GenSim使用纯代码LLM生成仿真任务，但复杂机器人任务（如多阶段操作、动态交互）需要更强的多模态理解和推理能力，纯代码LLM生成的任务多样性和真实性受限。
2. **核心方法**：利用多模态LLM（图像+代码+文本）和推理LLM（CoT推理）提升任务生成的复杂度和真实性；扩展到更复杂的6-DoF关节机器人任务，支持更真实的仿真数据生成。
3. **关键结果**：相比GenSim，多模态+推理能力显著扩展了可生成任务的复杂度和多样性，数据规模和任务质量均有提升。

## 💎 价值评估

- **🔬 研究价值**：GenSim的升级版，证明多模态和推理能力对复杂任务生成至关重要，与主人"LLM不是直接替代仿真器，而是调度大脑"的判断一致。
- **🚀 实践价值**：主人的低空数据工厂可参考——用多模态LLM理解渲染反馈，用推理LLM规划复杂巡检/穿越任务，使生成的任务更符合真实低空飞行的约束。
- **📈 扩展潜力**：可与主人的VLN/VLA评测结合——用GenSim2生成复杂的低空任务变体（如"有动态障碍的巡检"、"多目标搜索"），构建难度梯度课程。

## 🎯 可落地实验点

**实验设计**：参考GenSim2为主人构建多模态+推理增强的低空任务生成：多模态LLM（GPT-4V）接收UE渲染反馈理解场景状态，推理LLM（CoT）规划多阶段任务（如"先巡检A楼→再穿越施工区→最后搜索B楼"），生成任务代码存入task library。
- 对比基线：纯代码LLM任务生成、无推理的单阶段任务生成
- 度量指标：Task Complexity Score、Multi-stage Success Rate、Diversity Score
- 预期结果：多模态+推理LLM使复杂多阶段任务生成率从~25%提升到~70%以上
