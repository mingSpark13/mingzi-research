# 📖 花火格式笔记

## 🎯 题目

NavRAG: Generating User Demand Instructions for Embodied Navigation through Retrieval-Augmented LLM

## 📝 三句摘要

1. **问题背景**：现有导航指令生成多依赖模板或单一轨迹描述，无法模拟真实用户在不同场景、不同时段、向不同导航agent提出的多样化需求。
2. **核心方法**：自动构建scene description tree，然后利用RAG-LLM根据用户画像和场景上下文生成导航指令；在861个3D场景上标注了200多万条导航指令。
3. **关键结果**：生成指令比模板句更贴近真实用户需求，支持不同user persona（安保/配送/应急/维护）的差异化需求表达。

## 💎 价值评估

- **🔬 研究价值**：为路线A的语言生成提供了RAG增强的范式，scene tree + user persona的设置可让主人生成比"go to building_12"自然得多的低空任务指令。
- **🚀 实践价值**：主人的低空任务指令可以从"模板句"升级为"场景树+用户画像+RAG生成"，例如为security inspector生成"巡检左侧玻璃楼外立面"，为delivery operator生成"沿主路搜索停在路边的白色车辆"。
- **📈 扩展潜力**：可扩展到低空多任务类型的指令生成——巡检/搜索/穿越/绕飞各有不同的话术风格和关注点。

## 🎯 可落地实验点

**实验设计**：参考NavRAG为低空场景构建指令生成系统：scene tree（campus/building/entrance/windows/roof/road/parking lot/tree area）+ user persona（security inspector/delivery operator/emergency responder/maintenance engineer）+ RAG-LLM生成多样化指令。
- 对比基线：规则模板指令、单一轨迹描述
- 度量指标：Instruction-Goal Consistency、Spatial Referring Accuracy、Language Diversity、Hallucinated Landmark Rate
- 预期结果：RAG生成指令在多样性和自然度上显著优于模板句，且hallucination rate低于5%
