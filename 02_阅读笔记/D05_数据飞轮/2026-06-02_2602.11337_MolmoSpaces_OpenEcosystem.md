# 📖 花火格式笔记

## 🎯 题目

**MolmoSpaces: A Large-Scale Open Ecosystem for Robot Navigation and Manipulation**

> 副标题：230K 室内场景 + 130K 物体资产 + 42M 抓取标注 + 8 任务基准 + 3 大 simulator 全兼容，开源通用机器人数据/评测生态

## 📝 三句摘要

1. **问题背景**：现有机器人 benchmark 普遍存在三大瓶颈——**(a) 场景/物体规模小**（大多 < 200 场景、< 5K 物体），**(b) 缺乏物理级 mobile manipulation 支持**（要么"魔法抓取"绕过接触动力学，要么只能做 navigation），**(c) 与真实世界的泛化差距未被严格量化**。SOTA 通用策略（π0/π0.5/Gemini/GR00T）在标准 benchmark 上已经接近饱和，**亟需一个能测出"长尾分布外"泛化能力的大规模开放评测平台**。

2. **核心方法**：Ai2 团队提出 **MolmoSpaces 开源生态**，五大组件形成闭环：
   - **Scenes**：230K 室内场景，整合自 AI2-THOR / ProcTHOR / Holodeck 三套数据集，**5 个子集**（MSTwin 1 / MSCrafted 120 / MSProc 12K / MSProcObja 110K / MSMultiType 110K）
   - **Objects**：130K 高质量 3D 物体（来自 Objaverse + 自制 THOR 资产），含语义+物理元数据
   - **Grasps**：**42M 抓取标注**，rigid + articulated 两套 pipeline（antipodal 采样 + 抖动测试 + 关节驱动测试）
   - **Bench**：8 个基础任务（navigate-to / pick / pick-and-place / pick-and-place-color / pick-and-place-next-to / open / close / open-door）+ LLM 生成的 long-horizon 任务
   - **Simulator-agnostic**：**同一资产可加载到 MuJoCo / Isaac Sim / ManiSkill 三大物理引擎**
   - **多载体支持**：Franka FR3（3 种夹爪）+ Rainbow RB-Y1 双臂移动底盘 + 浮动 CAP/2F-85 夹爪

3. **关键结果**：
   - **Sim-to-Real 相关性极强**：Pick 任务在 752 个 RoboArena 真机试验中，**Pearson R = 0.96，Spearman ρ = 0.98**（论文说法），即"仿真中跑得好的策略在真机也跑得好"
   - **π0.5 在 Pick 上 84% vs π0 62% vs π0-FAST 55% vs CAP-EC1 35%** —— 验证了 VLA 模型的代际进步
   - **新发现敏感性**（distributional evaluation）：**(a)** prompt 用词用 DROID 高频动词（"pick" 25,985 次）成功率接近 π0.5，**用低频词（"pick→eat" 0 次）掉 14%** —— 揭示语言条件化是泛化瓶颈；**(b)** wrist camera 遮挡 → 成功率掉到 2%，third-person 遮挡 → 掉到 20%，**策略强依赖腕部视觉**；**(c)** 初始 joint position 偏移 → 性能下降；**(d)** π0.5 偏顶抓，CAP 偏侧抓，对应不同物体表现差异

## 💎 价值评估

### 🔬 研究价值
- **首个 simulator-agnostic 的大规模 mobile manipulation 基准**：之前 BeHAVE-1K、ManiSkill-HAB 等都是单一 simulator，MolmoSpaces 是第一个把同一资产同时跑通 MuJoCo / Isaac / ManiSkill 的生态
- **首个 sim-to-real 相关性如此高（R=0.96）的通用 VLA 评测**：之前 SIMPLER、PolaRiS 都是任务级小规模验证，MolmoSpaces 是全场景+全策略级别的强证据
- **首次系统化"distribution shift 诊断"**：prompt phrasing / 初始 joint / 相机遮挡 三类扰动，给出"策略为什么不行"的可量化归因
- **42M 抓取标注是当下最大规模**（比 GraspGen 还多一个量级），且第一个同时支持 rigid + articulated 物体的公开抓取集

### 🚀 实践价值
- **完整开源 + 工具链齐备**（data / code / blog 三件套），社区可直接复现或扩展
- **6 个真实机器人载体 + 3 个 simulator** = 实验室选型时不需要重新造轮子
- **TeleDex iPhone app** = 用 iOS ARKit 做真机数据采集，门槛低
- **LLM-based long-horizon 任务生成** = 可以低成本扩展到"开冰箱→取菜→装盘"这类组合任务

### 📈 扩展潜力
- 论文明说 future work 包括 **data generation at scale** + **reinforcement learning at scale** —— MolmoSpaces 本身定位是"基础设施"，后续很可能出 RL benchmark 或数据飞轮
- 230K 场景的 procgen 方法可迁移到 **3DGS / 数字孪生**（与主人的 D03 空地迁移/D05 数据飞轮直接相关）
- 多 simulator 兼容 → 未来如果出现新物理引擎（Genesis、Newton），MolmoSpaces 有可能扩展

## 🎯 可落地实验点

### 实验一：用 MolmoSpaces 评测主人自己的 VLA 策略
- **目标**：在 230K 场景/130K 物体上 zero-shot 评估 D02 VLA 项目的实际泛化能力
- **方案**：
  1. 选 1-2 个 sub-task（如 pick / pick-and-place）跑 200 个 episode
  2. 测三个维度：aggregate success rate + sim-to-real 相关性 + distribution shift 鲁棒性
  3. 对比 base VLA vs fine-tuned on AirSpark 数据的效果
- **基线**：参考论文 figure 10 的 π0.5 84% / π0 62% / CAP-EC1 35%
- **度量指标**：success rate / wrist 遮挡鲁棒性 / 初始 joint 偏移鲁棒性
- **预期结果**：在 AirSpark-fine-tuned 模型上 0-15% 提升，关键差异会出现在"非 DROID prompt 措辞"和"非默认 joint 起点"上

### 实验二：把 42M 抓取用到 AirSpark 灵巧手训练
- **目标**：用 MolmoSpaces-Grasp 的 labeled grasps 作为 imitation learning 的 warm-start
- **方案**：
  1. 选 D08 灵巧操作相关的 100-1000 个物体子集
  2. 把 42M 抓取中 rigid 部分导入到灵巧手训练 pipeline
  3. 训练 grasp prior 模型，初始化策略网络
- **度量指标**：收敛步数 / 最终成功率 / 跨物体泛化
- **预期**：相比从零训练，收敛时间下降 30-50%，泛化能力显著提升

### 实验三：Sim-to-Real 相关性验证（主人 AirSpark 真机版）
- **目标**：复现"仿真排名 ≈ 真机排名"的强相关性
- **方案**：
  1. 在 AirSpark UE5 中选 50-100 个 pick 任务
  2. 同一策略（π0.5 / D02 VLA / CAP 类）仿真 + 真机分别跑
  3. 算 Pearson R 和 Spearman ρ
- **预期**：如果主人 D03 项目做得好，R ≥ 0.85 是合理目标（论文做到 0.96 是天花板）

### 实验四：Prompt Phrasing 敏感性 → 反推到 D05 数据飞轮
- **目标**：验证"训练数据用词分布"对 VLA 性能的影响
- **方案**：
  1. 选 π0.5 在 DROID prompt 上的高频词（pick=25,985 / put 等）
  2. 在主人 D05 数据集里统计同类 prompt 分布
  3. 如果低频，重写 1% 训练数据 prompt 模板
- **预期**：prompt 重写后 zero-shot 任务上 +5-10% 提升

## 🧠 花火补充：与主人研究方向的连接

> 主人的研究方向覆盖 D01（世界模型）/D02（VLA）/D03（空地迁移）/D04（跨载体泛化）/D05（数据飞轮）/D06（空中VLN）/D07（Isaac RL）/D08（灵巧操作）。

**关键启示**：

1. **D02 VLA 警惕 prompt 措辞**：主人的 VLA 训练数据如果用自然语言 prompt，要明确统计 prompt 词频分布；DROID 高频词（pick/put）的成功率高 14%，这意味着**训练语料的"语言先验"会大幅影响 zero-shot 性能**
2. **D05 数据飞轮新基线**：MolmoSpaces 230K+130K+42M 是个"现状天花板"，主人 AirSpark 要达到这个规模需要：UE5 资产扩到 ~10K / 抓取标注 ~1M / 多 simulator 兼容至少 2 个
3. **D04 跨载体借鉴**：MuJoCo + Isaac + ManiSkill 三兼容的实现细节必看（特别是 asset format 转换 pipeline），主人在 D03/D04 跨载体项目上可参考
4. **D03 空地迁移直接复用**：R=0.96 的 sim-to-real 相关性方案是教科书级范例，可直接套到主人 D03 真机验证
5. **D08 灵巧手 warm-start**：42M 抓取标注是现成的"灵巧手 prior"，可以快速做 imitation learning 初始化
6. **D07 Isaac RL 路线**：论文 future work 提到 "RL at scale"，主人在 Isaac Sim 上做 RL 时可以直接基于 MolmoSpaces 的环境
7. **河套 Open Day 直接可用**：3 段 backup 视频中"多场景多物体泛化演示"用 MolmoSpaces 风格的 assets 最能展示主人 AirSpark 的扩展性

**主人节奏建议**：本论文是 D02 + D05 + D04 三方向交叉的"数据基座"参考，**优先级高于一般 D05 数据论文**（因为还提供了评测协议和 sim-to-real 验证标准）。
