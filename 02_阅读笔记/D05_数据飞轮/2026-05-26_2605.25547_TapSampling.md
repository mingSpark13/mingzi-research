# 📖 花火格式笔记

## 🎯 题目

TapSampling: Task Progress Verifiers Enable Inference-Time Scaling for Robot Policies

## 📝 三句摘要

1. **问题背景**：很多机器人策略在推理时只有单次动作输出，缺乏像大模型 test-time scaling 那样通过多采样 + 重排序来换性能的机制。
2. **核心方法**：TapSampling 引入 task progress verifier，对同一步的多个候选动作 rollout/采样结果按“任务推进程度”打分重排，在**不重训原策略**前提下提升执行质量。
3. **关键结果**：这条路线说明机器人策略也能吃到 inference-time scaling 红利，重点不再只是更大模型，而是更好的推理时选择机制。

## 💎 价值评估

⭐⭐⭐⭐ | 对 D05 很有价值。它把“多采样 + verifier 重排”引入机器人策略推理阶段，适合接到已有 policy / VLA / world model 后做低成本增强，也适合和数据飞轮里的自动评测器结合。

## 🎯 可落地实验点

- 给现有操作策略加一个轻量 **task progress verifier**，比较单样本 vs 多样本重排的成功率差异
- 在空中操作/导航里尝试把 verifier 设计成“离目标更近 / 更安全 / 更少碰撞风险”的综合分数
- 与世界模型 rollout 结合，做“想象中重排 + 真实执行”的 test-time scaling 验证
