# 📖 花火格式笔记

## 🎯 题目

RoboGen: Towards Unleashing Infinite Data for Automated Robot Learning via Generative Simulation

## 📝 三句摘要

1. **问题背景**：机器人仿真数据生成需要大量人工设计任务、资产、场景配置和训练监督信号，成本极高；现有仿真数据生成主要关注场景级多样性，忽视任务级多样性，制约了技能学习的泛化能力。
2. **核心方法**：提出RoboGen生成式机器人agent，自动生成所有仿真学习阶段所需信息——从高层任务/技能提议，到任务相关场景描述、资产选择/生成、策略学习选择和训练监督信号，形成propose-generate-learn循环。
3. **关键结果**：实现了端到端的生成式仿真流水线，覆盖任务提议→场景生成→策略训练→技能学习全过程。

## 💎 价值评估

- **🔬 研究价值**：是GenSim的扩展和深化，提出"生成式仿真"（generative simulation）概念，propose-generate-learn循环是主人"主动数据飞轮"的雏形。
- **🚀 实践价值**：主人的低空数据工厂可借鉴其流水线思想——LLM生成任务规格（propose）→ PCG生成场景（generate）→ 轨迹采集和策略训练（learn）→ 失败样本回流（反馈到propose）。
- **📈 扩展潜力**：可与主人的critic闭环结合——learn阶段发现失败样本 → 自动生成更难变体 → 扩充训练集，实现真正的主动数据生成。

## 🎯 可落地实验点

**实验设计**：参考RoboGen的propose-generate-learn循环为主人构建低空数据飞轮：Propose（LLM生成SceneSpec+TaskSpec）→ Generate（UE PCG执行+Critic验证）→ Learn（Isaac Sim采集+策略训练）→ Reflect（失败样本→更难变体生成）。
- 对比基线：固定任务集+固定场景、被动数据收集（无主动生成）
- 度量指标：Task Coverage Rate、Active Data Utility（OOD-SR提升）、Sim-to-Real Gap
- 预期结果：主动数据飞轮使OOD场景成功率从基线的~30%提升到~60%以上
