# 📖 花火格式笔记

## 🎯 题目

**FlyMirage** — 基于生成式世界模型的、面向空中 VLN 的全自动数据生成管线

## 📝 三句摘要

1. **问题背景**：空中 VLN 数据集难以兼顾规模、多样性与真实感：真实数据采集成本高、仿真数据视觉质量低；缺乏"主动数据工厂"性质的合成管线。
2. **核心方法**：作者提出 FlyMirage 自动化管线——**LLM 作为环境设计师**生成场景规格，**生成式世界模型** 实例化高保真 3DGS 场景，**自动场景探索+语义获取**，最后**动力学可行轨迹规划器**生成 UAV 可执行轨迹。生成 50K 轨迹 / 500 场景量级。
3. **关键结果**：该管线输出规模化、多样化、照片级真实的空中 VLN 训练数据，可作为现有真实数据 + 视觉受限仿真之外的第三条数据路径。

## 💎 价值评估

- **🔬 研究价值**：是当前最像"UAV embodied data factory"外部参照。完整链路 "LLM 场景设计 → 世界模型生成场景 → 自动探索+语义 → 动力学可行轨迹规划" 是范式级贡献。
- **🚀 实践价值**：**对 AirSpark 直接相关**——我们可以借鉴其"LLM 环境设计器 + 3DGS 场景生成 + 动力学可行轨迹规划" 三件套，主动生成园区/低空导航数据。
- **📈 扩展潜力**：3DGS 表示可与 VLA 训练管线深度结合，自动探索的语义标签可直接喂给 VLM 训练，可行性规划器输出可直接用于 VLA action 监督。

## 🎯 可落地实验点

**实验设计**：复现 FlyMirage 的"LLM 场景设计 + 3DGS 渲染 + 动力学可行轨迹"三件套，做一个 AirSpark 园区低空场景的主动数据生成原型。
- **场景**：在 AirSpark 仿真器基础上，引入 LLM 写场景规格（建筑物布局/高度/材质/动态干扰物），用 3DGS 渲染为高保真数据
- **基线**：纯规则化场景生成（无 LLM）/ 纯仿真渲染（无 3DGS）/ 随机轨迹
- **度量**：场景多样性（FID+类别分布）、轨迹动力学可行率、VLA 在生成数据上训练后在真实数据上的 zero-shot 增益
- **预期**：完整管线生成数据训练出的 VLA 比规则化数据训练出的 VLA zero-shot 真实数据测试高 ≥10%
