# 📖 花火格式笔记

## 🎯 题目

Latent Bridge: Feature Delta Prediction for Efficient Dual-System Vision-Language-Action Model Inference

## 📝 三句摘要

1. **问题背景**：双系统 VLA 虽然效果强，但每个控制步都跑一次大 VLM 太贵，时序上还存在大量冗余特征计算。
2. **核心方法**：论文提出 Latent Bridge 预测相邻时刻的 VLM feature delta，让 action head 在预测特征上运行，只周期性调用昂贵 backbone，并用 task-agnostic DAgger 管线训练这座“时序桥”。
3. **关键结果**：在 GR00T-N1.6 与 π0.5 两类架构上都保留 95-100% 性能，同时减少 50-75% VLM 调用，带来 1.65-1.73x episode 级加速。

## 💎 价值评估

- **🔬 研究价值**：它把 VLA 的瓶颈从“模型设计”拉到“推理缓存与时序冗余利用”，属于非常实用的系统层创新。
- **🚀 实践价值**：对主人这类想把大模型策略压到真实机器人控制频率上的场景很值钱，尤其适合做 low-latency policy serving。
- **📈 扩展潜力**：可以继续外推到 UAV-VLA、空中视觉语言导航、甚至带记忆的 world model 控制器上，看看 delta prediction 是否同样成立。

## 🎯 可落地实验点

**实验设计**：给现有 VLA 或 VLM-policy 增加 feature delta bridge，验证低频 backbone 调用的收益
- 对比基线：逐帧全量推理、固定帧跳跃、KV-cache 复用、latent bridge
- 度量指标：控制频率、任务成功率、累计延迟、GPU 占用、长时任务漂移率
- 预期结果：Latent Bridge 在保性能的前提下显著降低推理成本，比简单 frame skipping 更稳
