# 📖 花火格式笔记

## 🎯 题目

Pre-VLA: Preemptive Runtime Verification for Reliable Vision-Language-Action and World-Model Rollouts

## 📝 三句摘要

1. **问题背景**：VLA 与生成式世界模型在长程具身任务里很强，但动作生成不确定性高，错误动作会直接导致物理失败，错误 rollout 还会放大后续规划偏差。
2. **核心方法**：论文提出 Pre-VLA，在动作执行前用轻量 runtime verifier 对候选动作做预检和拦截，把 VLA/世界模型 rollout 里潜在失效先筛掉，再交给执行器或规划器。
3. **关键结果**：这条路线的价值不在于再训一个更大的策略，而在于给现有 VLA+WM 系统加一层低延迟安全护栏，适合长程任务和真实部署前的在线风险抑制。

## 💎 价值评估

- **🔬 研究价值**：它把“世界模型负责想象、策略负责生成”之间的安全断层单独拎出来，属于很实用的系统层创新，和主人关心的 world model + planning 结合点很强。
- **🚀 实践价值**：如果主人后面做无人机/机械臂长程决策，这种执行前验证器能当通用外挂，不必重训主模型就能降低危险动作落地概率。
- **📈 扩展潜力**：可以继续往 reachability-aware ranking、constraint checking、语言约束护栏扩展，形成“想象-验证-执行”三段式闭环。

## 🎯 可落地实验点

**实验设计**：给现有 VLA 或世界模型规划管线增加 pre-execution verifier，对每步候选动作做可行性与风险打分，再与原策略直接执行做对比。
- 对比基线：原始 VLA 直接执行、仅 world model rollout 过滤、加入 verifier 的混合执行
- 度量指标：任务成功率、危险动作拦截率、平均额外延迟、长程 rollout 崩溃率
- 预期结果：在复杂接触或长程场景里，verifier 版本应显著降低 catastrophic failure，同时额外时延保持在在线控制可接受范围
