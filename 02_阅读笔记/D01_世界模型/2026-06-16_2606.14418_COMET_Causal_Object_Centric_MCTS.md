# 📖 花火格式笔记

## 🎯 题目

Causal Object-Centric Models for Planning with Monte Carlo Tree Search

## 📝 三句摘要

1. **问题背景**：MuZero 风格 latent planning 在机器人任务里常把整个场景压成单块表示，导致规划时难以聚焦真正相关的物体，也不利于动作—对象因果关系建模。
2. **核心方法**：论文提出 COMET，用冻结的 object-centric encoder + transformer world model 在 slot latent space 做 MCTS，并通过 action-slot fusion 与 object-causal attention 显式把动作绑定到任务相关对象。
3. **关键结果**：在 Object-Centric Visual RL benchmark、ManiSkill、Robosuite 和 VizDoom 的 8 个任务上，COMET 在训练早期平均归一化得分优于 object-centric 与 monolithic baseline，说明“对象级因果归因 + latent tree search”对高效规划确实有增益。

## 💎 价值评估

- **🔬 研究价值**：这篇很适合补 D01 世界模型线里“latent planning 不只是会想象，还要知道动作到底作用到谁”这一层，给对象中心世界模型接规划器提供了更清晰的接口设计。
- **🚀 实践价值**：如果明子后面做带目标对象/障碍物交互的世界模型规划，这种 action-slot fusion 可直接拿来做 object-aware rollout 或 task-relevant state packet，避免 planner 把所有 latent token 一锅炖。
- **📈 扩展潜力**：后续可与 D06/D04 的 scene graph、语义导航、跨载体执行接口结合，往“对象级语义-动作绑定”的 embodied planner 方向推。

## 🎯 可落地实验点

**实验设计**：在现有世界模型规划任务里，把 monolithic latent dynamics 替换为 object-centric slots，并测试 MCTS / beam search 在对象干预任务上的收益。
- 对比基线：普通 monolithic latent world model、object-centric world model 无 causal attention、COMET 式 action-slot fusion + MCTS
- 度量指标：训练前期 sample efficiency、规划成功率、关键对象注意力集中度、错误动作归因率
- 预期结果：对象级建模会显著提升涉及交互对象选择与干预顺序任务中的早期学习效率与规划可解释性
