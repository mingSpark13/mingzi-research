# 📖 花火格式笔记

## 🎯 题目

Looped World Models

## 📝 三句摘要

1. **问题背景**：世界模型要在长时序仿真上"算得深"才能保持物理一致性，但更深意味着更多参数、更高部署成本，错误也会沿深度累积放大；这是「模型规模 / 训练数据」之外一直没有被充分挖掘的"深度"维度。
2. **核心方法**：LoopWM 是首个针对世界建模的 looped 架构，用一个参数共享的 transformer block 对环境 latent 状态做迭代式精化（iterative latent depth），并能根据每步预测复杂度自动调节迭代次数，实现自适应计算。
3. **关键结果**：在参数效率上比常规架构提升最多 100×，提出"迭代 latent 深度"作为世界模拟的第三正交 scaling 轴（区别于模型规模 / 数据规模），并展示了对应训练/推理策略。

## 💎 价值评估

- **🔬 研究价值**：D01 现在主流 scaling 路径是把模型/数据往上堆，LoopWM 直接开辟"latent depth"这条新轴，理论上和 D01「双动力学 WM」里"按需细化"的想法天然契合——可按场景复杂度动态决定仿真粒度。
- **🚀 实践价值**：参数共享 + 自适应计算特别适合无人机/边缘部署：在低算力板载平台上仍能跑得起长程 world model rollout，对 D06 空中 VLN 的端侧预演是直接收益。
- **📈 扩展潜力**：loop 块可以接 diffusion / flow matching 形式做"迭代去噪 = 隐式 rollout"，与 D02 的流匹配动作生成、动作分块等结构化输出方法形成互鉴。

## 🎯 可落地实验点

**实验设计**：把现有世界模型 rollout 替换为 looped block，测试参数效率与长程一致性。
- 对比基线：固定深度 transformer WM、depth-wise scaling、LoopWM（固定迭代数）、LoopWM（自适应迭代数）
- 度量指标：参数量 / FLOPs、未来 1s/2s 视频 FID、long-horizon reward prediction MSE、推理延迟-精度 Pareto
- 预期结果：在 10× 参数预算下达到与基线相当或更好的一致性，自适应迭代在简单场景显著降低延迟
