# 📖 花火格式笔记

## 🎯 题目

HANDOFF: Humanoid Agentic Task-Space Whole-Body Control via Distilled Complementary Teachers

## 📝 三句摘要

1. **问题背景**：现有人形全身控制往往要求 planner 输出密集关节/运动学参考流，高层任务规划器很难直接生成，导致“能规划但喂不动控制器”。
2. **核心方法**：论文提出 HANDOFF，把控制接口压缩成面向 planner 的 10 维 task-space command，并用多教师蒸馏 + context-conditioned gating，把运动跟踪、行走和跌倒恢复三类 specialist 融合成一个可部署 student controller。
3. **关键结果**：在 Unitree G1 上，HANDOFF 同时保持接近 SOTA 的速度跟踪与大操作工作空间，还能被 VLM 驱动的 agentic planner 直接调用完成多种自然语言任务，全程无需任务专属数据采集或再微调。

## 💎 价值评估

- **🔬 研究价值**：它切中的不是“控制器强一点”而是“planner 和 controller 之间该怎么接口”，这对主人关心的具身 agent 系统很关键。
- **🚀 实践价值**：10 维 planner-facing command 很像一种可工程化的中间动作语义层，适合借鉴到多平台机器人、甚至 UAV+机械臂的统一控制接口设计。
- **📈 扩展潜力**：后续可把这类 task-space interface 与 VLA/world model 输出对齐，形成更稳定的高层推理→低层执行桥梁。

## 🎯 可落地实验点

**实验设计**：为现有机器人系统设计一个低维 task-space command 接口，对比“直接输出底层动作”与“输出中间目标命令”两种 planner-controller 解耦方式
- 对比基线：端到端动作回归策略、传统全身运动流参考控制器
- 度量指标：任务成功率、planner 可控性、接口迁移成本、部署时的调试复杂度
- 预期结果：中间命令接口更利于多 planner 复用，也更适合接语言模型或世界模型输出
