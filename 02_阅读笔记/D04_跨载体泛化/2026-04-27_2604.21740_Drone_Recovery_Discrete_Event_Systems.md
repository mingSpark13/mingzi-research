# 📖 花火格式笔记

## 🎯 题目

A Case Study in Recovery of Drones using Discrete-Event Systems

## 📝 三句摘要

1. **问题背景**：群体无人机任务里，单机掉队、受攻击或状态异常后如何安全恢复并重新编队，是传统连续控制框架不太擅长显式保证的环节。
2. **核心方法**：论文用 discrete-event systems 与 supervisory control theory 设计高层恢复监督器，再与低层连续控制器组合，完成 lost drone 的区域重入与 regrouping 管理。
3. **关键结果**：作者在 pybullet-drones 十机仿真中验证了四类恢复场景，展示了从故障恢复到重新回归编队流程的可行性与规范性。

## 💎 价值评估

- **🔬 研究价值**：这篇不靠大模型噱头，而是把形式化高层监督重新带回无人机恢复问题，适合给主人补 D04 方向里“安全可验证调度”的方法栈。
- **🚀 实践价值**：对多无人机巡检、配送或灾害搜救都很有用，尤其适合作为 failsafe / recovery manager 的高层模板。
- **📈 扩展潜力**：后续可把 supervisor 与学习型策略、攻击检测器或语义任务规划器拼起来，形成可验证的混合自治框架。

## 🎯 可落地实验点

**实验设计**：在 pybullet-drones 或 ROS2 多机仿真里实现“掉队→重入→重组”恢复监督器，对比纯行为树/状态机恢复策略。
- 对比基线：手工状态机、行为树恢复器、无高层监督的直接返航策略
- 度量指标：恢复成功率、重组时间、冲突/碰撞次数、异常后任务继续率
- 预期结果：DES supervisor 在多约束恢复流程中更稳定，可显著减少恢复期间的行为冲突
