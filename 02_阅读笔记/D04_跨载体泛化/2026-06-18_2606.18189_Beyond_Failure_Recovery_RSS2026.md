# 📖 花火格式笔记

## 🎯 题目

Beyond Failure Recovery: An Engagement-Aware Human-in-the-loop Framework for Robotic Systems

## 📝 三句摘要

1. **问题背景**：传统 human-in-the-loop 把人当"故障兜底工具"，只在机器人失败时介入。问题是这让用户（尤其行动不便者）在长任务中沦为被动观察者，参与度低，违背 care-giving 场景的人本初衷。
2. **核心方法**：E-MPC (Engagement-aware MPC) = 用户参与度感知的 MPC 框架。建模 user interaction dynamics（频率 + 类型），在满足任务成功约束的同时显式优化 engagement，且控制用户工作负载上限。proactive 平衡"自主 vs 交互"，而非被动等待失败。
3. **关键结果**：仿真中跨多种 user persona 有效；真实用户研究（emulated mobility limitations 喂食场景）证明 E-MPC 在保持任务成功率的同时显著提升 user experience。

## 💎 价值评估

- **🔬 研究价值**：把人机交互从"失败兜底"提升到"engagement-aware control"范式；E-MPC 框架把心理学参与度模型嵌入 MPC 优化，开创了"用户状态作为规划目标"的新设计空间。RSS 2026 顶会，质量有保障。
- **🚀 实践价值**：与主人研究方向不直接相关（D04/D06），但对辅助机器人、远程陪伴机器人方向有参考价值；如果主人未来扩展到 social robotics / 助老机器人可借鉴。
- **📈 扩展潜力**：engagement model 可与 LLM/VLM 结合做更自然的人机意图理解；可扩展到 multi-user / shared autonomy 场景。

## 🎯 可落地实验点

**实验设计**：（优先级低，仅作为长期参考）若主人未来做空中远程陪伴/inspect-by-demo 场景，可借鉴 engagement-aware 设计：
- 对比基线：(a) Standard MPC，(b) Failure-recovery-only HITL
- 度量指标：user engagement score（问卷 + 行为）/ task success rate / workload
- 关键挑战：engagement model 需要领域定制，主人研究方向不是社会机器人
