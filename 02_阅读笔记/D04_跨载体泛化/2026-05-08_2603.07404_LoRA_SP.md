# 📖 花火格式笔记

## 🎯 题目

Adaptive Capacity Allocation for Vision Language Action Fine-tuning

## 📝 三句摘要

1. **问题背景**：预训练 VLA 落到新环境、新任务或新载体时常要做参数高效微调，但直接照搬 LLM 里“小 rank LoRA 就够了”的经验，在机器人迁移上往往失效。
2. **核心方法**：论文提出 LoRA-SP，用共享向量库加小型 router 做输入相关、层相关的自适应容量分配，通过能量阈值自动选择有效秩，替代固定 rank 的 LoRA 更新。
3. **关键结果**：在 unseen AgileX PiPER 机械臂上的 4 个真实操作任务、两种 VLA backbone（π0 与 SmolVLA）上，LoRA-SP 以更少可训练参数达到或超过 full fine-tuning，多任务成功率最高比标准 LoRA 提升 31.6%。

## 💎 价值评估

- **🔬 研究价值**：这篇很重要地指出 VLA 迁移的 intrinsic rank 明显高于 LLM 微调，等于把“适配容量”从调参细节升级成了机器人迁移研究变量本身。
- **🚀 实践价值**：主人以后做多平台 VLA、无人机-机械臂共享策略、或 unseen embodiment 适配时，不能再迷信低 rank LoRA，一开始就该把容量分配、层选择、任务干扰抑制纳入设计。
- **📈 扩展潜力**：LoRA-SP 不只适合机械臂，理论上也可外推到空地迁移、跨传感器适配、甚至跨模态 action head，对 D04 主线很补。

## 🎯 可落地实验点

**实验设计**：在主人未来的跨平台 VLA 适配任务里，对比固定-rank LoRA、full fine-tuning 与 LoRA-SP 风格的自适应容量微调。
- 对比基线：r=8/16/32/128 标准 LoRA；full fine-tuning；按层手工开关 adapter
- 度量指标：新载体 success rate、多任务负迁移、参数量、训练稳定性、少样本适配效率
- 预期结果：自适应容量分配会比固定小 rank 更稳，尤其在多任务与跨载体场景中优势更大
