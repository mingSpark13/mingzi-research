# 📖 花火格式笔记

## 🎯 题目

HumanoidUMI: Bridging Robot-Free Demonstrations and Humanoid Whole-Body Manipulation

## 📝 三句摘要

1. **问题背景**：人形机器人 whole-body 技能学习需要感知-姿态-移动-操作协同的高质量示教数据，但传统 robot-in-the-loop 遥操作受硬件访问/操作员技能/安全成本限制，大规模采集代价极高；现有 UMI / HuMI / HoMMI 类便携式方案要么没覆盖 whole-body，要么硬件（Vive）+ 重定向与控制器强耦合、难复刻。
2. **核心方法**：提出 **portable robot-free 数据采集 + 三层 hierarchical policy** 框架——硬件用 PICO 4 VR（头显+手柄+三 tracker）捕获 5 keypoint（pelvis/双 gripper TCP/双脚）+ 腕部鱼眼相机 + gripper 宽度，构成稀疏空间表示；在线 Spatial Keypoint Retargeting（SKR）实时把人体 keypoint 映射成 robot-native root pose 与 joint positions；最后由 learned whole-body controller 在 Unitree G1 上执行。
3. **关键结果**：在 Unitree G1 真实平台上 5 个任务（单臂操作 / 双手协调 / 动态投球 / whole-body 弯腰 / locomotion-manipulation）验证，robot-free 采集即可迁移为可部署 whole-body 技能，相对 teleoperation baseline 采集更高效；high-level 扩散策略 + 显式 retargeting + 底层 whole-body controller 的三层解耦比 HuMI 端到端 IK 紧耦合更易解释与跨任务复用。

## 💎 价值评估

- **🔬 研究价值**：把"robot-free 示教"从桌面 UMI 推到 humanoid whole-body，关键贡献是 5-keypoint 稀疏空间表示 + 显式 SKR 重定向，把"人体-机器人形态 gap"解耦成几何层（keypoint）+ 运动学层（SKR）+ 控制层（whole-body controller）三段契约，对 D04「跨载体形态 gap 用接口契约拆解」是直接的方法层支撑。
- **🚀 实践价值**：PICO 4 + 自研 gripper + fisheye 的硬件成本远低于真机遥操 + 全身动捕系统，给"低成本大规模人形数据采集"提供现实可行路径；稀疏 5-keypoint 比 dense 全身 pose 训练便宜得多，更易扩展到多人多任务。
- **📈 扩展潜力**：① 把 sparse keypoint 推广到 UAV / 灵巧手 / 双足（不同 embodiment 都可重定向到 5 个 anchor point）② 与 D05 数据飞轮对接：robot-free 采集 → 关键点 SKR 自动生成 pseudo-target → 扩数据池 ③ SKR 模块本身就是「跨 embodiment 功能空间接口」的活样本，可直接借鉴到主人 WAM VLA 平台的中层 packet 设计。

## 🎯 可落地实验点

**实验设计**：在主人具身平台复现"robot-free 示教 → 跨形态重定向 → 部署"链路，重点验证 5-keypoint SKR 接口的形态无关性
- **数据准备**：用 PICO / Quest 等消费级 VR + 自研 gripper 采集人类演示；同步录制腕部鱼眼视频与 gripper 状态；针对 3 种 embodiment（机械臂 / 人形 / 灵巧手）各采集 50-100 episode
- **对比基线**：① end-to-end 形态特化策略（每 embodiment 独立训练）② HuMI 风格紧耦合 IK + controller ③ 本文三层解耦（5-keypoint + SKR + 控制器）
- **度量指标**：① robot-free 采集效率（episodes/hour、operator skill 门槛）② 跨形态部署成功率（不重训练直接迁移）③ 重定向的物理可行性（关节限位/平衡违规率）④ 数据复用率（同一 human 演示跨 embodiment 重用比例）
- **预期结果**：三层解耦在「跨 embodiment 数据复用率」上显著高于端到端特化；SKR 接口违反率 <5%；robot-free 采集速度相对 teleop 提升 3-5×

## 📌 待探索问题

- 5-keypoint（pelvis + 双 gripper + 双脚）是否足够支撑"动态全身"任务（如摔倒恢复 / 跑跳）？如果不够，应加哪些 anchor（膝盖 / 肘部 / 头部朝向）？
- SKR 的"物理可行性"是否需要在线校验（关节限位 / ZMP 平衡 / 碰撞）？如果校验失败应该回退到哪一层的修正？
- PICO 4 + 自研 gripper + fisheye 的硬件 setup 在跨实验室复刻时，标定偏差会如何影响 SKR 的精度？是否需要"形态无关的轻量标定协议"？
- 高层扩散策略学到的是"5-keypoint + gripper"语义，与下游 SKR 之间是否存在分布外 OOD 风险（如未见任务结构）？如何给 SKR 配 fallback（保守 human-likely motion）？
- 本方法对"形态 gap"的拆解（几何 + 运动学 + 控制三层契约）能否推广到 UAV 跨机型 / 跨旋翼尺寸？
