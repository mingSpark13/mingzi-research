# 📖 花火格式笔记

## 🎯 题目

From Prompt to Physical Actuation: Holistic Threat Modeling of LLM-Enabled Robotic Systems

## 📝 三句摘要

1. **问题背景**：LLM 一旦进入机器人感知—规划—执行闭环，错误提示、对抗输入或 tool use 失控就可能直接穿透到物理动作层，传统软件威胁模型不够用了。
2. **核心方法**：论文把 LLM-enabled robot 抽象成分层数据流图，用 STRIDE 风格分析贯穿用户输入、视觉理解、规划生成、工具调用到 actuator dispatch 的跨边界风险链路。
3. **关键结果**：结论不是“某个模型不安全”，而是指出缺少独立语义校验、跨模态翻译无护栏、provider-side tool use 直接越边界这三类架构缺陷最容易把语言风险放大成物理风险。

## 💎 价值评估

- **🔬 研究价值**：它把具身智能安全从“加个拒答”推进到系统架构级 threat model，适合作为后续机器人 agent 安全论文的分析底板。
- **🚀 实践价值**：主人若做 VLM/VLA 无人机或空中操作系统，这篇能直接指导哪里该插 verifier、safety filter、command sanitizer，而不是出事后补锅。
- **📈 扩展潜力**：后续很适合和 TD calibration、verifier-based VLA、task-level safety wrapper 组合成“高层可信 + 低层可控”的完整部署范式。

## 🎯 可落地实验点

**实验设计**：给现有 LLM/VLA 控制栈加入独立语义校验与动作安全过滤层，做对抗提示和异常观测注入测试
- 对比基线：裸 LLM/VLA 控制链路 / 加语义 verifier / 加 verifier + actuator safety filter
- 度量指标：危险动作放行率、误触发率、任务成功率、额外延迟
- 预期结果：双层护栏能显著降低 unsafe action pass-through rate，代价是可控的小延迟增加
