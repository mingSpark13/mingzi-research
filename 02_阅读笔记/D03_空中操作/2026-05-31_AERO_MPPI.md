# 📖 花火格式笔记

## 🎯 题目

**AERO-MPPI: Anchor-Guided Ensemble Trajectory Optimization for Agile Mapless Drone Navigation**

## 📝 三句摘要

1. **问题背景**：杂乱3D环境中的敏捷地图无关导航对自主无人机挑战极大，传统"建图-规划-控制"串行流水线计算成本高，且误差会逐级传播。

2. **核心方法**：提出 **AERO-MPPI**，基于 Model Predictive Path Integral (MPPI) 框架的全 GPU 加速框架，通过**锚点引导的 MPPI 集成**统一感知与规划，并行采样数千条轨迹以跳出局部最优。

3. **关键结果**：基于 Warp 加速实现 1000+ 并行环境的数千条轨迹滚动模拟；原生集成 Isaac Sim / Isaac Lab；在 ICRA 2026 发表，开源可复现。

## 💎 价值评估

- **🔬 研究价值**：
  - 首个将 MPPI 与 GPU 并行采样深度结合的四旋翼规划器
  - 锚点引导的轨迹集成方法有效解决局部最优问题
  - MPPI 框架本身具有概率最优性保证（理论扎实）

- **🚀 实践价值**：
  - 原生 Isaac Sim/Isaac Lab 集成，仿真到实物桥接顺畅
  - Warp 内核加速，工程落地可行
  - GitHub 已开源（ICRA 2026 官方接收）

- **📈 扩展潜力**：
  - 可扩展到多无人机协同
  - 锚点引导策略可迁移到其他 MPPI 变体
  - 可与 Learning-based 感知融合（如深度点云避障）

## 🎯 可落地实验点

**实验设计**：在 Fast-Drone-250 或 RotorS 仿真环境中对比 AERO-MPPI 与现有 EGO-Planner

- **对比基线**：EGO-Planner、RMPL 等主流四旋翼规划器
- **实现方案**：
  1. 搭建相同仿真环境（杂乱室内/森林场景）
  2. 统一使用深度传感器输入（RealSense D435i）
  3. 对比：成功率、规划时间、轨迹平滑度、路径长度
  4. 消融锚点数量对局部最优逃逸率的影响
- **度量指标**：
  - 成功率 (%)
  - 端到端任务时间 (s)
  - 轨迹 Jerk (m/s³)
  - GPU 内存占用
- **预期结果**：AERO-MPPI 在高密度障碍场景下成功率更高，规划时间更稳定

## 📌 待探索问题

1. **锚点数量与计算开销的权衡**：锚点越多逃逸局部最优能力越强，但 GPU 并行度是否受限？anchor-guided ensemble 的最优规模如何确定？
2. **与 Learning-based 感知的结合**：当前方法假设点云输入可免费获取（如深度相机），若换成端到端感知网络（OccPred 等），MPPI 的计算图如何保持可微？
3. **Warped MPPI 的实时性边界**：1000+ 并行环境采样在高端 GPU（如 RTX 4090）上表现如何？嵌入式 Jetson 上的可行性？
