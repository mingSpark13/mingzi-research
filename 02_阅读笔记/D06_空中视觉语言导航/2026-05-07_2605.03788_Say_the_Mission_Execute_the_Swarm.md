# 📖 花火格式笔记

## 🎯 题目

Say the Mission, Execute the Swarm: Agent-Enhanced LLM Reasoning in the Web-of-Drones

## 📝 三句摘要

1. **问题背景**：让用户直接用自然语言控制无人机群很诱人，但多机异构接口、实时闭环执行和安全约束会让纯 LLM 推理很容易失控。
2. **核心方法**：论文提出 agent-enhanced LLM 框架，把 Agent Core、MCP gateway 与基于 Web of Things 的 Web-of-Drones 抽象结合起来，用工具调用、状态观察和受约束执行替代直接代码生成。
3. **关键结果**：在 ArduPilot 仿真四类群体任务上，论文发现通用 LLM 单靠推理并不可靠，而 grounded tools、task-specific planning 和 runtime guardrails 能显著提升执行鲁棒性。

## 💎 价值评估

- **🔬 研究价值**：这篇把 D06 从“单机语言导航”往“多机任务级语言执行”推了一步，还明确证明了 grounding/guardrails 是空中 agent 系统的必要条件。
- **🚀 实践价值**：对主人后续做无人机 agent、群体调度、MCP 工具体系特别贴脸，可以直接拿来当“自然语言任务→多机执行”的系统设计参考。
- **📈 扩展潜力**：后续可以把高层任务分解、形式化约束、安全监控和地图/世界模型结合，做更强的可验证多机空中 agent。

## 🎯 可落地实验点

**实验设计**：搭一个最小版“语言任务→多机无人机执行”原型，验证 grounded agent 框架的收益
- 对比基线：纯 LLM 直接规划、LLM+工具调用、LLM+工具调用+runtime guardrails
- 度量指标：任务成功率、重规划次数、越界/碰撞率、平均响应延迟
- 预期结果：仅靠 LLM 文本推理会在闭环任务中频繁失稳，而加入工具接地与安全护栏后成功率和稳定性显著提升
