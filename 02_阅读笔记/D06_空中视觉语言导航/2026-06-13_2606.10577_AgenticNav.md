# 📖 花火格式笔记

## 🎯 题目

AgenticNav: Zero-Shot Vision-and-Language Navigation as a Tool-Calling Harness

## 📝 三句摘要

1. **问题背景**：现有 zero-shot VLN-CE 往往依赖 waypoint predictor 和冗长历史上下文，既限制动作空间，也让深度和记忆信息利用得不够聪明。
2. **核心方法**：AgenticNav 把 VLM 与环境的接口重构成 tool-calling harness，让模型按需调用三类工具——像素级 action tool、局部 depth tool、选择性 memory recall——而不是被固定候选航点和长上下文绑死。
3. **关键结果**：在 R2R-CE 上，AgenticNav 在相同 VLM backbone 下刷新 zero-shot SOTA，SR 达 55%、SPL 达 48.41%，并通过真实机器人实验验证了更强的零样本泛化。

## 💎 价值评估

- **🔬 研究价值**：这篇把 VLN 里的“动作/深度/记忆”全改成 agentic tool 接口，很像主人现在常做的 planner-executor 拆分，只不过落到导航里更干净。
- **🚀 实践价值**：对 D06 特别有价值，因为它直接说明 depth 和 memory 不该一股脑塞给大模型，而应该按需查询，能减少无效上下文和错误几何推断。
- **📈 扩展潜力**：如果把 action tool 从地面像素落点换成无人机可飞行局部目标，再接上现有避障/控制器，就很适合做空中版 agentic VLN。

## 🎯 可落地实验点

**实验设计**：在 D06 现有空中 VLN 框架里，实现一个 tool-calling planner：大模型只负责调用“目标像素选择 / 指定点深度查询 / 关键帧回看”三个工具，底层继续由已有飞控执行。
- 对比基线：Goal2Pixel、Beyond Waypoints、AgentVLN
- 度量指标：成功率、SPL、每步平均 token 消耗、depth 查询次数、无效回忆率
- 预期结果：tool-calling 版会减少冗长上下文堆积，并在未见场景里给出更稳的决策链
