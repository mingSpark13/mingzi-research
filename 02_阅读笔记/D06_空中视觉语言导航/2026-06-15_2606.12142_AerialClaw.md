# 📖 花火格式笔记

## 🎯 题目

AerialClaw: An Open-Source Framework for LLM-Driven Autonomous Aerial Agents

## 📝 三句摘要

1. **问题背景**：现有 UAV 系统大多仍靠预定义指令链和手工拼接感知-规划-控制模块，难以支持开放任务下的闭环自主决策。
2. **核心方法**：论文提出 AerialClaw，把自然语言任务理解、文档驱动 soft skill、可执行 hard skill、运行时安全校验和记忆反思整合成 brain-skill-runtime 的空中智能体框架。
3. **关键结果**：该框架支持 mock/PX4 SITL/AirSim 多执行后端，重点价值不在单点 SOTA 指标，而在于给 LLM 驱动空中 agent 提供了可复现实验底座和安全边界设计。

## 💎 价值评估

- **🔬 研究价值**：它把“LLM 作为空中高层决策器”从概念 demo 推进到可复现系统框架，正好贴住 AirSpark 当前 executor-facing packet contract 之外的 agent runtime 层研究空白。
- **🚀 实践价值**：对主人做 UAV agent / OpenClaw 风格 aerial stack 很有参考性，尤其是 skill 抽象、运行时反馈闭环、文档驱动边界约束这三块。
- **📈 扩展潜力**：后续可直接拿来对照 AirSpark 的 planner-side / executor-side 分层，分析哪些能力属于高层 agent orchestration，哪些不能冒领成飞控/执行器增益。

## 🎯 可落地实验点

**实验设计**：把 AerialClaw 式 brain-skill-runtime 分层映射到 AirSpark 原型，单独评估高层 agent 决策收益与底层执行稳定性的边界。
- 对比基线：纯预定义任务树 / 无记忆的 LLM planner / AerialClaw 式闭环 agent
- 度量指标：任务成功率、重规划次数、平均工具调用步数、异常恢复成功率、端到端时延
- 预期结果：AerialClaw 式架构应明显提升开放任务适应性，但若 executor 接口不稳，收益会停在 planner 层而无法转化为真实飞行表现。
