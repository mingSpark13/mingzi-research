# 📖 花火格式笔记

## 🎯 题目

RL-STPA: Adapting System-Theoretic Hazard Analysis for Safety-Critical Reinforcement Learning

## 📝 三句摘要

1. **问题背景**：安全关键场景里的强化学习策略很黑箱，传统评测很难系统性找出部署时会触发的危险行为，尤其在无人机导航和降落任务里风险更高。
2. **核心方法**：论文把 STPA 危害分析迁到 RL 领域，结合分阶段子任务拆解、coverage-guided perturbation testing 和训练回注闭环，形成 RL-STPA 框架。
3. **关键结果**：在自主无人机导航与降落案例中，RL-STPA 能挖出常规 RL 评测漏掉的潜在失效路径，并把这些风险重新反馈进奖励设计与课程训练。

## 💎 价值评估

- **🔬 研究价值**：这不是单纯提性能，而是把“怎么系统找出 RL 导航会死在哪”讲清楚了，特别适合主人后面做 UAV 安全跟随/降落。
- **🚀 实践价值**：可直接迁成主人实验流程里的 safety checklist，在每轮训练后做扰动扫描、危险场景回放、奖励修补。
- **📈 扩展潜力**：能和 Isaac Sim 的自动评测、early-stop、异常报警结合，形成更完整的训练—评测—修复闭环。

## 🎯 可落地实验点

**实验设计**：在 Isaac/AirSim 中给 UAV 跟随或降落任务加入 RL-STPA 式 hazard review，对策略做阶段化风险扫描。
- 对比基线：只看 success rate 的常规 RL 评测
- 度量指标：危险状态覆盖率、近失控事件数、修补后成功率与安全率
- 预期结果：同等训练预算下，RL-STPA 流程能更早发现脆弱状态分布，并降低真实部署炸机风险
