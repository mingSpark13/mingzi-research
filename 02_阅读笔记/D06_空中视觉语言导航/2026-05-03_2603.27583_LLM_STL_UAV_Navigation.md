# 📖 花火格式笔记

## 🎯 题目

LLM-Enabled Low-Altitude UAV Natural Language Navigation via Signal Temporal Logic Specification Translation and Repair

## 📝 三句摘要

1. **问题背景**：低空无人机自然语言导航在城市环境里很容易被模糊指令、时空约束和安全要求卡死，单纯把语言转航点不够稳也不够可解释。
2. **核心方法**：论文把自然语言先翻译成 STL（Signal Temporal Logic）规格，再用 MILP 合成轨迹；同时加入 LLM+MILP 联合的 specification repair，在指令不可行时只放松任务约束、不放松安全约束。
3. **关键结果**：仿真和真实飞行都表明，这套 NL→STL→轨迹→修复闭环能显著提升翻译鲁棒性和任务可执行性，适合复杂城市场景下的安全导航。

## 💎 价值评估

- **🔬 研究价值**：它不是泛泛地做“语言到动作”，而是把 UAV 语言导航落到形式化时序逻辑和可验证修复机制上，正好补 D06 里“可解释、安全可验证导航”的缺口。
- **🚀 实践价值**：对明子的 UAV 人体跟随/低空服务任务很实用，后续可把“绕开人群、经过地标、保持高度窗口”这类约束直接变成 STL 模板。
- **📈 扩展潜力**：可以继续接 D01/D02 路线，把 world model 或 VLA 输出作为 STL grounding 模块前端，形成“语言理解—约束生成—安全控制”三段式系统。

## 🎯 可落地实验点

**实验设计**：在 AirSim / Isaac Sim 无人机场景中复现语言→STL→MILP 规划链路，并加入指令不可行时的自动修复。
- 对比基线：语言直接生成航点、语言直接生成动作、无 repair 的 STL 规划
- 度量指标：任务成功率、约束满足率、碰撞率、修复次数、规划时间
- 预期结果：带 repair 的 STL 框架在复杂约束和模糊指令下更稳，尤其在“既要到目标，又要避障/限高/限时”任务上优势明显
