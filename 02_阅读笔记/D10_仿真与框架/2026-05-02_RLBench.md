# 📖 花火格式笔记

## 🎯 题目

RLBench: The Robot Learning Benchmark & Learning Environment

## 📝 三句摘要

1. **问题背景**：机器人学习缺乏统一的大规模 benchmark，现有数据集任务单一、规模小，难以评估算法的泛化能力和多任务学习能力
2. **核心方法**：提出 RLBench，包含 100+ 完全独特的手工设计任务，基于 CoppeliaSim (PyRep)，提供丰富的观测模式（RGB/Depth/Segmentation/State），支持无限 demo 生成
3. **关键结果**：成为 VLA/机器人学习领域的事实标准 benchmark，支持 RL/IL/多任务学习/few-shot learning 等多种研究方向，已被广泛用于评测 OpenVLA、RT-2、PerAct 等模型

## 💎 价值评估

- **🔬 研究价值**：首个大规模、多样化的机器人操作 benchmark，100+ 任务覆盖从简单到复杂的完整难度梯度，为机器人学习提供了统一评测标准
- **🚀 实践价值**：基于 CoppeliaSim，部署成本低（不需要 NVIDIA GPU），支持 Gym 接口，易于集成到现有 RL/IL 框架，支持无限 demo 生成（motion planner）
- **📈 扩展潜力**：可作为 UAV 操作 benchmark 的参考（任务设计、评测指标、demo 生成），或用于 VLA 预训练/评测（大规模多任务数据）

## 🎯 可落地实验点

**实验设计**：RLBench-style UAV manipulation benchmark
- **对比基线**：
  - RLBench（地面机械臂）
  - UAV-RLBench（空中机械臂，本文方法）
  - Isaac Sim（NVIDIA GPU 加速）
- **度量指标**：
  - Task success rate（任务成功率）
  - Demo generation speed（demo 生成速度）
  - Sim-to-real gap（仿真到真实的差距）
  - Multi-task learning performance（多任务学习性能）
- **预期结果**：UAV-RLBench 在任务多样性、demo 生成速度上与 RLBench 相当，证明 RLBench-style benchmark 对 UAV 操作的适用性

## 🛠️ 技术细节

### 核心依赖

- **仿真器**：CoppeliaSim v4.1.0（前身为 V-REP）
- **Python 接口**：PyRep（`stepjam/PyRep`）
- **机械臂**：默认 Franka Panda，可替换为 Mico 等
- **物理引擎**：Bullet/ODE/Vortex（可选）

### 观测模式

- **RGB**：over-the-shoulder 双目 + eye-in-hand 单目
- **Depth**：深度图
- **Segmentation**：分割 mask
- **State**：关节位置/速度/力矩、末端执行器 6D pose

### 任务类型

- **简单任务**：reach target、push button、open door
- **中等任务**：pick-place、stack blocks、insert peg
- **复杂任务**：open oven and place tray、empty dishwasher、water plant

### Demo 生成

- 基于 motion planner（RRT/PRM）+ waypoints
- 无限生成（不需要人工演示）
- 保证无碰撞、可执行

### Gym 接口

```python
import gym
import rlbench.gym

env = gym.make('reach_target-state-v0')  # state observation
# or
env = gym.make('reach_target-vision-v0')  # vision observation
```
