# 📖 花火格式笔记

## 🎯 题目

SceneAssistant: A Visual Feedback Agent for Open-Vocabulary 3D Scene Generation

## 📝 三句摘要

1. **问题背景**：现有text-to-3D方法一次性生成场景，难以保证空间关系符合文本描述，且缺乏对生成过程的有效控制。
2. **核心方法**：提出SceneAssistant，让VLM接收渲染反馈后，通过Scale/Rotate/FocusOn等action API逐步修改场景，形成"渲染-反馈-修改"的视觉反馈循环，而非一次性生成。
3. **关键结果**：通过迭代修正显著提升场景与文本描述的空间一致性，验证了视觉反馈在open-vocabulary场景生成中的有效性。

## 💎 价值评估

- **🔬 研究价值**：路线B的另一个代表性工作，与SAGE互补——SAGE侧重physics critic，本文侧重visual feedback loop；两者共同构成完整的"生成-验证-修复"体系。
- **🚀 实践价值**：视觉反馈循环可让主人实时看到场景生成效果并干预，特别适合调试低空场景的语义完整性（"左侧建筑是否真的在左侧？"）。
- **📈 扩展潜力**：可与主人的VLM Critic结合，形成"低空场景Spec → PCG生成 → 渲染top-down map → VLM反馈 → 修正"的交互式生成界面。

## 🎯 可落地实验点

**实验设计**：将SceneAssistant的视觉反馈循环迁移到低空场景生成：UE PCG生成初版场景 → 渲染top-down semantic map → VLM对比Spec检查"道路连通性/建筑位置/目标分布" → 有误则修正后重新生成。
- 对比基线：无视觉反馈的一次性生成、人工审核后修正
- 度量指标：Prompt-Scene Alignment Score、修正轮次、最终场景合格率
- 预期结果：视觉反馈循环使场景合格率从~70%提升到~90%以上
