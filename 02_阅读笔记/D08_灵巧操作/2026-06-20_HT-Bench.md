# 📖 花火格式笔记

## 🎯 题目

**HT-Bench: Benchmarking and Learning Dexterous Full-Hand Tactile Representations with Egocentric Vision**

## 📝 三句摘要

1. **问题背景**：触觉表示学习在机器人灵巧操作中缺乏统一评测基准——触觉传感器形态、数据格式、机器人本体各不相同，难以横向对比。
2. **核心方法**：提出 HT-Bench 大规模多任务基准（10M RGB 帧 + 7.8M 触觉帧，覆盖 226 个任务），并提出 HandTouch 向量量化视觉-触觉编码器，通过渐进式空间/跨模态/时序训练学习触觉表征。
3. **关键结果**：在细粒度触觉相似度检索上 Recall@5 从 74.65% 提升到 85.23%；掩码触觉修复 RMSE 从 0.022 降到 0.010；OOD 视觉到触觉合成 cIoU 从 0.628 提升到 0.705。

## 💎 价值评估

- **🔬 研究价值**：HT-Bench 是当前规模最大的全手触觉-视觉联合基准，首次以"四任务"（相似度检索/掩码修复/视觉到触觉合成/帧预测）系统评测触觉编码器，给出了后续研究的对照基线。
- **🚀 实践价值**：HandTouch 编码器作为预训练 backbone 可直接用于下游灵巧操作策略；10M 帧数据集对数据飞轮（D05）有直接借鉴价值，可考虑迁移到无人机空中操作触觉感知场景。
- **📈 扩展潜力**：①把视觉-触觉对齐范式从手内操作扩展到足式/无人机平台；②与 D01 隐空间世界模型结合，把触觉观测作为 latent space 的额外模态约束；③可作为主人在 D07 Isaac 仿真中的触觉信号补充源。

## 🎯 可落地实验点

**实验设计**：在 Genesis 或 Isaac Sim 仿真中复现 HT-Bench 的"vision-to-tactile synthesis"任务，比较 HandTouch 与现有 VT 编码器在跨视点/跨材质 OOD 场景下的 cIoU；进一步把训练好的编码器冻结 backbone，微调下游抓取预测任务（D07 抓取检测 + D08 灵巧操作），验证表征迁移性。

- 对比基线：GelSight 触觉编码器 / 现有 VT 编码器 / HandTouch
- 度量指标：cIoU、相似度 Recall@5、下游任务成功率
- 预期结果：HandTouch 在 OOD 合成任务上应明显优于基线，下游抓取成功率提升 5-10%
