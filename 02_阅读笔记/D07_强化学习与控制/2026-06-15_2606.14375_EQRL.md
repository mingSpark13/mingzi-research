# 📖 花火格式笔记

## 🎯 题目

Elastic Queries Reinforcement Learning: Self-Aware Policy Execution for VLA Models

## 📝 三句摘要

1. **问题背景**：VLA 机器人策略通常固定推理步数、固定重规划频率、固定动作 chunk 长度，完全没管不同状态的难度差异，导致算力浪费或关键时刻算不够。
2. **核心方法**：EQRL 把每次 policy query 变成可弹性调度的联合动作，用 RL 学一个 latent-schedule adaptor，同时决定 latent input、denoising budget 和 action chunk length，并用 critic 集成分歧估计状态难度。
3. **关键结果**：在仿真和真实机器人 manipulation 上，EQRL 能在降低平均推理开销的同时保持甚至提升任务成功率，说明“算力按难度分配”比固定执行策略更稳。

## 💎 价值评估

- **🔬 研究价值**：这篇抓住了 VLA 执行层真正痛点——不是再换 backbone，而是让 query 频率、去噪步数、动作 chunk 长度随状态难度动态变化。
- **🚀 实践价值**：对主人现在 D07 和 D02 都值钱，尤其适合作为“有限机载算力 / 板端部署 / 实时控制”下的执行层改造思路。
- **📈 扩展潜力**：后面可以和异步执行、事件触发重规划、planner-side uncertainty 结合，做成统一的 compute-aware policy execution 框架。

## 🎯 可落地实验点

**实验设计**：给现有 VLA 或 diffusion policy 外挂一个调度头，动态决定推理步数和动作 chunk 长度，测试算力-成功率折中。
- **对比基线**：固定 denoising steps + 固定 chunk / 仅动态 chunk / 仅动态重规划频率
- **度量指标**：任务成功率、平均 NFE、单步延迟、接触阶段失败率、重规划次数
- **预期结果**：在接触密集阶段自适应增加 query 与去噪预算，在简单开环阶段拉长 chunk，整体算力更省且成功率不降
