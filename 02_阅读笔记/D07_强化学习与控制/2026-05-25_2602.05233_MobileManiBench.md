# 📖 花火格式笔记

## 🎯 题目

MobileManiBench: Simplifying Model Verification for Mobile Manipulation

## 📝 三句摘要

1. **问题背景**：移动操作模型常缺统一、可扩展且低人工成本的验证基准，导致策略泛化与评测难以稳定比较。
2. **核心方法**：论文基于 Isaac Sim 与强化学习自动生成多样化移动操作轨迹，输出多视角 RGB-D、分割、机器人状态和语言指令，构成面向模型验证的数据与任务基准。
3. **关键结果**：它更像 D07 的评测/数据基础设施补丁而非控制算法突破，价值在于把移动操作验证流程标准化并降低合成数据构建成本。

## 💎 价值评估

- **🔬 研究价值**：为 D07 补上“Isaac Sim 中如何系统验证移动操作模型”的基准位，适合作为 whole-body/局部补偿路线的共同评测壳。
- **🚀 实践价值**：可直接借鉴其自动轨迹生成、多模态导出与任务验证流程，减少自建 benchmark 的工程成本。
- **📈 扩展潜力**：后续可改造成空中机械臂或 fixed-flight manipulation 的数据/验证管线，而不必直接照搬其任务定义。

## 🎯 可落地实验点

**实验设计**：把 MobileManiBench 的“自动生成验证集 + 多视角观测导出”思想迁移到 D07 首轮实验协议。
- 对比基线：手工构造少量验证任务 vs 自动生成验证任务集
- 度量指标：任务覆盖率、失败模式覆盖率、验证集构建时间、不同控制范式的区分度
- 预期结果：自动化验证集能更快暴露 warm-start / hybrid / local compensation 三路线的真实差异
