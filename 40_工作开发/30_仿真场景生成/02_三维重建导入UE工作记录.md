# 三维重建导入 UE 工作记录 — Q9000 点云 → 3DGS → UE5

> 创建日期：2026-05-14 | 维护者：花火
> 
> 本文档记录探索三维重建导入 UE 使用的工作，包括 Q9000 点云处理、3DGS 训练、UE5 导入等完整工作流。

---

## 一、当前痛点分析

### Q9000 点云导入 UE 效果差的根本原因

- **LiDAR 点云是离散点**，没有表面/纹理，UE 渲染的是"点的集合"而非连续表面
- **几千万点直接渲染性能极差**，视觉上仍然是点云感
- **3DGS 解决的正是这个问题**：用高斯椭球表达连续表面+外观，渲染效果接近照片级

## 二、工作流分析

网上找到的工作流**整体方向是对的**，但有几个关键问题需要指出：

### 优点

| 步骤 | 评价 |
|------|------|
| Q9000 点云作为尺度/对齐参考 | 非常正确，LiDAR 提供精准的几何真值 |
| SuperSplat 清理 | 必要步骤，训练完通常有漂浮高斯 |
| 控制点辅助对齐 | 专业做法，保证 3DGS 和 LiDAR 尺度一致 |

### 问题与风险

#### 1. Nerfstudio Splatfacto（步骤4-6）—— 不是最佳选择

Splatfacto 是学术向实现，工程化稳定性一般。更推荐：

- **3DGS 官方实现**（graphdeco-inria/gaussian-splatting）：质量更可控
- **Luma AI / Polycam**（商业方案）：如果场景允许云端处理，质量上限更高
- **InstantSplat++**：适合稀疏视角快速验证，**不适合**高精度工程化场景（后面详述）

#### 2. 步骤9 的尺度对齐是难点，没有说清楚具体方法

3DGS 坐标系是任意的，需要手动对齐到 LiDAR 的真实坐标系，建议用 CloudCompare 的 ICP（迭代最近点）或手动 4 点对齐。

#### 3. 步骤11 的"AirSim 相机能否捕获 3DGS"——需要验证

UE 的 SceneCapture 支持 3DGS（MLSLabsRenderer 文档里有说明），但 AirSim/UE 联合仿真中 3DGS 的相机可见性需要单独测试。

## 三、InstantSplat++ 是否适合这个工作流？

**简短回答：不适合作为主力，适合快速验证。**

| 维度 | InstantSplat++ | 标准 3DGS 工作流 |
|------|----------------|-----------------|
| 输入图片数量 | 2-6 张（设计目标） | 100-1000+ 张 |
| 重建精度 | 中等，稀疏视角局限 | 高，多视角全覆盖 |
| 场景覆盖完整性 | 局部场景 | 完整场景 |
| 工程化稳定性 | 弱（研究代码） | 较强 |
| 适合校园级场景 | 否 | 是 |
| 适合快速概念验证 | 是 | 否（慢） |

InstantSplat++ 的 MASt3R 位姿估计在**无 COLMAP 的情况下**很有价值，但你有 Q9000 点云，意味着已经有精准几何基础，完全可以用 COLMAP 做相机标定，然后用标准 3DGS 训练。

## 四、推荐方案

针对"校园扫描→高质量 3DGS→UE5"这个目标：

### 完整工作流

```
[数据采集]
 Q9000 扫描 → PLY/LAS 点云（几何真值）
 相机/手机拍摄 → 视频或照片
 要求：每平方米至少 3-5 张覆盖，90%+ 重叠率，避免纯色大面积（白墙难匹配）

[位姿估计]
 COLMAP（推荐）或 ns-process-data
 可以用 Q9000 点云作为 sparse points 初始化 → 提升精度

[3DGS 训练]
 gaussian-splatting 官方实现（首选）
 或 Nerfstudio Splatfacto（备选）
 关键参数：--iterations 30000，大场景建议分块训练

[后处理]
 SuperSplat 清理漂浮高斯
 CloudCompare ICP 对齐到 Q9000 坐标系

[UE5 导入]
 MLSLabsRenderer（你已在方案中）
 导入后用 Q9000 点云搭建 collision mesh（CloudCompare → MeshLab 简化 → UE）
```

## 五、关于 Q9000 导入 UE 效果差的解决思路

如果已经有精准点云，有两条路：

### 1. 纯点云路线

用 MeshLab/CloudCompare 对点云做 Poisson 重建（生成 mesh）→ 贴纹理 → 导入 UE。

- **效果**：中等
- **适合**：需要碰撞的场景

### 2. 3DGS 路线（推荐）

用点云做 COLMAP 初始化，配合照片训练 3DGS → 导入 UE。

- **效果**：视觉效果最好
- **要求**：需要拍照片补充外观信息
