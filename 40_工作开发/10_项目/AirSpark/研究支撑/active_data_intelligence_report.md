# 面向移动自主机器人的主动数据智能
## 从静态数据集到受约束的自进化数据闭环（重点研究报告）

**研究口径：** 截至 2026 年 6 月；对象覆盖人形机器人、四足机器人、移动操作机器人、无人车、无人机与更广义具身智能体。报告重点讨论数据范式，不以某一模型、数据集或世界模型为中心。

## 一、核心结论

**第一，主动数据智能尚不是国际上定义统一的成熟术语，但其技术内核已经清晰出现。** 现有综述分别讨论数据中心 AI、机器人主动学习、自进化 Agent 和自进化具身智能，却尚未以“机器人如何自主决定下一步需要什么数据，并完成获取、标注、组织、评估和回流”为统一主线。[1–4]

**第二，主动数据智能的关键分界线不是“是否自动化”，而是系统是否能够根据能力缺口自主决定下一项数据行动。** 自动标注、批量仿真和数据清洗仍可能只是人工设定目标的流水线；只有当系统基于失败、不确定性、分布偏移、任务缺口和安全风险，主动选择采集、生成、请求人类、标注、筛选、混合或舍弃数据，才进入主动数据范式。

**第三，2024 年以来已经出现从局部技术走向闭环系统的明确信号。** AutoRT 用基础模型为机器人车队提出并筛选采集任务；Autonomous Improvement 用 VLM 自动采集、评价并利用无人工标注经验；Re-Mix 和 DataMIL 开始以模型收益而不是人工直觉评估数据；Robot-Powered Data Flywheel 和 RoboClaw进一步把部署、采集、标注、训练与再部署串联起来。[13–21]

**第四，世界模型、数字孪生和生成式 AI 是主动数据系统的“数据生产与推演工具”，不是最终归宿。** 可靠的自进化必须由真实环境反馈锚定，由仿真和生成扩大覆盖，由独立评测、安全门控和回滚机制防止错误自强化。

> **中心判断：主动数据智能正在成为移动机器人与具身智能的关键基础设施，但当前仍处于“自动化数据管线”向“主动数据智能体”过渡阶段，尚未形成成熟、通用、可验证的自进化体系。**

## 二、概念凝练：什么是主动数据智能

### 2.1 工作定义

**主动数据**是由任务目标、能力缺口、模型不确定性、失败事件和环境变化驱动，经过有目的选择、获取、生成或标注的数据。

**主动数据智能**是机器人或数据智能体依据模型状态、任务表现和部署反馈，自主决定并执行数据获取、生成、标注、组织、评估、利用与回流，以最大化单位成本和风险下能力增益的系统能力。

可将其抽象为数据行动决策：

> 在真实采集、请求示范、仿真生成、自动标注、历史检索、数据筛选、重放训练和数据舍弃等候选行动中，选择预期能力增益最大、成本与风险可控的下一项行动。

### 2.2 与相邻概念的边界

| 范式 | 核心问题 | 数据目标由谁提出 | 是否形成部署反馈闭环 |
|---|---|---|---|
| 静态数据集 | “已有数据能否训练模型？” | 人工预先规定 | 否 |
| 自动化数据管线 | “如何更快采集、标注和清洗？” | 人工规定，机器执行 | 通常否 |
| 主动数据 | “当前最缺、最值得获得的数据是什么？” | 模型或系统根据反馈判断 | 局部形成 |
| 主动数据智能体 | “如何规划并调度完整数据生命周期？” | 数据 Agent 自主编排机器人、仿真器、模型与人 | 是 |
| 受约束自进化 | “如何长期增长能力且不错误自强化？” | 系统自主，但受验证、安全和治理约束 | 持续闭环 |

**主动学习**主要解决“标哪些样本”；**持续学习**主要解决“模型如何利用新数据更新”；**世界模型**主要提供预测、反事实和合成环境。主动数据智能则负责把这些能力组织为面向能力增长的数据决策系统。

## 三、从大型静态数据集到主动闭环的演进

传统机器人数据范式首先解决“规模不够、平台不统一、场景不多”的问题。RoboNet、BridgeData V2、Open X-Embodiment、DROID、RoboMIND 和 AgiBot World不断扩大跨场景、跨任务和跨本体数据规模；Waymo Open Motion 已出现兴趣场景挖掘和自动标注等数据引擎特征；但这些数据一旦发布，训练目标、采集策略和数据组成通常仍固定，模型上线后的失败不会自动决定下一轮数据生产。[5–12]

### 表 1 代表性传统或过渡型数据资源

| 数据集/平台 | 对象与规模 | 数据生产特点 | 相对主动数据智能的局限 |
|---|---|---|---|
| RoboNet（2019）[5] | 操作机器人；1500 万帧、7 种平台 | 多机构、多机器人经验汇聚 | 静态共享池，不诊断具体能力缺口 |
| Waymo Open Motion（2021）[6] | 无人车；10 万场景、570 小时 | 挖掘有交互价值场景，3D 自动标注 | 具有数据引擎雏形，但发布后仍是固定数据集 |
| BridgeData V2（2023）[7] | 操作机器人；60,096 条轨迹、24 个环境 | 低成本平台、跨环境示范 | 采什么、如何配比仍由人工预先决定 |
| Open X-Embodiment（2023）[8] | 22 种机器人、527 项技能、160,266 个任务 | 统一异构数据格式，促进跨本体迁移 | 数据聚合强，在线反馈与主动补数弱 |
| AerialVLN（2023）[9] | 无人机；25 个城市级仿真场景 | 固定路线指令和离线基准 | 缺少部署失败驱动的再采集与再训练 |
| DROID（2024）[10] | 操作机器人；7.6 万条轨迹、350 小时、564 场景 | 分布式真实场景遥操作 | 采集规模大，但基本仍是人主导离线建设 |
| RoboMIND（2024）[11] | 多本体操作；10.7 万轨迹、479 任务 | 含 5000 条失败示范与数字孪生 | 已重视失败数据，但尚未自主决定如何补齐失败模式 |
| AgiBot World（2025）[12] | 具身操作；超过 100 万轨迹、217 任务 | 标准化采集、人机复核、跨夹爪/灵巧手 | 代表工业级规模化数据工程，闭环数据决策仍有限 |

演进的实质不是简单从“小数据”走向“大数据”，而是依次跨越四个门槛：

1. **数据规模化：** 能持续采集和统一多源经验；
2. **数据自动化：** 自动标注、仿真扩增、失败检测和质量控制；
3. **数据主动化：** 根据模型能力缺口决定采什么、生成什么、保留什么；
4. **数据智能体化：** 自主调度真实机器人、数字孪生、生成模型和人类反馈，完成训练与验证闭环。

## 四、2024 年以来最具代表性的主动数据工作

### 表 2 新型优质工作与主动数据环节对比

| 工作 | 主动数据环节 | 关键证据 | 研究判断与不足 |
|---|---|---|---|
| AutoRT（2024）[13] | 主动探索、任务提出、车队调度、安全筛选 | 53 台机器人、7 个月、7.7 万真实回合；LLM/VLM提出并过滤任务 | 首个强系统级“数据采集编排器”；但能力缺口诊断和自动训练闭环仍弱 |
| Autonomous Improvement（2024）[14] | 自主采集、VLM 自动评价、无标注学习 | 5 个真实环境、3.05 万轨迹，策略在新环境中提升约 2 倍 | 接近完整数据—模型闭环，但任务范围仍以桌面操作为主 |
| Re-Mix（2024）[15] | 主动组织与数据混合优化 | 学习到的数据权重平均优于均匀混合 38%，优于人工配比 32% | 证明“数据组成”比单纯规模更重要；不负责新数据获取 |
| SRDF（2024）[16] | 生成—筛选—再生成的数据自精炼 | 导航 SPL 从 70% 提升至 78%，生成器与导航器互相改进 | 清晰的数据飞轮；主要依赖离线/仿真导航，现实锚定不足 |
| DataMIL（2025）[17] | 面向任务成功率的数据价值评估与选择 | 在 60 余项仿真和真实任务中，用策略本身判断哪些数据有益或有害 | 从“人认为相似”转向“模型实际受益”；尚未统一采集与标注 |
| CoReVLA（2025）[18] | 失败/接管触发采集、偏好学习、持续改进 | 在驾驶仿真中用接管标记长尾失败，再以 DPO 优化 | 自动驾驶数据闭环的典型形态；实车规模和安全验证仍待加强 |
| Robot-Powered Data Flywheel（2025）[19] | 部署即采集、外部知识自标注、再训练 | 移动操作机器人两周扫描 2103 个书架，识别率 32.0%→71.8% | 最直接证明“机器人从模型消费者变为数据生产者”；任务域较窄 |
| DexFlyWheel（2025）[20] | 种子示范、IL+RL、策略 rollout、数据增强、迭代 | 1 条种子示范起步，生成 2000+ 示范，挑战集成功率 81.9% | 展示生成式数据飞轮；高度依赖仿真和数字孪生可信度 |
| RoboClaw（2026）[21] | 采集—学习—执行统一、自动复位、在线策略精炼 | 长时任务成功率较基线提升 25%，人工投入下降 53.7% | 已接近“主动数据智能体”原型；仍局限于封闭任务和预设技能集合 |

这些工作共同表明，领域重点正在发生三次迁移：

- 从**采集更多数据**转向**发现最有边际价值的数据**；
- 从**离线数据集建设**转向**部署过程持续生产数据**；
- 从**独立的采集、训练和评测工具**转向**由 Agent 编排的闭环数据系统**。

但目前尚无系统同时成熟解决“能力缺口诊断、数据行动规划、多源生产、可靠标注、价值评估、持续学习、安全验证和治理”。因此，现有成果更准确地说是**局部数据飞轮或主动数据组件**，而非完整自进化机器人。

## 五、未来主动数据智能的技术体系

未来的基础设施不应是单一世界模型，而应是一套位于机器人本体、真实环境、仿真平台、基础模型与训练系统之间的“数据操作系统”。建议凝练为五个模块：

| 模块 | 核心能力 | 代表技术 |
|---|---|---|
| 1. 能力缺口感知 | 识别失败、低置信、分布偏移、技能空白和安全风险 | 不确定性估计、异常检测、失败归因、任务覆盖图谱 |
| 2. 数据行动规划 | 决定采集、生成、请求人类、检索、标注、筛选或舍弃 | 数据价值预测、成本—风险约束规划、主动学习、数据 Agent |
| 3. 多源数据生产 | 调度真实机器人、车队、遥操作、仿真、数字孪生和生成模型 | 主动探索、自动复位、场景生成、反事实轨迹、群体协同采集 |
| 4. 数据理解与治理 | 自动标注、轨迹分段、质量校验、语义组织、血缘与版本管理 | VLM 标注、多传感器一致性、数据知识图谱、价值归因、去重与混合优化 |
| 5. 学习—验证—发布闭环 | 增量更新、独立评测、安全门控、回滚和再部署 | 持续学习、离线/在线评测、影子模式、闭环仿真、红队测试 |

其基本闭环为：

> **能力监测 → 数据需求诊断 → 数据行动规划 → 真实/仿真/生成协同生产 → 智能标注与价值评估 → 模型更新 → 独立安全验证 → 再部署。**

世界模型主要进入第三和第五模块，用于反事实生成、场景推演和闭环验证；真实运行数据则承担事实锚点和最终裁决者角色。

## 六、适合项目申请凝练的科学问题

### 1. 能力缺口如何转化为可执行的数据需求？

现有系统多以不确定性或失败触发采集，但尚缺少跨任务、跨平台统一的“能力—数据”映射。需要研究如何从任务失败、感知盲区、动作不可达和环境变化中，推断应补充何种场景、模态、轨迹和反馈。

### 2. 数据价值如何随模型、任务和训练阶段动态变化？

一条数据对不同模型和不同阶段的贡献不同。核心问题是估计数据的边际能力增益，而不是静态质量分数；Re-Mix 和 DataMIL 已证明人工经验或均匀采样并非最优。[15,17]

### 3. 如何在真实、仿真、生成和人类数据之间主动调度？

真实数据可信但昂贵，仿真数据便宜但存在域差，生成数据覆盖长尾但可能失真，人类数据高质量但难扩展。需要建立面向成本、风险、时效和可信度的多源数据组合决策。

### 4. 如何避免智能标注和自训练造成错误闭环？

自动标注器、策略和评测器往往共享基础模型，容易形成同源偏差；错误数据可能在多轮训练中被放大。数字 Agent 研究已观察到“错误进化”风险，机器人系统还叠加了物理安全后果。[22]

### 5. 如何评价“自进化”而非一次性模型性能？

新的评价应同时衡量能力增长率、单位数据收益、真实部署成功率、长尾覆盖、安全违规、灾难性遗忘、跨本体迁移、人工干预成本和可回滚性，而不是只报告固定测试集分数。

## 七、面向项目本子的建议落点

建议将项目主线表述为：

> **研究面向移动自主机器人的主动数据智能理论与系统，突破能力缺口驱动的数据需求建模、多源数据行动规划、智能标注与数据价值评估、真实—仿真—生成协同生产，以及安全可验证的持续学习闭环，构建能够自主发现数据需求、主动组织数据资源并持续提升任务能力的数据智能体。**

可凝练为四项研究内容：

1. **能力缺口驱动的主动数据需求模型；**
2. **面向真实—仿真—生成协同的数据智能体与调度机制；**
3. **多模态智能标注、数据价值评估和动态数据组织；**
4. **安全门控、可追溯、可回滚的机器人自进化闭环。**

最终判断是：主动数据智能不会取代 VLA、世界模型或数字孪生，而会成为组织它们的上层基础设施。未来真正有竞争力的移动机器人系统，不仅要拥有更强模型，还要拥有能够持续回答“下一批最值得获得的数据是什么、怎样安全获得、如何证明它确实提升能力”的数据智能。
