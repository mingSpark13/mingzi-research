## 一、研究背景
空中 VLN/VLA 相比地面面临四大根本差异：
- **6-DoF 运动**：无人机需处理俯仰、翻滚等三维运动
- **动力学约束**：速度、能量、安全距离等物理限制
- **视角剧烈变化**：俯仰角大范围变化导致目标大幅位移
- **导航-操作统一**：天然可拓展为「飞到 → 操作」全链路
---

## 二、五维统一坐标系
本报告用五个维度构成统一分析框架：
**维度一 — 任务类型**
Route-following VLN / Object Search / ObjectNav / High-level VLA / Fine-grained Flow Control / Nav+Manipulation / Dialog Nav / Visual Tracking / Aerial Manipulation
**维度二 — 动作空间**
离散航点（3-4 DoF）/ 连续 6-DoF 轨迹 / 速度控制 / 低层电机控制 / 分层技能动作
**维度三 — 观测模态**
单视角 RGB / 多视角 RGB / RGBD / LiDAR / 状态向量 / 力/力矩 / 子任务标注
**维度四 — 评测指标**
传统 VLN 指标（NE/SR/OSR/SPL）/ 过程导向（TCR/CR/CSPL）/ 部署性能（延迟/吞吐）/ 动力学感知（接触力/负载/恢复）
**维度五 — 现实性**
Synthetic / Real Reconstruction / Real-world Deployment / 代码数据公开度

## 三、各工作详细分析
### 3.1 路线跟随型 VLN
**AerialVLN**（ICCV 2023）— D+B
- 首个空中 VLN 基准，城市级开放环境
- AirSim 仿真，8,446 条轨迹，离散 4-DoF 航点
- 评测：SR / SPL / NE
**AVDN**（ACL 2023 Findings）— D+B+M
- 首个空中多轮对话导航，人类指挥官+无人机专家交互采集
- 6,269 条轨迹，HAA-Transformer 预测航点+人类注意力
- 价值：验证「持续对话 > 单轮指令」
### 3.2 目标搜索 / 连续控制
**CityNAV**（arXiv 2406.14240）— D
- 首个大规模真实城市空中 VLN 数据集
- Cambridge + Birmingham 真实航拍，32,637 条轨迹，4.65 km²
- 核心挑战：地标空间关系理解
**AirNav**（arXiv 2601.03707）— D+B+M（v3.0 新增）
- 当前最大规模真实城市 UAV VLN 数据集，137K navigation samples
- human-LLM collaborative pipeline，10 类 user personas 增强自然语言多样性
- 方法：AirVLN-R1（SFT + RFT 训练范式），真实世界初步测试
- 对城市低空语言导航数据构成直接压力
- 注意：纯离散导航，无连续控制+操作
**OpenUAV / UAV-Need-Help**（arXiv 2410.07087，ICLR 2025）— P+D+B+M
- 基于 UE4 + AirSim 的仿真平台，吸收 CARLA 场景资源
- 22 场景，89 类目标，12,149 条连续 6-DoF 轨迹
- 五视角 RGB/Depth + UAV 状态
- 助手引导三档：L1 高频贴近 GT 连续指导 / L2 偏航时低频纠偏 / L3 仅危险场景安全兜底
- 四层内容统一：平台 + 数据集 + 基准 + 方法
- 最大价值：连续 6-DoF 轨迹数据 + 多视角感知平台
**IndoorUAV**（AAAI 2026）— B（v3.0 新增）
- 连续室内 UAV VLN，基于 Habitat
- 1000+ 室内场景，16K+ instruction-trajectory pairs
- 提示室内/半室内复杂结构导航是低空 benchmark 的一条支线
- 与城市低空操作方向互补，不构成直接竞争
**UAV-ON**（arXiv 2508.00288，ACM MM 2025）— B
- 首个空中开放世界 ObjectNav 基准
- 14 个高保真 UE 环境，1,270 个实例级目标
- 不依赖细粒度路线指令，给高层语义目标自主探索
- 补上「VLN 与 ObjectNav 的边界」
### 3.3 高层 VLA / 过程执行
**HUGE-Bench**（arXiv 2603.19822）— B
- 4 个真实数字孪生场景，3DGS-Mesh 表示
- 8 类高层任务，256 万米轨迹
- 评测重心：TCR / CR / CSPL（过程导向 + 碰撞感知）
- **与 OpenUAV 的本质区别**：OpenUAV 解决「怎样在逼真仿真里完成带辅助的目标搜索」；HUGE-Bench 解决「能否理解一句高层命令并安全完成多阶段任务」。前者是 realistic VLN/search，后者是 high-level VLA/procedural execution
### 3.4 细粒度飞行控制
**UAV-Flow Colosseo**（arXiv 2505.15725，NeurIPS 2025）— D+B+M
- 定义 Flying-on-a-Word (Flow) 任务：语言→细粒度短程反应式飞行控制
- 首个真实世界语言条件细粒度 UAV 控制 benchmark
- 首次真机 VLA 部署，无 sim-to-real gap
- VLA 范式优于 VLN 范式
### 3.5 目标跟踪 / 接近对齐（v3.0 新增）
**CosFly-Track**（arXiv 2605.17776）— D+B（v3.0 新增）
- 城市 UAV 视觉跟踪，约 12,000 expert/perturbed 轨迹，2.4M timesteps，334 小时数据
- 通道：RGB、metric depth、semantic segmentation、6-DoF drone pose、target state/visibility flag、中英双语指令、trajectory-pair metadata
- MuCO（多约束轨迹优化器）：在连续 3D 空间里约束 visibility、viewpoint quality、collision、smoothness、kinematic feasibility
- 定位：tracking-approach-align 阶段的关键数据基础
- 对 AirSpark 的启发：可借鉴 MuCO 设计操作版轨迹优化器
**UAV-Track VLA**（arXiv 2604.02241）— D+B+M（v3.0 新增）
- 890K+ 帧，176 任务，85 目标物
- 基于 π0.5：temporal compression + dual-branch decoder + flow matching action expert
- 面向语义化 UAV tracking，将自然语言与跟踪动作联合建模
- 与 CosFly-Track 互补：CosFly-Track 侧重数据与优化器，UAV-Track VLA 侧重端到端方法
### 3.6 导航 + 操作
**AIR-VLA**（arXiv 2601.21602）— P+D+M
- 首个空中操作标准化数据集，UAV 4D pose + 机械臂关节
- 3000 条人工遥操作 demonstrations
- 覆盖 base manipulation、object/spatial understanding、semantic reasoning、long-horizon planning
- 浮基动力学 + UAV-机械臂耦合
- 局限：实验室场景，无城市低空，无多类接触，无 Sim2Real 配对
**AirVLA**（arXiv 2603.25038）— M（v3.0 新增）
- "π, But Make It Fly"：把 π0 这类 manipulation-pretrained VLA 迁移到 aerial pick-and-place
- 3DGS 合成导航数据，Payload-Aware Guidance（PAG）：在推理时把负载约束注入 flow-matching sampling
- 460 次真实实验：导航 100%（+合成数据）/ 抓取 50%（+PAG）/ 长时域 62%
- 关键发现：负载变化导致高度下沉是空中操作的核心挑战
- 局限：无城市低空、无多类接触、无系统性 Sim2Real 配对
**Flying Hand**（arXiv 2504.10334，RSS 2025）— M（v3.0 新增）
- end-effector-centric 全驱六旋翼 + 4-DoF arm
- whole-body MPC + 高层 policy
- 真实任务：writing、peg-in-hole、pick-and-place、changing light bulbs
- 核心贡献：把"空中操作接口"从机体控制转成"末端执行器控制"
- 对 AirSpark 的启发：末端中心控制接口可作为 baseline/系统设计参考
**AERMANI-VLM**（arXiv 2511.01472）— M（v3.0 新增）
- 分层架构：高层 VLM 推理 + 离散安全技能库 + 底层控制
- 解决直接 VLM 输出动作的幻觉和动力学不可行问题
- 与 AirSpark 的"RPC 低频控制面、数据面模块化"系统思想一致
- 对 AirSpark 的启发：保留分层/模块化策略，而不是纯端到端一把梭
**DroneVLA**（arXiv 2601.13809，HRI 2026）— M
- 全链路：Grounding DINO + VLA + A* + MediaPipe
- 真机定位误差 0.164m（max）/ 0.070m（mean）
### 3.7 综合平台
**OpenFly**（arXiv 2502.18041，ICLR 2026）— P+D+B+M
- UE/GTA V/Google Earth/3DGS 四引擎统一接口
- 100K 级轨迹，自动生成工具链
- keyframe-aware OpenFly-Agent
**AeroVerse**（arXiv 2408.15511）— P+D+B
- 全链路：仿真→预训练→微调→评测
- 5 类下游任务，AerialAgent-Ego10k + CyberAgent-Ego500k
### 3.8 端到端 VLA 方法
**AutoFly**（ICLR 2026 Poster）— M+D
- pseudo-depth encoder，渐进两阶段训练
- TravelUAV seen +14.0%，unseen +7.9%
**OnFly**（arXiv 2603.10682）— M
- 双 agent 频次解耦 + 语义-几何验证器
- 零样本 SR 26.4%→67.8%，Jetson Orin NX 0.81s
**VLA-AN**（arXiv 2512.15258）— M
- 3DGS 域适应 + 三阶段训练 + 几何安全校正
- 机载推理吞吐提升 8.3x
**LiteVLA-H**（arXiv 2605.00884）— M（v3.0 新增）
- 256M 参数机载 VLA，双速率推理
- Jetson AGX Orin：动作分支 19.74Hz，语义输出 6.08-6.67Hz
- 混合三类数据微调：反应式飞行 + 航空语义 + 通用字幕/VQA
- 定位：机载边缘部署的轻量 VLA 参考实现
**CognitiveDrone**（arXiv 2503.01378）— M+B
- VLM 推理 + VLA 执行两段式
- R1 版成功率 77.2%
**AerialVLA（TravelUAV SOTA 版）**（AAAI）— M
- 极简端到端，双视角 RGB，模糊方向提示
- TravelUAV unseen SOTA
- 注意：此为 v2.1 中的 AerialVLA，与下方 AAAI 2026 在线对话版是不同工作
**AerialVLA（AAAI 2026 在线对话版）**（AAAI 2026）— M（v3.0 新增/修正）
- 在线对话导航，UNOD benchmark
- UAV 在导航过程中主动判断何时提问，引入实时问答导航能力评测
- 与 TravelUAV SOTA 版 AerialVLA 是不同工作，需注意区分
- 可作为后续"agent 交互式任务采集"的补充参考
### 3.9 认知评测基准
**MM-UAVBench**（arXiv 2512.23219）— B
- MLLM 感知 / 认知 / 规划，19 类任务
- 上游认知评测锚点
**UAVReason**（arXiv 2604.05377）— B
- 22 类空间/时序 aerial reasoning
- 暴露通用 VLM 在俯视/小目标/重复纹理下的脆弱性

## 四、演进脉络
```plaintext
第一层 Route-following VLN (2023-2024)
  AerialVLN → AVDN → CityNAV
  「按照详细路线指令，一步步飞过去」
      ↓
第二层 Realistic Object Search (2024-2025)
  OpenUAV / UAV-Need-Help → AirNav（137K 真实城市）→ IndoorUAV（室内支线）
  「在逼真仿真/真实场景里，靠连续轨迹完成目标搜索」
      ↓
第三层 Open-World ObjectNav (2025)
  UAV-ON
  「给高层语义目标，在开放世界里自主探索」
      ↓
第四层 High-Level VLA (2026)
  HUGE-Bench
  「理解一句高层命令，安全地做完整套多阶段任务」
      ↓
第五层 Fine-Grained Flow Control (2025)
  UAV-Flow Colosseo
  「语言引导下的精确短程飞行控制」
      ↓
第六层 Tracking-Approach-Align (2025-2026)  ← v3.0 新增
  CosFly-Track → UAV-Track VLA
  「动态目标跟随、可见性保持、视角质量优化、接近对齐」

跨层 Navigation + Manipulation
  AIR-VLA + DroneVLA
  「飞到目标后执行抓取/放置/交接」

跨层扩展 Contact-Rich Aerial Manipulation (2025-2026)  ← v3.0 新增
  AirVLA（π0 迁移+PAG）→ Flying Hand（末端中心+whole-body MPC）→ AERMANI-VLM（分层安全）
  「接触操作、负载变化、末端中心控制、分层安全策略」

边缘部署层 Onboard VLA (2025-2026)  ← v3.0 新增
  LiteVLA-H（256M，19.74Hz）→ OnFly（双 agent 解耦）→ VLA-AN（3DGS 域适应）
  「机载边缘推理，低延迟，真机部署」
```

核心趋势：从「细粒度指令 + 路线跟随」走向「粗粒度目标 + 自主多阶段执行」，同时从「纯导航」拓展到「导航 + 跟踪 + 接近 + 接触操作 + 恢复」，边缘部署成为新的关注点。

## 五、关键缺口

| 缺口 | 现状 | 影响 |
|---|---|---|
| 真实轨迹 + 导航-操作联合数据 | AIR-VLA 和 AirVLA 已部分填补，但城市低空+多类接触+Sim2Real 配对仍为空白 | C3 创新点缺实验支撑 |
| Track/Approach 阶段数据 | CosFly-Track 开始填补但仍不足；UAV-Track VLA 侧重语义跟踪 | 导航→操作中间桥梁缺数据 |
| 负载变化动力学数据 | AirVLA 验证了问题（PAG），但无系统性数据集 | 接触操作后动力学评测无基准 |
| Recovery/失败恢复数据 | 几乎所有工作都忽略 | 抓取失败重试、接触扰动恢复无评测 |
| 机载边缘部署评测 | LiteVLA-H/OnFly/VLA-AN 是少数 | 大部分方法缺乏实测指标 |
| 多机协同 + VLA 联合 | GoalSwarm 只做多机导航 | 与空中操作的交集为空白 |
| 统一评测标准 | SR/SPL vs TCR/CR/CSPL vs 延迟 vs 接触力 | 跨工作不可比 |
| 真实世界 VLA 部署 | UAV-Flow 仅覆盖细粒度控制；AirVLA 做了 pick-and-place | 高层 VLA 真实部署仍为空白 |
| **动力学感知评测** | 几乎所有 benchmark 忽略 | 生成轨迹物理不可行 |
| **Sim2Real Paired Data** | 无一工作提供系统性配对数据 | sim2real gap 无法量化 |

## 六、推荐组合
**训练数据**：OpenFly（100K 自动生成）+ AeroVerse（预训练资源）+ CosFly-Track（跟踪阶段）

**评测-VLA**：HUGE-Bench（高层 VLA）+ UAV-Flow（细粒度控制）
**评测-操作**：AIR-VLA（空中操作基准）
**认知压测**：MM-UAVBench + UAVReason
**方法骨架**：APEX + AirNav（主线）/ AutoFly + AerialVLA（端到端对照）/ OnFly + VLA-AN + LiteVLA-H（机载部署参考）/ AirVLA + Flying Hand（操作方法参考）
