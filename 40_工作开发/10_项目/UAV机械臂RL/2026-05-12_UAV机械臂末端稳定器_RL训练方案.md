## 一、目标理解确认

```
无人机 → 随机飘移干扰 → 机械臂基座
 ↓
 RL策略 (本次训练)
 ↓
 关节动作补偿
 ↓
 末端位姿稳定跟踪
```

**核心本质：主动稳定控制** —— 末端跟踪精度 = Reach任务 + 基座随机扰动 + 抗扰奖励设计。

### 2.2 无人机扰动建模（核心设计）

无人机飘移用**基座随机化+实时扰动**两层叠加模拟：

**Reset时随机化（每episode随机初始基座位置/姿态）**

```python
# EventCfg中
reset_base_pose = EventTerm(
    func=mdp.reset_robot_base_with_noise,
    mode="reset",
    params={
        "pos_range": {"x": (-0.05, 0.05), "y": (-0.05, 0.05), "z": (-0.02, 0.02)},
        "rot_range": {"roll": (-0.1, 0.1), "pitch": (-0.1, 0.1), "yaw": (-0.2, 0.2)},
    },
)
```

**运行时连续扰动（每step施加基座速度/力）**

这是关键：Isaac Lab的EventManager支持 `mode="interval"` 周期性触发，用于模拟持续飘移：

```python
# 低频大幅度漂移 + 高频小幅度振动 叠加
disturb_base_low_freq = EventTerm(
    func=mdp.apply_base_disturbance,
    mode="interval",
    interval_range_s=(0.5, 2.0),  # 每0.5~2秒一次大扰动
    params={"vel_range": (-0.08, 0.08), "ang_vel_range": (-0.15, 0.15)},
)
disturb_base_high_freq = EventTerm(
    func=mdp.apply_base_vibration,
    mode="interval",
    interval_range_s=(0.05, 0.1),  # 高频振动
    params={"force_std": 2.0},
)
```

**课程学习**：从小扰动开始，逐步增大到真实无人机量级。

### 2.3 观测空间设计

```
观测向量 (约42维):
├── joint_pos (7)      关节位置 (相对default)
├── joint_vel (7)       关节速度
├── base_lin_vel (3)    基座线速度 (世界系) ← 关键抗扰信息
├── base_ang_vel (3)    基座角速度 ← 关键抗扰信息
├── base_lin_acc (3)    基座线加速度 (可选,模拟IMU)
├── ee_pose_error (7)   末端位姿误差 (位置3 + 四元数4)
├── ee_pose_command (7) 目标末端位姿命令
└── last_action (7)    上一步动作
```

**核心差异**：比普通Reach任务多了 `base_lin_vel` + `base_ang_vel`，让策略能感知扰动来源主动补偿，而不是被动响应末端误差。

---

### 2.4 奖励函数设计

```python
# 主任务奖励
ee_position_tracking       weight=-0.2  # 末端位置L2误差惩罚
ee_position_tracking_fine   weight=+0.1   # tanh核精细奖励 std=0.05 (比reach更严格)
ee_orientation_tracking     weight=-0.1   # 末端姿态误差惩罚

# 运动平滑性惩罚 (防止抖动)
action_rate                 weight=-0.001  # 动作变化率
joint_vel                   weight=-0.0001 # 关节速度

# 安全约束 (可选)
joint_limit_penalty         weight=-0.5   # 关节超限惩罚
```

精度标准比Reach更严（std从0.1降到0.05），因为目标是"稳定平台"而不是"粗略到达"。

---

### 2.5 课程学习设计

| 阶段 | 扰动幅度 | steps | 目标 |
|------|----------|-------|------|
| Phase 1 | 无扰动 | 0~2000 | 学会基本Reach能力 |
| Phase 2 | 小扰动 (±2cm, ±5°) | 2000~5000 | 适应轻微飘移 |
| Phase 3 | 中等扰动 (±5cm, ±15°) | 5000~8000 | 模拟悬停抖动 |
| Phase 4 | 大扰动 (±10cm, ±30°) | 8000~12000 | 模拟无人机机动 |

通过 `CurriculumTerm` 动态修改 EventTerm 的扰动参数。

## 八、MuJoCo Sim2Sim 工作流方案

**目标**：把 Isaac Lab 训练的策略迁移到 MuJoCo 中验证，测试 sim2sim gap。

### 工作流三步

```
Isaac Lab 训练  →  导出策略  →  MuJoCo 推理验证
     model_1999.pt       ONNX/TorchScript       相同obs/action接口
```

### Step 1：导出策略（ONNX/TorchScript）

```python
import torch

checkpoint = torch.load("model_1999.pt")
actor_state = checkpoint["model_state_dict"]  # 只需要 actor 部分

# 构建同名 actor 模型结构并加载权重
from rsl_rl.algorithms import PPO
actor = PPO.load_actor_from_checkpoint(checkpoint)

# 导出 ONNX
dummy_obs = torch.zeros(42)  # 观测维度对齐
torch.onnx.export(
    actor,
    dummy_obs,
    "policy.onnx",
    input_names=["obs"],
    output_names=["action"],
    dynamic_axes={"obs": {0: "batch"}, "action": {0: "batch"}}
)
```

### Step 2：MuJoCo Franka 环境搭建

MuJoCo 官方有 Franka Panda MJCF（`mujoco/franka_emika_panda`），直接用。关键对齐项：
- 观测顺序：`joint_pos(7) + joint_vel(7) + base_lin_vel(3) + base_ang_vel(3) + ee_cmd(7) + last_action(7)`
- 动作：关节位置 delta，scale=0.5
- 基座：通过 `freejoint` 实现自由漂移（模拟无人机扰动）

## 八、MuJoCo Sim2Sim 验证结果

已完成环境搭建和交互测试，实测静态误差约 7cm，存在明显 sim2sim gap。

### 8.1 环境配置

对齐项（Isaac Lab → MuJoCo）：
- 观测顺序：`joint_pos(7) + joint_vel(7) + base_lin_vel(3) + base_ang_vel(3) + ee_cmd(7) + last_action(7)`
- 动作：关节位置 delta，scale=0.5
- 基座：mocap + weld freejoint 悬浮（模拟无人机悬停）
- Franka Panda：MuJoCo 官方 MJCF（`mujoco/franka_emika_panda`）

### 8.2 交互测试脚本

- 路径：`sim2sim/interactive_test.py`
- 功能：键盘/鼠标实时注入扰动，实时显示末端误差
- 已验证：静态误差 ~7cm（无外加扰动）

### 8.3 Sim2Sim Gap 分析

| 指标 | Isaac Lab | MuJoCo | Gap |
|------|-----------|--------|-----|
| 静态 position_error | 0.013m | ~7cm | ~7cm |
| 扰动 position_error | 测量 | 测量 | 待测 |

7cm gap 来源推测：Isaac Lab 物理引擎（PhysX）与 MuJoCo 关节摩擦/阻尼模型差异，末端执行器累计误差积累。
