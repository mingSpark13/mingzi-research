## 一、核心理念

> AI 时代，算法研发的生产力 = AI 运用效率。
> 用最少 token 干最多的活，让 AI 做 AI 擅长的事，人做人擅长的事。

**时代判断**：
- 在算法开发研究领域，AI 运用效率越高，生产力越高
- 未来的开发范式是**给 AI 做接口、做身体**——MCP 也好、具身也好，都是为 AI 打好躯体框架
- 工具开发面向 AI 使用者，而非传统用户工具

**个人定位**：
- 从 GPT 初期就高强度使用 AI，经历完整进化周期
- 时刻学习新方法，走在 AI 应用前一步，提前探知新应用
- 不断突破 AI 潜力边界

## 三、代码开发生产力体系

### 3.1 多智能体编排开发

**工具链**：VSCode + Codex + Copilot，安全工作空间隔离

**编排架构**：
```
Claude 4.6 Opus（旗舰）── 制定详细分工计划 + 验收 + 主线任务
       ↕ py 交互工具（实时通信）
Codex（高效执行）── 快速完成分配的代码开发任务
       ↕
人（我）── 代码检查 + 方向指导 + Bug 修复方案 + 高效方案指引
```

**核心特点**：
- 研究与代码始终使用旗舰模型，不降级
- 多 Agent 同时工作，通过 py 工具按规范实时交流讨论
- 人机协作：我提供方向指导，Agent 执行高效开发

### 3.2 收敛的工作空间管理方法

每个工作空间配备：

| 组件 | 作用 |
|------|------|
| **py 交互工具** | Agent 间规范高效通信、讨论、任务分配 |
| **md 开发准则** | 多智能体合作规范、代码风格、架构约定 |
| **持久工作记录** | py 自动存储，自动向量化维护开发记录 |
| **工作空间索引** | 统一收敛的简介，新会话无需重新梳理仓库 |

**效果**：
- 新会话也能记得之前的任务（向量化上下文注入）
- 高质量低成本上下文（专属工作空间的精炼记录）
- 无需每次重新梳理仓库架构（索引简介即用即查）

## 四、AI 个人助手体系

### 4.1 自研助手架构（OpenClaw 爆火前已独立完成）

> 从全 md 规则助手起步 → 发现纯 md 给模型带来大量 token 负担 → 秉承「用最少 token 干最多活」原则 → 开发模块化架构

**五大模块**：

#### 🫀 器官模块（Organ）
> 能不让主模型亲自动手的，就交给器官自动化

| 器官 | 功能 |
|------|------|
| 状态器官 | 自动更新时间/天气/系统状态 |
| 语音器官 | 自动语音→文本处理 + TTS 输出 |
| 记忆器官 | 自动记忆存取、压缩、向量化 |
| 信息器官 | 配套信息自动提供，无需主模型调用 |

**设计哲学**：主模型只需清晰履行灵魂 + 完成任务编排，其余全部自动化

#### 🧠 直觉模块（Intuition）
> 提前判断 + 提前处理，减少工具调用和思考过程

- **技能按需加载**：不把所有工具一次性暴露，直觉模块提前判断本次任务需要的技能，按需加载给主模型
- **预处理加速**：提前判断需要截图 → 自动截图完成 → 主模型无需手动调用截图工具
- **效果**：大幅提升响应回复速度

#### 🎭 编排模块（Orchestration）
> 按能力特长分配子代理

| 任务类型 | 分配模型 | 原因 |
|---------|---------|------|
| 研究型任务 | Claude | 深度推理 |
| 代码开发 | GPT Codex | 高效执行 |
| 日程管理 | MiniMax | 快速响应 |
| 长文本分析 | Kimi | 超长上下文 |

#### 💾 记忆模块（Memory）
> 自动化高效记忆注入

- 向量化存储 + 按需检索
- 跨会话记忆连续性
- 主模型使用简单快速模型即可，记忆自动注入补充上下文

#### 🛠️ 技能模块（Skill）
> md 精炼 + py 工具配套

- md 中只暴露最原始抽象简单的 API 用法描述
- 复杂逻辑收敛到 py 脚本工具
- 减少 token 同时减少工具调用负担

### 4.2 整体效果

通过五大模块编排，实现：
- ✅ **极低成本**
- ✅ **极快响应**（人性化回复速度）
- ✅ **功能完善强大**
- ✅ **多智能体合作**
- ✅ **灵魂遵守不丢失**

### 4.3 工具开发策略

- 接入 **Dify** 直接使用已有的众多 MCP 工具轮子

## 五、交互前端体系

### 5.1 多前端统一架构

> 秉承高质量的统一 AI 交互体验

**同时支持的前端**：

| 前端 | 输入 | 输出 |
|------|------|------|
| CLI 终端 | 文本/命令 | 文本回复 |
| Web UI | 文本/语音/图片 | 富文本/特效 |
| 游戏内接口 | 游戏事件/文本 | 游戏行为/对话 |
| Live2D | 语音/文本 | 语音+动作同步 |
| 现实机器人 | 语音/传感器 | 语音+物理动作 |

**统一机制**：
- 所有前端的交互信息（语音/文本）统一传递给主模型
- 主模型回传同步到各前端：Live2D 动作、游戏表现、UI 特效、机器人动作

- 获得生动的 AI 游戏交互感知同步体验
