# RoboClaw - 首个机器人自我进化Agent框架

## 三句话摘要
1. **核心问题**：真实世界机器人训练中"机器人干5秒，人reset 30秒"的数据采集效率痛点，长期依赖人工复位导致数据成本极高。
2. **解决方案**：提出 Entangled Action Pairs（EAP）机制，让机器人同时学会"执行动作"和"恢复动作"，实现自主数据采集和自我环境复位。
3. **核心贡献**：首个让 Agent 直接参与数采全流程的机器人框架，VLM 作为智能指挥官调度 VLA skills，通过闭环迭代实现机器人自我进化。

## 核心方法

### 1. 纠缠动作对 Entangled Action Pairs (EAP)
- **前向动作**：执行任务（如拿起盖子放入篮子）
- **逆向动作**：环境复位（把盖子放回原位）
- **核心思想**：机器人像玩溜溜球一样自主练习同一技能，无需人类帮助复位
- **意义**：数据和恢复过程成对纠缠，训练不间断

### 2. 智能指挥官 VLM Controller
- 利用视觉语言模型（VLM）作为"大脑"
- 进行高层推理，动态调度各种 VLA skills（Policy Primitives）
- Skill 不是单点预设动作，而是 VLA 级别的技能
- 使用 GPT5.4 接口作为 Agent 监督，判断各 subtask 完成状态

### 3. 闭环学习（Closed-Loop Learning）
- 机器人不再训练完就一成不变
- 执行过程中不断积累经验，通过多次 Rollout 自我优化
- 极少人工干预下的持续自主训练

## 核心创新点

| 创新点 | 说明 |
|--------|------|
| EAP 机制 | 首个让机器人学会"逆向复位"的框架，解决数据采集效率痛点 |
| VLM 调度 VLA | Agent 调度不再是固定 API 调用，而是调用 VLA skill |
| 自我进化 | 执行+复位+采数+学习合为一体，机器人自己越来越强 |

## 实验结果

- **复杂任务成功率**：比传统方法提升 **25%**
- **人工干预时间**：减少 **53.7%**
- **长程任务（插唇膏）**：从 2/50 → 23/50（五轮自主迭代后）

> "OpenClaw 像是给机器人装了一个会调度的项目经理，RoboClaw 像是在这个项目经理下面，养出了一批会自己成长的员工。"

- RoboClaw：试图把底层也变成可持续进化的部分
- 未来愿景：给机器人一个初始 skill，它在真实世界里自己越来越强
