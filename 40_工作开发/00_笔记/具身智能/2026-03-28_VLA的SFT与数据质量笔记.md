# VLA 的 SFT 与数据质量评估笔记

## 一句话总览

> **VLA 的 SFT = 多模态条件行为克隆**：给定视觉观测 + 语言指令 + 机器人状态，监督模型预测专家动作或决策轨迹。本质是「感知输入 + 语言输入 → 动作输出」的监督学习，而非单纯文本对话。

---

## VLA 的 SFT 本质

### LLM SFT vs VLA SFT

| 维度 | LLM SFT | VLA SFT |
|------|---------|---------|
| 监督目标 | 文本输出 | 动作 / 轨迹 |
| 输入 | 文本指令 | 图像 + 状态 + 语言 |
| 误差敏感性 | 较低（语义相近即可）| 极高（一步错全盘错）|
| 时间一致性 | 无要求 | 极强（误差沿时间累积）|
| 多模态对齐 | 无 | 必须严格（图像/状态/动作/语言四对齐）|

---

## VLA SFT 的四种输出形式

### 1. 监督单步动作
$$(o_t, l, s_t) \rightarrow a_t$$
- 机械臂末端位姿、关节角速度、gripper 开合
- 无人机速度命令、base 移动命令

### 2. 监督动作序列（Action Chunking）
- 预测未来 K 步动作（如 8 步 / 16 步）
- 提升时序一致性，减少抖动

### 3. 监督高层 Action Token
- 动作离散化为 token（如 move-left-small、close-gripper）
- 训练形式接近 LLM 的 next-token prediction

### 4. 监督子目标 / 中间表示
- 输出：子目标点、waypoint、skill label、affordance 区域
- 下游 controller / policy 执行

---

## VLA 数据长什么样

一条 VLA SFT 样本：
$$\{(o_1, s_1, a_1), (o_2, s_2, a_2), \dots, (o_T, s_T, a_T), l\}$$

| 符号 | 含义 |
|------|------|
| $o_t$ | 视觉观测（图像/视频/深度/点云）|
| $s_t$ | 机器人本体状态（关节角/速度/末端位姿）|
| $a_t$ | 专家动作 |
| $l$ | 语言任务描述 |

---

## 数据来源

### 1. 人工遥操作示教
- 设备：VR / 手柄 / 3D 鼠标 / 力反馈设备 / kinesthetic teaching / 飞行遥控器
- 优点：动作质量高
- 缺点：成本高、规模小

### 2. 仿真环境自动采集
- 控制器 / 规划器 / RL 教师 / Oracle policy 生成
- 优点：量大、便宜、可覆盖长尾
- 缺点：sim2real gap、视觉不真实

### 3. 现有机器人数据集拼接
- 统一坐标系统一、动作表示统一、语言模板统一
- 类似 LLM 混合 corpus，但难度更高

### 4. 视频 / 图文弱监督迁移
两段式训练：
1. 视觉-语言预训练（看懂场景 + 理解语言）
2. 机器人动作数据 SFT（学会行动）

---

## 数据质量六维评估

> **高质量数据 = 对齐 × 成功 × 平滑 × 高效 × 安全 × 语义清晰**

### ① 对齐（最重要）
- 当前图像 ↔ 当前动作 ↔ 当前状态 ↔ 语言指令 → 必须四对齐
- 任何时间偏移都会严重污染 SFT

### ② 成功
- 任务是否完成（粗筛第一关）
- 注意：成功 ≠ 高质量，可能是"勉强成功"

### ③ 平滑稳定
- 无动作尖峰、无高频抖动、无状态跳变
- 飞行器对动作抖动**极其敏感**

### ④ 高效
- 路径不过绕、时间不过长、动作不冗余、能量消耗合理

### ⑤ 安全
- 无危险姿态、无过度贴近障碍、无鲁棒性差的侥幸成功
- **飞行器尤其要剔除"成功但危险"的轨迹**

### ⑥ 语义清晰
- 语言标签具体明确（"pick up the red mug on the left side of the table"）
- 不是"do the task"、"move there"

## 常见毒数据

| 类型 | 问题 |
|------|------|
| 错误示范 | 操作者自己做错了仍被收入训练集 |
| 失败恢复段 | 前半已搞砸，后面在补救，未标注导致学到奇怪策略 |
| 风格冲突 | 不同操作者习惯差异大（从左抓 vs 从右抓），模型学成平均动作 |

---

## 质量筛选方法

1. **任务结果打分**：success rate、轨迹长度、碰撞次数、任务时间
2. **轨迹统计异常检测**：动作尖峰、抖动、状态跳变、时间戳异常
3. **人工抽查关键片段**：可视化 + 动作回放，看是否"成功但很脏"
4. **Reward / Critic 再评分**：训练 reward model 综合打分（成功+平滑+效率+安全+语义）
5. **多示范一致性筛选**：同场景多条轨迹比较，剔除离群轨迹

---

## 四阶段训练 pipeline

类比 LLM 的预训练 → instruction tuning → preference alignment：

| 阶段 | VLA | 内容 |
|------|------|------|
| 第一层 | 视觉-语言预训练 | 学"看懂图 + 听懂话" |
| 第二层 | 大规模机器人示教 SFT | 学"基本动作映射" |
| 第三层 | 高质量精品数据再 SFT | 干净 + 稳定 + 复杂expert数据拉高性能 |
| 第四层 | RFT / RL / Reward 优化 | 成功率 + 安全性 + 推理能力 |

---

## 飞行机器人 / 空中操作特别注意点

1. **动作标签稳定性**：速度命令是否平滑、yaw/pitch/roll 变化是否合理、末端与机体是否解耦清晰
2. **严格的时间同步**：相机帧时间戳 = 状态估计时间戳 = 控制输出时间戳，**延迟是飞行器的致命问题**
3. **剔除"成功但危险"轨迹**：靠近障碍太近、机体姿态过激、arm-body coupling 扰动大
4. **覆盖不同扰动与初值**：不同起始位姿 / 目标位置 / 障碍布置 / payload / wind / arm pose
5. **高层意图分层标注**：高层子目标 / whole-body reference / waypoint → NMPC / WBC 执行（与 RAPID-UAM 思路对接）

---

## 一句话总结

> **VLA 的高质量数据 = 对齐 × 成功 × 平滑 × 高效 × 安全 × 语义清晰**，比 LLM 更难因为动作误差沿时间累积且必须多模态严格同步。
