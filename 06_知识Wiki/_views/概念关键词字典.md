# 概念关键词字典 v1.1（权威白名单）

> 本字典是知识 Wiki 的唯一概念命名真源。L1 笔记的 `tags` / `related_concepts`、L2 `concepts/` `sources/` 命名必须以本字典为准。

## 版本变更（v1 → v1.1）

- v1：一级 11 / 二级 78 / 别名 ~140 / 黑名单 ~30（细粒度过度）
- v1.1：**一级 11（不变）/ 二级 47 / 别名扩充 / 黑名单扩充**
- 主人明子指示：L1 笔记 tags 应挂在"大概念、重点概念"上，以建立知识树连接；零散细粒度一律下放为别名
- 疑难 3 个全部下放：`评测基准` → 跨载体泛化别名；`物理AI` → 物理一致性别名；`人形机器人` → 跨载体泛化别名

## 使用规则

1. **L1 阅读笔记**：`tags` ≤5 个、`related_concepts` ≤5 个，必须是字典规范名
2. **新概念提案**：写入 `inbox.md`，由知识库管理者 cron 审核后批准入字典
3. **别名表**是只读映射，纠偏代理依此重写 L1 旧 tag
4. **黑名单**标签：发现即删除，不得用作 tag
5. 一级概念**只用于导航**，L1 `tags` 填写二级概念（一级仅在 `related_concepts` 粗层标注时使用）

## 二级概念（47 个）

### 世界模型（4 个）
- **隐空间世界模型** — 在 latent space 中建模环境动力学，支持高效想象式规划
- **物理一致性** — 世界模型生成的状态/视频符合物理定律（重力/接触/摩擦/能量）
- **动作条件预测** — 以机器人 action 为显式条件输入，预测动作施加后的世界状态
- **视频生成** — 面向具身任务的视频/场景生成模型（含多视角一致性、视觉生成）

### VLA（5 个）
- **VLA架构** — 视觉 + 语言 + 动作统一端到端模型的核心架构（含微调策略）
- **ACT动作分块** — 通过 Transformer 对动作序列做 chunking，一次预测多步
- **扩散策略** — Diffusion Policy，用条件扩散建模多模态动作分布
- **流匹配** — Flow Matching，学习连续速度场完成噪声→动作的迁移
- **多模态统一架构** — 用统一 token/表征处理视觉、语言、动作、触觉等多模态

### 空地迁移（3 个）
- **Sim2Real** — 从仿真到真实世界的训练与部署迁移（含域适应/Domain Adaptation）
- **跨视角地理定位** — 空中—地面视角的地理位置匹配与对齐
- **空中操作** — 无人机与机械臂/抓取器结合的空中交互作业

### 跨载体泛化（3 个）
- **跨载体泛化** — 不同形态机器人之间的策略迁移（含形态泛化、人形机器人、评测基准）
- **动作空间统一** — 跨异构机器人的动作表示与接口统一
- **PMI** — Perceptual-Motor Interface，主人明子原创的可验证意图接口理论

### 数据飞轮（4 个）
- **数据合成** — 通过模型生成训练数据（含真实机器人数据、大规模数据集）
- **遥操作** — 人工远程操控机器人采集专家示范
- **自动标注** — 用大模型/启发式对原始数据自动打标
- **仿真平台** — Isaac Sim / Genesis / MuJoCo 等仿真环境

### 空中视觉语言导航（4 个）
- **空中VLN** — 无人机的 Vision-Language Navigation 任务设定与方法
- **单目无地图导航** — 仅单目相机、无预建地图的在线导航
- **语义导航** — 基于语义目标/场景图谱的导航（含场景图谱）
- **无人机避障** — 无人机在复杂环境的障碍感知与安全飞行

### Isaac强化学习（4 个）
- **强化学习** — 通过奖励信号优化策略（含稀疏奖励、博弈强化学习）
- **灵巧操作** — 高自由度末端完成精细接触、装配、手内操作
- **抓取检测** — 机器人抓取前置感知，输出候选位姿
- **课程学习** — 由易到难分阶段训练，加速复杂任务收敛

### 感知与3D视觉（5 个）
- **3D重建** — 从图像/点云恢复三维结构（含 3D场景流、4D重建）
- **3D高斯溅射** — 3D Gaussian Splatting，高效可微的三维场景表示
- **深度估计** — 从单目/双目/视频恢复场景深度
- **6D位姿估计** — 6 自由度物体位姿估计，操作前置感知
- **SLAM** — 同步定位与建图（含相机-激光雷达融合）

### 规划与控制（6 个）
- **MPC** — Model Predictive Control，滚动优化的经典控制框架
- **最优控制** — 在约束下求解使目标最优的控制序列（LQR/变分法等）
- **运动控制** — 机器人底层运动/轨迹执行控制
- **任务与运动规划** — 联合高层任务分解与低层运动求解（含安全护栏、MAPF、时序逻辑控制）
- **长程任务规划** — 跨多步骤的长程目标规划（含 LLM规划、LLM推理）
- **实时推理** — 在低延迟约束下完成生成/控制/场景理解

### 学习范式（4 个）
- **模仿学习** — 从专家示范中学习策略
- **零样本泛化** — 无需目标域训练即可迁移的泛化能力
- **主动感知** — 系统主动改变观察位置或策略以获得更有利信息
- **持续学习** — Continual Learning，在任务流中不断吸收新知识而不灾难遗忘

### 具身智能（5 个）
- **LLM驱动机器人** — 以大语言模型作为高层决策/调度核心驱动机器人
- **腿足机器人** — 四足/人形等腿足平台的控制与规划
- **全身协调运动** — 多部位协同生成动作以满足任务与约束
- **多机器人协调** — 多机器人共享环境的协同规划、避碰、任务分配
- **力-触融合** — 融合力觉/触觉/视觉等多模态接触信号
