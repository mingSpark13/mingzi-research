# 总览：世界模型技术路线图

## 一句话结论

D01 现在最稳的主线不再是“更大全能世界模型”，而是 **deployment-time supervisor**：核心回答执行前预筛、`F1已知失败 / F2未知异常` 分诊、阶段化路由与 hover-bounded recovery 是否真的带来**净安全收益**，而不是只换来更复杂的系统。

## 当前技术格局

### 7 条主线

| 主线 | 代表方法 | 当前价值判断 | 成熟度 |
|------|---------|-------------|--------|
| **隐式 Latent WM** | DIAL, LatentPilot | 适合作为低成本 planning/evaluation 底座，但物理一致性仍弱 | 🟡 |
| **动作中心 WM** | WAM, Fast-WAM, GigaWorld-Policy | 直接服务动作决策，适合比较“会生成”与“会执行”差异 | 🟡 |
| **多模态/触觉 WM** | OmniVTA, CMLF | 更贴近接触 rich 任务，但数据成本高 | 🔴 |
| **组合式/分层 WM** | RISE, Hierarchical Planning | 适合支撑分诊、阶段化路由和恢复协议 | 🟢 |
| **显式物理 WM** | Digital Twin WM | 真机前可解释性强，但建模维护成本高 | 🟡 |
| **Physics-aligned WM** | [[sources/ABot-PhysWorld_2603.23376|ABot-PhysWorld]], EVA-style executability alignment | 最适合承接“物理真”与“动作可执行”双重验收 | 🟢 |
| **训练器/评测器/验证器** | WorldEval, WAV, [[sources/2026-04-10_A1|Interactive World Simulator]], Dream2Fix | 已成为 D01 当前最贴近部署的主战场 | 🟢 |

## 关键 Gap

1. **部署前净收益还需单独证明**：不能再用 rollout 漂亮替代 `danger-action 漏放率 / late stop / misroute` 的真实改进。
2. **接口层价值被低估**：`rank_score / triage / route_action` 可能比继续扩主干更决定部署上限。
3. **恢复线必须阶段化**：当前更像 `hover-bounded / phase-bounded recovery`，不再默认宣传通用在线恢复器。
4. **空中操作双动力学仍未被直接建模**：飞行动力学与操作接触动力学耦合仍是后续真正的方法壁垒。

## 我们的切入点

- **主叙事收束**：从全能世界模型收成 `interface-first / stage-aware supervisor / hover-bounded recovery` 三路竞争，并以 `danger-action 漏放率 / anomaly recall / human handoff burden` 等核心安全指标决定唯一主线。
- **最新推进（2026-05-16）**：D01 已把部署前验收扩展为 `预筛 → F1/F2 分诊 → R1/R2/R3 恢复 → SRY/AER/HHB/RDR/RCS 统一记账`，不再接受只报 rollout reward 或 recovery success rate 的路线。
- **当前默认标题方向**：若分诊与恢复统一记账后仍显著改善净安全收益，优先收成 **stage-aware route supervisor**；若增益主要来自接口预筛，则收成 **interface-first execution supervisor**；若恢复债务过高，则退回 **hover-bounded recovery** 辅线。
