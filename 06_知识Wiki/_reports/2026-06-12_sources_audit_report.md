## 📊 二、sources/ 现状体检（全目录 1027 个文件）

### 1️⃣ ViSA-Enhanced AVLN 错填（**最严重·173 个**）
- **H1 标题**统一是 `# ViSA-Enhanced AVLN (2603.08007)`（应为各自论文真名）
- **YAML 头 id / tags / summary / origins 全部正确**（未被污染）
- **波及模式**：cron 2026-06-02 11:23 用 `sed` 批替换 origins 时，替换模式太宽，把 H1 一起误改了
- **影响**：Obsidian 视图里所有这些 source 的标题都是错的，**但 wiki 链接和数据本身都完好**
- **修复成本**：低（173 个 × 自动从 id 或 origins 推断真名）
- **修复样本**：
  ```
  source.2025-03-13_An-Integrated-Approach-to-Aerial-Grasping-Bistable-Gripper-Adaptive-Control.md
  source.2025-03-13_Flying-Hand.md
  source.2025-03-13_UMI-on-Air.md
  source.2025-03-14_Seamless-Transition-Spring-Legged-Quadrotors.md
  ... (共 173 个)
  ```

### 2️⃣ 命名规则混用（**62 组·轻症**）
- **同一论文 2 种命名风格并存**：
  - `source.2026-04-07_SpaceBlender.md`（日期前缀）
  - `source.SpaceBlender_2409.13926.md`（arxiv 前缀）
  - `SpaceBlender_2409.13926.md`（连 source. 都没）
- **根因**：早期 cron 没强制命名规范，后期 cron 改了规范，**旧文件没改**
- **影响**：Obsidian 出现"重名但不同 id"的混淆
- **修复成本**：低（合并为 1 份 + 改 wiki 反向引用）

### 3️⃣ "orphan 占位"空 source（**55 个**）
- **特征**：summary 写"⚠️ 论文已被引用但 L1 笔记未入库（orphan）。暂作占位索引。"，正文只有 5-7 行
- **设计目的**：被引用但 L1 笔记暂未入库时，先用占位 source 记录引用关系
- **本次发现**：这 55 个里，**有 32 个的 L1 笔记其实已经入库**（origins 指向的文件存在且非空）—— **该被"补全"而非继续占位**
- **修复成本**：中（32 个 × 从 origins 指向的 L1 笔记读取真名 + summary，剩下 23 个确认真无 L1 后保留占位）

### 4️⃣ "弱 source"——summary 字段空（**29 个**）
- **特征**：有 origins 但 summary 字段是空字符串
- **修复成本**：低（从 origins 指向的 L1 笔记读一段填充）

### 5️⃣ "无 tags"——tags 字段空（**82 个**）
- **特征**：tags = [] 空数组
- **修复成本**：低（从 summary 自动抽取关键词）

### 6️⃣ "无 origins"（**73 个**）
- **特征**：origins = [] 空数组
- **可能原因**：① 真的无 L1 笔记（公众号/视频类）② cron 没填
- **修复成本**：高（73 个逐一人工判断属于"公众号类无需 origins"还是"该删"）
