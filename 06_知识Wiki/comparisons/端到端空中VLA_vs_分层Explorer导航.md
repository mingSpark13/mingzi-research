# 对比：端到端空中VLA vs 分层Explorer导航

## 比较对象简介

| 路线 | 核心思想 | 代表来源 |
|------|---------|---------|
| **端到端空中VLA** | 直接从视觉/语言观测输出连续控制 | [[sources/source.2603.14363_AerialVLA]], [[sources/source.2604.14125_HiVLA]] |
| **分层Explorer导航** | 高层语义探索/目标推理 + 中层地图/前沿选择 + 低层控制执行 | [[overview/方向_空中VLN_技术路线图]], [[sources/OnFly_2603.10682]], [[sources/source.2604.08883_HTNav]] |

## 详细对比

| 维度 | 端到端空中VLA | 分层Explorer导航 |
|------|---------------|------------------|
| **系统形态** | 单模型直接控 | 规划/探索/控制解耦 |
| **训练门槛** | 依赖大规模示范与对齐数据 | 模块可分开训练与替换 |
| **开放词汇搜索** | 弱，常受限于训练分布 | 强，可显式加入语义评分与前沿探索 |
| **3D地图记忆** | 通常隐式 | 通常显式，可维护体素/拓扑地图 |
| **动力学约束注入** | 不自然，多靠数据隐式学到 | 自然，可在代价函数/规划器中显式加入 |
| **可解释性** | 低 | 高 |
| **故障恢复/重规划** | 弱 | 强 |
| **导航→操作衔接** | 需额外头或后处理 | 更容易通过中间状态/地图自然衔接 |
| **部署链路** | 短，工程简单 | 长，模块集成更复杂 |
| **适合场景** | 目标明确、闭集任务、低延迟演示驱动控制 | 开放场景搜索、长程任务、约束明显的空中导航 |

## 结论

- 若目标是**快速做出可飞的端到端 baseline**，端到端空中VLA更合适。
- 若目标是**开放世界搜索、导航-操作统一、动力学可控**，分层Explorer导航更有研究空间。
- 对明子的 D06 主线来说，最优策略不是二选一，而是：**高层用分层Explorer保持可解释与可重规划，中低层吸收VLA/flow-matching动作建模能力**。
