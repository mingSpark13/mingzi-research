# 概念：Latent 世界模型（Latent World Model）

## 定义

Latent 世界模型是在**隐式低维latent space**中学习环境动力学的世界模型。它不直接预测高维像素级未来帧，而是将环境状态编码为紧凑的latent表示，再在该空间中进行想象、规划和策略优化，从而绕过像素级预测的巨大计算开销。

## 核心原理

### 典型架构（DIAL / LatentPilot / WAM）

```
观测 o_t → Encoder ε → latent state z_t
                ↓
        World Model p(z_{t+1} | z_t, a_t)
                ↓
        Decoder θ → 预测观测 ō_{t+1}
                ↓
        RL Policy  π(a_t | z_t)  via imagined rollout
```

1. **编码**：将高维观测（图像+本体感知）编码为低维latent state $z_t$
2. **动态预测**：World Model 学习 $p(z_{t+1}|z_t, a_t)$，预测未来latent状态
3. **解码**：可选地解码回像素空间用于可视化或重建损失
4. **策略学习**：在latent space中用Dreamer系列方法进行 imagined rollout，更新策略

### 关键设计：Action-Aware Latent

WAM (2603.28955) 在标准 latent WM 基础上增加 **inverse dynamics loss**，让 latent 表示同时编码"发生了什么变化"和"执行了什么动作"，使 latent space 更适合动作规划。

### 与显式物理世界模型的根本区别

| 维度 | Latent 世界模型 | 显式物理世界模型（Digital Twin） |
|------|----------------|-------------------------------|
| 表示方式 | 隐式学习，端到端 | 物理方程/仿真引擎，手工建模 |
| 数据需求 | 需要大量轨迹数据 | 需要精确物理参数标定 |
| 泛化能力 | 数据驱动，泛化强 | 物理先验，特定场景精准 |
| 部署方式 | 神经网络直接推理 | 数字孪生 + 仿真迁移 |
| 适用场景 | 数据丰富、形态多样 | 参数可标定、场景可控 |

## 优缺点

**优点**
- 计算效率高：latent space 维度远低于像素空间，rollout 速度快 10-100x
- 数据驱动泛化：无需手工物理建模，可自然迁移到新场景
- 端到端可微：可与策略联合优化
- 支持长时序想象：latent space 的非线性预测比像素空间更稳定

**缺点**
- 缺乏物理一致性保证：latent space 可能学习到虚假相关性
- 训练不稳定：kl-balance、reconstruction vs prediction 的权衡
- Hallucination 风险：imagined rollout 可能偏离真实动力学（WoVR 专门研究此问题）
- 解释性差：latent state 语义不透明，难以 debug
