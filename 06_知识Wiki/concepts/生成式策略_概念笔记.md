## 1. 什么是生成式策略？

核心思想：把决策问题建模成**条件分布**生成问题。

$$
p(a_t \mid o_t, g) \quad \text{或} \quad p(a_{t:t+H} \mid o_{1:t}, g)
$$

其中 $o$ 是观测，$g$ 是目标/语言指令，$a$ 是动作，$H$ 是未来动作 horizon。

### 关键特征

不是只输出一个固定答案，而是学习：**在这个场景下，合理行为的分布是什么**。

例如机器人看到杯子，指令是"把杯子拿起来"，可能有很多合理动作：
- 从左边伸手抓
- 从右边伸手抓
- 先调整底盘再抓
- 先靠近再抬手
- 避开障碍后绕过去抓

生成式策略学的是一个动作分布/轨迹分布，然后从里面**采样或生成**一个动作序列。

### 涵盖范围

Diffusion Policy、Flow Matching Policy、ACT、VLA action tokenizer、world-model rollout 都可以归到"生成式策略"范畴。

## 2. 非生成式策略分类

### A. 确定性回归策略 / 直接回归策略

$$
a = f(o, g)
$$

输入图像、状态、语言目标，网络**直接回归一个动作**。

典型例子：
```
image + instruction → [vx, vy, vz, yaw_rate, gripper]
```

**问题**：如果一个场景有多个合理动作，回归模型容易学出"平均动作"。例如专家数据里一半从左绕，一半从右绕，回归模型可能输出"直着往中间撞"——这就是**多模态问题**。

- 英文：deterministic policy / regression policy / direct action regression
- 中文：确定性策略、回归式策略、直接动作回归策略

### B. 判别式策略 / 判别式模型

判别式模型不负责生成完整动作，而是**判断、分类、打分**。

$$
p(y \mid x)
$$

例如：
```
当前图像 + 候选动作 → 这个动作好不好？
图像 → 目标类别
图像 + 动作 → 成功概率
状态 → Q 值
```

典型例子：传统 RL 里的 Q-learning / DQN，本质上学一个动作价值函数，然后选最大值动作：

$$
a = \arg\max_a Q(s,a)
$$

- 英文：discriminative policy
- 中文：判别式策略、价值评估策略、打分式策略

### C. 规划/优化式策略

不是从模型里直接生成动作，而是**在线求解优化问题**。

例如 MPC/NMPC：

$$
\min_{u_{0:H}} J(x,u)
$$

每一帧都解一个优化问题，找未来一段最优控制输入。

- 英文：planning policy / optimization-based policy / model predictive control policy / trajectory optimization policy
- 中文：规划式策略、优化式策略、模型预测控制策略

**混合路线**：世界模型 + MPC，或者生成轨迹作为 MPC 初值。

### D. 规则式/反应式策略

```python
如果目标在左边 → 向左转
如果距离太远 → 前进
如果距离太近 → 后退
```

不学习复杂分布，也不生成长 horizon 动作序列，只是根据当前状态触发规则。

- 英文：rule-based policy / heuristic policy / reactive policy
- 中文：规则策略、启发式策略、反应式策略

## 3. 各领域中的"生成式"理解

### VLA：生成动作 token

动作被离散化成 token，模型像生成文字一样生成动作：

```
图像 + 语言指令 → action tokens
```

本质：

$$
p(a_1, a_2, ..., a_H \mid image, instruction)
$$

例如 `move_forward, turn_left, arm_up, gripper_close`。

### 世界模型：生成未来状态

世界模型不一定直接生成动作，但会**生成或预测未来**：

```
当前状态 + 动作 → 未来状态
当前图像 + 动作序列 → 未来视频/latent
```

所以它生成的是：

$$
p(s_{t+1:t+H} \mid s_t, a_{t:t+H})
$$

然后策略可以在这个"想象出来的未来"里做规划。

### 模仿学习 (IL)

| 类型 | 方式 |
|------|------|
| 普通 Behavior Cloning | $observation \to expert action$（回归式） |
| 生成式 IL（ACT、Diffusion Policy、Flow Matching） | 学专家动作的**分布**（生成式） |

区别：普通 BC 学专家动作的平均映射；生成式 IL 学专家动作的分布。

### 强化学习 (RL)

RL 本身**不一定是生成式**：
- DQN → 判别/价值式
- DDPG/TD3 → 回归式/确定性策略

但如果是随机策略：

$$
\pi(a \mid s), \quad a \sim \pi(a \mid s)
$$

它就有生成味道（如 SAC）。

通常不直接叫"生成式策略"，除非用 diffusion、flow matching、VAE 等生成模型来表示策略分布。

### Flow Matching：典型生成式策略

Flow Matching 的核心是学一个从**噪声分布到真实动作分布**的连续变换：

```
随机噪声动作 → 沿着 learned flow 逐渐变成合理动作序列
```

不是直接输出一个动作，而是学会"怎么把随机样本搬运到专家动作分布"。非常标准地属于**生成式策略**。

Diffusion Policy 同理，通过多步去噪生成动作序列。

## 4. 一张表总结

| 类型 | 它在做什么 | 典型名字 |
|------|-----------|---------|
| **生成式策略** | 生成动作/轨迹/未来状态 | generative policy |
| **回归式策略** | 直接预测一个动作 | regression policy |
| **判别式策略** | 判断动作好坏/分类/打分 | discriminative policy |
| **价值式策略** | 学 Q/V 值再选动作 | value-based policy |
| **规划式策略** | 在线优化求动作 | planning / MPC |
| **规则式策略** | 按人工规则反应 | heuristic / reactive policy |

## 5. 推荐表述

### 学术表述（PPT / 博士计划）

> 生成式智能策略旨在学习条件动作分布、轨迹分布或未来状态分布，从而在给定观测与任务指令的条件下生成可执行的动作序列或决策轨迹。该范式覆盖 VLA、世界模型、生成式模仿学习、生成式强化学习以及 diffusion/flow matching policy 等路线。

### 与传统方法对比表述

> 区别于直接动作回归、判别式价值评估或基于规则/优化的控制策略，生成式策略更强调对多模态行为分布的建模，能够生成多样化且上下文一致的动作序列。
