# Xiaomi-Robotics-0: An Open-Sourced Vision-Language-Action Model with Real-Time Execution

- **arXiv**: [2602.12684](https://arxiv.org/abs/2602.12684)
- **日期**: 2026-02（v2: 2026-03-25）
- **方向**: D02 VLA

## 一句话总结
小米把 VLA 的重点从“只拼效果”拉回到“真机实时可连续执行”，核心抓手是跨载体预训练、异步执行训练、以及 action chunk 时序对齐。

## 核心贡献
1. 大规模跨载体轨迹 + 视觉语言数据预训练，强化泛化能力。
2. 通过异步执行训练处理真机 rollout 中的推理延迟。
3. 部署时对连续 action chunk 做时间对齐，实现平滑实时执行。
4. 在仿真和双臂精细操作真机任务上都取得很强结果，并且能在消费级 GPU 上运行。

## 与主人研究的相关度
⭐⭐⭐⭐ 这篇最值钱的不是“又一个VLA”，而是它认真回答了主人的核心工程问题：**VLA 怎么在有限算力上真正跑起来**。

## 对 Paper A 的启发
- 可先不追求完整三层超大系统，先把 **延迟可控** 做出来。
- 异步执行机制很适合迁移到空中平台，因为飞行控制天然不能等慢速 VLM 每次完整出结果。
- action chunk 对齐说明低层执行器不一定要逐步重算，可以用滚动式片段续接降低抖动。

## 可落地实验点
1. 用 π₀.₅ / OpenVLA 做基座，单独复现异步执行。
2. 记录 Jetson / 工控机上的 chunk latency、控制抖动、轨迹连续性。
3. 先在 AirSim 验证“异步 chunk + PID/MPC”组合，再决定是否接真机。

## 局限 / 不足
- 仍是端到端 VLA 主体，语义意图层与精细执行层没有真正解耦。
- 论文主要验证双臂操作，对空中平台的高动态约束没有直接回答。
- 摘要显示工程很强，但后续还需补读 method/experiments 细节，确认异步训练的具体实现。
