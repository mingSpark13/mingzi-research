# D07: Isaac 强化学习机械臂控制

> 新建方向 | 待系统研究

## 一、研究目标

在 Isaac Sim 仿真环境中，通过强化学习训练机械臂快速响应策略，实现：
1. 空中机械臂与无人机解耦控制
2. 抵抗无人机扰动漂移
3. 室外无动捕条件下完成精确操作

| 机械臂 RL 策略训练 | 训练快速响应策略，抗干扰精确操作 | 🔴 P0 |
| sim-to-real 迁移 | 从仿真到真机的策略迁移 | 🟡 P1 |

### 关键发现
- **Isaac Lab = Isaac Gym的下一代替代者**，适合大规模并行RL训练
- Isaac Sim专注高保真单场景，Isaac Lab专注GPU并行多场景
- **TIAGo案例说明：sim-to-real 不只是换更真模拟器，而是要同时看关节阶跃误差、碰撞约束和控制架构匹配**
- **Dexterous Sim-to-Real** 说明视觉型、接触丰富的灵巧操作也开始跑通 sim-to-real RL，D07 后续不必只盯“状态输入 + 单机械臂”简化设定
- **ARM** 进一步说明长时序 manipulation 的关键瓶颈不只是控制器，而是 **奖励建模**，后续应把 progress supervision / recovery-aware evaluation 纳入协议
- **Robo-Dopamine** 把奖励层再往前推进到 **多视角 + step-aware 过程奖励模型 + policy-invariant reward shaping**，说明 D07 不该只比较训练器，还应把“奖励模型质量 / 一次示教适配效率 / 在线 rollout 成本”单列成协议
- **TacEx** 说明“触觉/接触 realism”本身也该被视作基础设施轴，而不只是传感器实现细节；后续可把 **是否需要软体接触仿真、是否能输出可训练的 GelSight/RGB 观测** 单列成基础设施对照
- **Tac2Real** 进一步补强了这条主线，说明 D07 后续不该只问“有没有触觉模拟”，还应把 **interactive-rate visuotactile throughput / multi-GPU online RL compatibility / sim-to-real TacAlign calibration cost** 单列成对照，专门比较高保真触觉是否能在可训练吞吐下成立
- **2604.12628** 直接把 **RL vs pseudo-spectral optimal control** 摆到同一张桌子上，提醒 D07 后续必须保留“学习式策略 vs 最优控制/混合控制”这条强基线，而不是只在 PPO/SAC/FlashSAC 之间横向比较
- **2502.02133** 进一步补了一个方法论缺口：hybrid MPC+RL 至少应区分 **teacher-student（MPC教策略）**、**safety shield（MPC做在线安全兜底）**、**warm-start / policy prior（策略给优化器初值）**、**optimizer-on-top-of-policy（策略提案+优化修正）**，否则“hybrid”很容易写成含糊口号
- 机械臂+Isaac的sim-to-real链路已基本打通（TIAGo案例）
