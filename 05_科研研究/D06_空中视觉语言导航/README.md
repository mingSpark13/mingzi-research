# D06: 空中视觉语言导航 (Aerial Vision-Language Navigation)

> ★ 当前重点方向 | 与龙虾 GoalSearch 项目直接对齐 | 可拓展为导航+操作

## 研究背景

视觉语言导航 (VLN) 在地面机器人已取得显著进展（VLFM/NaVILA/ApexNav 等），但**空中 VLN 几乎是空白**。现有地面方法假设平面运动、低速、稳定视角，无法直接迁移到无人机——6-DoF 运动、高速、大视角变化、动力学约束是根本差异。同时，空中 VLN 天然可拓展为**导航+操作**（先飞到目标 → 再执行抓取/放置），形成完整的空中具身智能闭环。

## 核心研究问题

1. **地面 VLN → 空中 VLN 的迁移瓶颈**：视觉表征可迁移吗？动力学 gap 如何弥补？
2. **语言引导的空中自主探索策略**：如何高效探索未知环境找到语言描述的目标？
3. **导航→操作的无缝衔接**：到达目标后如何切换到精细操作模式？
4. **空中视角下的场景理解**：俯仰视角变化大，如何做鲁棒的视觉-语言接地？

## 技术路线图

```
现有方法（地面VLN VLFM/NaVILA）
    │ Gap: 无空中动力学建模, 无6-DoF运动, 无视角大变化处理
    ▼
我们的切入点：空中VLN层次化框架
    │
    ├── 高层：VLM 语义目标理解 + 场景图/前沿图
    ├── 中层：空中感知探索策略（3D前沿选择 + 安全约束）
    ├── 低层：无人机运动控制（考虑动力学约束）
    │
    ▼ 拓展方向
导航+操作：到达后切换操作模式（参考 Unified Aerial Grasping）
```

## 可创新点

1. **C1: 空中视觉语言前沿地图 (Aerial VL-Frontier Map)**
   - 将 VLFM 的 2D 前沿地图拓展为 3D 体素前沿图，适配空中 6-DoF 探索
   - 融合 VLM 语义相关性评分引导探索方向选择
   - **支撑**：VLFM 只做 2D 平面，空中场景需要 3D；龙虾项目已有初步框架

2. **C2: 动力学感知的 VLN 策略 (Dynamics-Aware VLN Policy)**
   - 在导航策略中显式建模无人机动力学约束（能量/速度/安全距离）
   - 参考 π-Make-It-Fly 的 Payload-Aware Guidance 思路
   - **支撑**：现有地面 VLN 完全忽略动力学，这是空中场景的根本差异

3. **C3: 导航-操作统一框架 (Navigate-then-Manipulate)**
   - 导航阶段 → 到达后无缝切换操作模式
   - 导航时积累的场景理解直接为操作提供上下文
   - **支撑**：Unified Aerial Grasping 做了初步尝试，但没有与 VLN 框架结合

## 与龙虾项目的关系

> 🏁 **龙虾 GoalSearch 已完结**（2026-04-16）—— 任务目标达成，不再需要多操心。

龙虾项目实现的是：**语音命令 → 视觉理解 → 自主探索 → 定位目标**，这就是空中 VLN 的工程实现。项目经验可直接转化为论文实验数据。
