# 自动化研究链路质量审计（Claw 定时代理）

> 日期：2026-05-13  
> 范围：D01-D07 研究方向、阅读笔记、source 页面到 LLM Wiki 的沉淀链路

## 1. 现状判断

1. **研究推进文档化过重**：`REPORT/PAPER` 中 route/freeze/gate 叙事密度高于实验结果密度。  
2. **Source→Wiki 证据链不稳定**：部分结论缺“原始 source 明确证据点”，存在二手摘要循环风险。  
3. **概念索引噪声高**：阅读笔记命名和目录粒度不统一，方向内出现重复题材分散记录。  
4. **跨方向可比性弱**：D02/D03 执行层文档相对薄，导致 D01-D07 不易统一验收。

## 2. 机制改进（建议直接纳入代理心跳流程）

### M1. 证据门控（Evidence Gate）
- 新结论入库前必须满足：`Claim + Source证据 + 指标/实验位置 + 证据等级(L0-L3)`。
- 缺任何一项，结论降级为“候选观点”，不可写入方向主结论。

### M2. 写作-实验配额器（Narrative-to-Execution Ratio）
- 设硬规则：当轮新增叙事段落数 ≤ 当轮新增可执行实验项数。
- 超限时，自动把新增内容写入 `rounds/` 草稿区，不进 `PAPER.md`。

### M3. Source 质量分层
- S级：有清晰任务设定、对照、关键表格/曲线、可映射到本方向实验。
- A级：方法清楚但实验细节不全。
- B级：观点性或二手总结。
- 只有 S/A 允许成为“锚点论文”；B 仅做背景。

### M4. 阅读笔记标准模板
- 每篇顶部固定字段：`任务/方法/指标/结果/限制/可复现性/复用价值`。
- 每篇末尾固定字段：`是否进入实验池(Y/N)`、`进入哪条实验线`。

### M5. 概念索引重建
- 每方向 README 增加：
  - `核心概念树（不超过3层）`
  - `Top10 锚点论文`
  - `近14天新增`
  - `待实验验证清单`
- 重复概念统一收敛到“主词条 + 别名”，减少同义多入口。

## 3. 立即可执行清单（1周）

- Day1-2：给 D01-D07 的 OWNER_NOTES 注入统一引导机制（已完成）。
- Day3：补齐 D02、D03 的 experiments 文档骨架。
- Day4：为每个方向挑 1 条最小实验线，输出命令/输入/指标/判停。
- Day5：清理阅读笔记命名和重复条目，建立每方向 Top10 索引。
- Day6-7：跑首轮最小实验并回填证据等级，禁止只更新叙事。

## 4. 预期效果

- 2 周内：方向文档“可执行感”明显提升，主结论与证据对齐。
- 4 周内：D01-D07 可横向比较，能用统一看板做路线裁剪。
- 长期：降低 Claw 定时代理“文档自增殖”风险，形成稳定 research-to-evidence 闭环。
